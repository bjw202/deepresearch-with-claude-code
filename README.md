# Deep & Wide Research with Claude Code

Claude Code의 멀티 에이전트 오케스트레이션을 활용한 **딥리서치 프레임워크**.

하나의 질문을 던지면 여러 서브 에이전트가 병렬로 조사하고, 비평하고, 검증하고, 통합하여 구조화된 리서치 보고서와 프레젠테이션을 자동 생성한다.

---

## 왜 만들었나

일반적인 AI 리서치는 단일 프롬프트 → 단일 응답 구조다. 이 방식은 관점이 편향되기 쉽고, 검증이 없으며, 결과가 휘발된다.

이 프로젝트는 다르게 접근한다:

- **다관점 조사**: 같은 주제를 여러 Researcher가 서로 다른 관점에서 동시 조사
- **비평과 검증**: Critic이 논리적 허점, 상충점, 과도한 확신을 잡아냄
- **축적과 재활용**: 모든 산출물이 구조화된 마크다운 파일로 저장되어 지식이 쌓임
- **프레젠테이션 자동화**: 리서치 결과를 바로 .pptx 슬라이드로 변환

```
│       ├── create-presentation/      # PPT 생성 스킬 + 폰트 + 디자인 시스템
│       └── push/                     # Git push 스킬
├── scripts/
│   ├── search.sh                # 검색 API 통합 스크립트 (Perplexity + Tavily)
│   └── validate-pptx-code.js   # PPTX 코드 검증
├── docs/
│   ├── research/                # 리서치 산출물 (주제별 폴더)
│   │   └── {YYYY-MM-DD}-{topic}/
│   │       ├── 00-synthesis.md      # 종합 보고서 (여기서 시작)
│   │       ├── 01-{관점1}.md        # Researcher 산출물
│   │       ├── 02-{관점2}.md        # Researcher 산출물

│   └── claude-md-adaptation-guide.md  # 다른 프로젝트 적용 가이드
└── user-input-context/          # 리서치 요청 시 참고 자료
```

## 작동 원리

### 멀티 에이전트 오케스트레이션

```
사용자 질문
    │
    ▼
┌─────────────────────┐
│   메인 에이전트       │  문제 정의, Question Expansion, 태스크 분배
│   (오케스트레이터)     │
└────────┬────────────┘
         │
    ┌────┴────┬──────────┐
    ▼         ▼          ▼
Researcher  Researcher  Journal      ← 병렬 발사
  (sonnet)   (sonnet)   (sonnet)
    │         │
    ▼         ▼
┌─────────────────┐
│     Critic      │                  ← Researcher 완료 후
│     (opus)      │
└────────┬────────┘
         │
         ▼
┌─────────────────────┐
│  메인: 최종 통합      │              Synthesis 작성
│  (중복 제거, 상충 정리, │
│   근거 평가, 불확실성)  │
└─────────────────────┘
         │
         ▼
    프레젠테이션 제안
```

### 에이전트 역할

| 역할 | 모델 | 하는 일 |
|------|------|---------|
| **메인** | 현재 모델 | 문제 정의, 에이전트 분배, 중간 검토, 최종 통합 |
| **Researcher** | Sonnet | 자료 조사, 개념 정리, 관점별 상세 보고서 작성 |
| **Critic** | Opus | 논리 허점, 상충점, 과도한 일반화, 누락 관점 탐지 |
| **Verifier** | Sonnet | 사실 검증, 원문 대조, 근거 수준 구분 |
| **Journal** | Sonnet | 세션 기록, 의사결정 로그, TODO 관리 |

### 검색 도구

외부 검색은 `scripts/search.sh`를 통해 수행한다. MCP 서버 의존성 없이 API를 직접 호출하여 서브 에이전트에서도 안정적으로 동작한다.

| 도구 | 용도 | 비용 |
|------|------|------|
| `perplexity search` | 종합 파악, 개념 정리 | ~$0.01/요청 |
| `tavily search` | 구체적 URL 발견, 제품/코드 탐색 | 2크레딧 |
| `tavily extract` | 특정 URL 원문 확보 | 1크레딧/URL |
| `tavily research` | 심층 다면적 분석 | 5크레딧 |
| `perplexity research` | 심층 합성 | ~$0.01 |
| `perplexity reason` | 복잡한 비교/추론 | ~$0.01 |

## 사용법

### 사전 준비

1. [Claude Code CLI](https://docs.anthropic.com/en/docs/claude-code) 설치
2. 프로젝트 클론 후 의존성 설치:
   ```bash
   npm install
   ```
3. `.env.example`을 복사하여 API 키 설정:
   ```bash
   cp .env.example .env
   ```
   `.env` 파일을 열고 실제 API 키를 입력한다:
   ```
   PERPLEXITY_API_KEY="pplx-your-api-key-here"
   TAVILY_API_KEY="tvly-your-api-key-here"
   ```
   - Perplexity API 키: https://docs.perplexity.ai 에서 발급
   - Tavily API 키: https://tavily.com 에서 발급 (월 1,000크레딧 무료)

### 리서치 실행

Claude Code를 실행하고 질문을 던지면 된다:

```bash
claude
```

```
# 예시 질문
> 초고속 레이저의 산업 응용 현황과 전망을 조사해줘

> 양극산화 피막의 내구성 평가 방법론을 정리해줘

> Palantir Ontology의 아키텍처와 실제 적용 사례를 분석해줘
```

메인 에이전트가 자동으로:
1. Question Expansion으로 질문을 확장
2. Researcher 다수 + Journal을 병렬 발사
3. Critic으로 검증
4. 최종 Synthesis 작성
5. 프레젠테이션 생성 제안

### 산출물 확인

리서치 결과는 `docs/research/{날짜}-{주제}/` 폴더에 저장된다. `00-synthesis.md`부터 읽으면 된다.

### 프레젠테이션 생성

리서치 완료 후 프레젠테이션 생성을 요청하면 PptxGenJS 기반으로 .pptx 파일을 자동 생성한다:

```
> 이 리서치 결과로 발표자료 만들어줘
```

## 핵심 철학

### 속도보다 품질

멀티 에이전트를 쓰는 이유는 빠르게 하기 위해서가 아니다. **다른 관점에서 보고, 검증하고, 통합하기 위해서**다. 느려도 된다. 근거 없는 강한 결론은 안 된다.

### 축적과 재활용

모든 리서치는 구조화된 파일로 저장된다. 한 번의 리서치가 끝이 아니라, 다음 리서치의 기반이 된다. 지식이 휘발되지 않는다.

### 검증 내장

Researcher 2명 이상이면 Critic은 필수다. 도메인 적용성, 수치 근거, 상충점, 누락 관점, 확신도를 체계적으로 검증한다.

### 설계 우선

새로운 기능이나 방법론 변경은 바로 실행하지 않는다. 먼저 문제 정의가 맞는지, 더 나은 접근이 있는지, 트레이드오프는 무엇인지 논의한다.

### 비용 인식

검색 API에는 비용이 따른다. 낭비를 피하되, 필요한 도구 사용을 억제하지는 않는다. 얕은 곳에서 깊은 곳으로 진행하며, 충분하면 멈춘다.

## 다른 프로젝트에 적용하기

이 프레임워크의 에이전트 오케스트레이션 정책은 리서치 외 용도(설계, 분석, 구현)에도 변환 적용할 수 있다. `docs/claude-md-adaptation-guide.md`에 변환 가이드가 있다.

핵심은 `CLAUDE.md`다. 이 파일이 에이전트의 행동 정책, 역할 분리, 품질 기준, 검색 전략을 모두 정의한다.
