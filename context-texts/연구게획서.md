# 공정 전문가 시스템 기반 무개입 물리 가공 기술 — 리서치 과제 컨텍스트

> **과제 성격**: 본격 개발 이전의 선행 리서치 과제. 기술 가능성 검증 및 MVP 수준 프로토타입 구현이 목표. **소속 목표**: 목표1 AX성과창출 (가중치 35%) **기간**: 2026.01 \~ 2026.12

---

## 1. 해결하려는 문제 (As-Is)

현재 제조 공정에서 설비 이슈가 발생하면 다음 흐름으로 대응한다:

1. **설비 이슈 발생** — 이상 신호, 불량 발생, 공정 파라미터 이탈
2. **엔지니어 호출** — 담당자가 대기 중이거나 출근 후 조치 가능
3. **수작업 분석·판단** — 경험 기반 원인 추정, 과거 사례 수동 검색
4. **수동 레시피 조정** — 시행착오 반복, 설비사 협조 필요

**핵심 문제:**

- 설비 이슈가 생기면 반드시 사람이 있어야만 대응 가능 → 24시간 가동 공정에서 치명적 취약점
- 베테랑 엔지니어의 경험은 개인 머릿속에만 존재 → 퇴직·이직 시 노하우 손실
- 설비 제조사에 의존해야 하는 레시피 변경 → 긴 리드타임, 높은 비용

## 2. 목표 시스템 (To-Be)

**핵심 한 줄**: "설비가 스스로 판단하고, 전문가처럼 처방을 내리는 시스템"

```
As-Is: 설비 이슈 → 엔지니어 직접 대응(수시간~수일) → 수작업 원인 분석 → 레시피 수동 조정
To-Be: 설비 이슈 → AI 자동 감지 및 분석(수초~수분) → 과거 사례+지식 기반 원인 추론 → 최적 레시피 자동 생성 및 적용
```

---

## 3. 핵심 개념 (AI 비전문가 대상 설명)

### LLM (거대 언어 모델)

ChatGPT 같은 AI 기술. 방대한 기술 문서와 공정 데이터를 학습하여 엔지니어처럼 상황을 이해하고 판단한다.

### 전문가 시스템 (Expert System)

수천 건의 과거 공정 사례와 규칙을 DB에 저장해, AI가 새로운 문제 발생 시 유사 사례를 빠르게 찾아 해결책을 제안한다.

### 멀티 에이전트 (Multi-Agent)

여러 AI가 팀을 이뤄 협업. 한 AI는 원인 분석, 다른 AI는 레시피 계산, 또 다른 AI는 품질 검증을 담당.

## 4. 시스템 작동 흐름 (End-to-End)

```
[설비 이슈 발생]
      ↓  (자동 요청 전송)
[AI 자동 분석]
  - 과거 사례 검색 (벡터 DB)
  - 공정 지식 그래프 조회 (Neo4j 온톨로지)
  - 멀티 에이전트 복합 분석 (LangGraph)
      ↓
[최적 레시피 생성]
  - 분석 결과 종합
  - 최적화된 공정 조건 자동 생성
      ↓
[설비에 자동 전송]
  - API/MCP 통신으로 설비에 직접 전송
  - 즉시 적용
      ↓
[자동 리포트 생성 → 제조 네트워크 전송]
```

**목표 수치:**

- 레시피 추천 정확도: 70% 이상
- 자동 대응률: 50% 이상
- 엔지니어 대응 시간 단축: 30% 이상

## 5. 시스템 3대 핵심 구성 요소

### 두뇌: AI 분석 엔진 (멀티 에이전트)

- 기술: **LangGraph 기반 StateGraph**
- 구조: 17개 분석 노드로 구성된 워크플로우
- 동작: 설비 이슈 → 원인 분석 → 솔루션 도출 자동화
- 비유: 전문의 여러 명이 회의를 열어 진단을 내리는 것처럼, 각 에이전트가 역할 분담

### 기억: 공정 지식 데이터베이스

- 기술: **온톨로지 기반 지식 그래프 (Neo4j)** + 벡터 DB (과거 이력 검색)
- 구조: 공정 파라미터 ↔ 레시피 ↔ 품질 결과 간 관계망
- 동작: 새 문제 → 유사 과거 사례 즉시 검색 → 빠른 레시피 추천
- 비유: 수천 개 환자 차트를 즉시 검색하는 병원 전자의무기록 시스템

### 손발: 설비 통신 모듈

- 기술: **RESTful API + MCP(Model Context Protocol) 이중 인터페이스**
- 구조: FastAPI 기반 비동기 처리
- 동작: 설비 ↔ 서버 ↔ 제조망 3단계 통신
- 비유: 두뇌의 명령을 손발이 실행하듯 AI 판단을 설비에 전달

### 1단계 (3\~6월): 핵심 구성 요소 개발

#### 태스크 3: 온톨로지 기반 공정 지식 표현 체계 연구

- **기간**: 3월\~5월
- **목적**: 공정 파라미터-레시피-불량 유형 간 관계를 컴퓨터가 이해 가능한 구조로 정리
- **내용**:
  - OWL, RDF 국제 표준 분석
  - 공정 도메인 온톨로지 설계 방법론 수립
- **키워드**: OWL, RDF, 온톨로지(Ontology), 지식 표현(Knowledge Representation)
- **비유**: 백과사전의 목차와 색인처럼, AI가 지식을 체계적으로 찾을 수 있도록 지식 지도를 만드는 작업

#### 태스크 4: 사내 LLM LangChain/LangGraph Wrapper 개발

- **기간**: 3월\~4월
- **목적**: 사내 Gauss LLM을 범용 AI 프레임워크에서 사용 가능하도록 연결
- **내용**:
  - Gauss LLM → LangChain 커스텀 Wrapper 구현
  - LangGraph 멀티에이전트 시스템 구성으로 전문가 시스템의 추론 엔진 역할
- **키워드**: LangChain, LangGraph, Custom LLM Wrapper, Gauss LLM
- **비유**: 다른 브랜드 가전제품을 한 리모컨으로 조작하는 멀티 리모컨 제작

#### 태스크 5: LangGraph 기반 멀티에이전트 공정 분석 시스템

- **기간**: 4월\~5월
- **목적**: 핵심 AI 추론 엔진 구현
- **내용**:
  - 17노드 규모 StateGraph 설계 및 구현
  - 플로우: 공정 이슈 입력 → 자동 쿼리 생성 → 병렬 분석 → 결과 통합 → 품질 검증 → 솔루션 도출
  - 설비 요청에 대해 전문가처럼 다각도 분석 및 종합 판단하는 핵심 엔진
- **키워드**: LangGraph, StateGraph, Multi-Agent, 병렬 분석
- **비유**: 전문의·방사선과·약사가 팀을 이뤄 진단하듯, 각 AI가 역할 분담하여 종합 판단

#### 태스크 6: 공정 데이터 저장 및 검색 시스템

- **기간**: 5월\~6월
- **목적**: 과거 공정 데이터 저장 및 유사 사례 빠른 검색
- **내용**:
  - 과거 공정 파라미터, 레시피, 품질 결과 데이터 저장 DB 구축
  - 새로운 공정 조건 입력 시 유사한 과거 사례 빠르게 검색하는 시스템 개발
- **키워드**: 벡터 데이터베이스(Vector DB), 유사도 검색, RAG (Retrieval-Augmented Generation)
- **비유**: 수천 개 환자 차트를 즉시 검색하는 병원 전자의무기록 시스템

## 9. 기대 효과 (5가지)

1. **공정 기술 주도권 확보**: 설비 업체 협조 없이 공정팀이 독립적으로 레시피 제어·개선 가능. 설비 변경 최소화로 도입 비용 절감.

2. **공정 개발 속도 향상**: 레시피 최적화 사이클 50% 단축, 자동 대응률 50% 이상으로 엔지니어가 더 창의적인 업무에 집중 가능.

3. **범용 프레임워크로 수평 전개**: 파일럿 검증 후 레이저, CNC, 연삭, 폴리싱, 화학공정 등 다양한 공정에 동일 구조로 빠르게 확장.

4. **차세대 공정 기술 기반 마련**: 전문가 시스템으로 기존 불가능했던 신공정 구현 가능. 공정 노하우의 형식지화로 무인 운전 기반 구축.

5. **조직 AI 자립 개발 역량 강화**: 공정 전문가 직접 MVP 개발로 아이디어→구현 주기가 6\~12개월에서 1\~2개월로 단축.
