## 1. 먼저 알아야 할 개념

### 의사결정 트리 (Decision Tree)

스무고개처럼 질문을 던져서 답을 찾는 구조다.

```
                  나이 > 30?
                 /          \
              Yes            No
             /                \
        소득 > 5000?         학생인가?
        /       \            /      \
      Yes       No        Yes       No
    승인(0.9)  거절(0.2)  승인(0.7)  거절(0.3)
```

- 각 분기점(node)에서 하나의 질문을 던진다
- 질문의 답에 따라 왼쪽/오른쪽으로 내려간다
- 맨 아래 잎(leaf)에 도달하면 예측값을 출력한다

### 앙상블 (Ensemble)

하나의 트리는 단순하고 틀리기 쉽다. 여러 트리의 의견을 모으면 더 정확해진다.

| 방식 | 설명 | 대표 모델 |
| --- | --- | --- |
| **Bagging** | 여러 트리를 **독립적으로** 만들고 평균 | Random Forest |
| **Boosting** | 트리를 **순서대로** 만들며, 앞 트리의 실수를 다음 트리가 보완 | XGBoost, LightGBM |

XGBoost는 **Boosting** 방식이다.

### 손실 함수 (Loss Function)

"예측이 얼마나 틀렸는가"를 숫자로 표현한 것이다. 시험에서 틀린 문제 수를 세는 것과 같다. 틀린 문제가 적을수록 좋은 시험 결과이듯, 손실 함수 값이 작을수록 좋은 모델이다.

- **회귀**: MSE = (실제값 - 예측값)^2. 틀린 정도를 제곱해서 크게 벌점을 준다.
- **분류**: 비슷한 개념으로 "예측 확률이 정답에서 얼마나 벗어났는가"를 측정한다.

학습의 목표는 이 손실 함수를 최소화하는 것이다.

## 2. XGBoost의 핵심 아이디어

> 이 섹션에서 알게 되는 것: XGBoost가 무엇이고, 기존 방법과 뭐가 다른가

### 이름의 의미

**X**treme **G**radient **Boost**ing = 극한의 경사 부스팅

### 한 문장 요약

> 첫 번째 트리가 대략 예측하고, 두 번째 트리가 첫 번째의 실수를 고치고, 세 번째 트리가 아직 남은 실수를 고치고... 이걸 반복한다.

### Gradient Boosting과의 관계

XGBoost는 Gradient Boosting의 **개선된 구현**이다. 개념은 같고, 수학적 최적화와 엔지니어링이 추가됐다.

```
Gradient Boosting (개념/알고리즘)
    └── XGBoost (고성능 구현체)
            ├── 틀린 정도뿐 아니라 "확신도"까지 고려하여 더 정밀하게 보정
            ├── 너무 복잡한 모델에 자동으로 벌점 (정규화 내장)
            ├── 빈 칸(결측치)을 알아서 처리
            ├── 특성별 분할 탐색을 동시에 수행 (병렬 처리)
            └── 불필요한 가지를 자동으로 잘라냄 (가지치기 최적화)
```

> 각 항목의 상세 내용은 뒤 섹션에서 하나씩 설명한다.

## 3. 학습 과정 단계별 상세

> 이 섹션에서 알게 되는 것: XGBoost가 실제로 어떤 단계를 거쳐 학습하는가

### Step 0: 초기 예측값 설정

학습이 시작되기 전, 모든 샘플에 대해 동일한 초기 예측값을 부여한다.

- **회귀**: 타겟의 평균값 (예: 집값 평균 = 3억)
- **분류**: 양성 비율에서 계산한 기본값

```
데이터:  [집A, 집B, 집C, 집D, 집E]
실제값:  [2억, 5억, 3억, 4억, 1억]
초기예측: [3억, 3억, 3억, 3억, 3억]  ← 전부 평균값
```

이 시점에서 모델은 아무것도 학습하지 않은 상태다. 여기서부터 트리를 하나씩 추가한다.

### Step 1: 잔차(Residual) 계산

잔차 = 실제값 - 현재 예측값. \*\*"아직 못 맞춘 부분"\*\*이다.

```
실제값:   [2억, 5억, 3억, 4억, 1억]
현재예측: [3억, 3억, 3억, 3억, 3억]
잔차:     [-1억, +2억, 0, +1억, -2억]
           ↑ 너무 높게 예측    ↑ 너무 낮게 예측
```

> **용어 정리**: 정확히는 XGBoost는 단순 잔차가 아니라 **그래디언트**를 사용한다. 그래디언트란 "현재 예측을 어느 방향으로 얼마나 고쳐야 하는가"를 뜻한다. MSE(회귀)에서는 그래디언트 = 잔차이므로, 이 가이드에서는 잔차로 설명한다. 다른 손실 함수에서도 의미는 동일하다.

### Step 2: 잔차를 예측하는 새 트리 학습

새 트리는 **원래 타겟이 아니라 잔차를 예측**하도록 학습된다.

```
새 트리의 학습 데이터:
  입력: [집A의 특성, 집B의 특성, ...]
  타겟: [-1억, +2억, 0, +1억, -2억]  ← 잔차가 타겟!

새 트리가 학습한 결과:
  집A → -0.8억 (실제 잔차 -1억에 근접)
  집B → +1.5억 (실제 잔차 +2억에 근접)
  ...
```

왜 완벽하게 잔차를 맞추지 않을까? **일부러 조금만 맞춘다.** 이것이 과적합을 방지하는 핵심이다.

### Step 3: 학습률(learning rate)을 곱해서 예측 업데이트

새 트리의 예측을 그대로 더하지 않고, \*\*학습률(eta)\*\*을 곱해서 조금씩만 반영한다.

```
학습률 = 0.3 이라면:

새 예측 = 이전 예측 + 학습률 × 새 트리 예측

집A: 3억 + 0.3 × (-0.8억) = 2.76억  (실제 2억에 가까워짐)
집B: 3억 + 0.3 × (+1.5억) = 3.45억  (실제 5억에 가까워짐)
집C: 3억 + 0.3 × (0)      = 3.00억  (이미 맞았으니 그대로)
```

- 학습률이 작을수록 → 한 번에 조금씩 고침 → 더 많은 트리 필요 → 보통 더 정확
- 학습률이 클수록 → 한 번에 많이 고침 → 적은 트리로 끝남 → 과적합 위험

### Step 4: 반복 (Step 1\~3을 n_estimators번)

```
Round 1: 초기예측(3억) + 트리1 보정 → 예측1
Round 2: 예측1의 잔차 계산 → 트리2 학습 → 예측2
Round 3: 예측2의 잔차 계산 → 트리3 학습 → 예측3
...
Round 100: 예측99의 잔차 계산 → 트리100 학습 → 최종 예측
```

매 라운드마다 잔차는 줄어들고, 예측은 실제값에 가까워진다.

### Step 5: 최종 예측

최종 예측 = 초기값 + 모든 트리 예측의 합. 마치 여러 명의 전문가가 각자 조금씩 수정 의견을 내고, 그 의견을 모두 합산하여 최종 답을 내는 것과 같다.

```
최종 예측 = 초기값 + η·트리1(x) + η·트리2(x) + η·트리3(x) + ... + η·트리N(x)

(η = 학습률)
```

## 4. 숫자로 따라가는 미니 예제

> 이 섹션에서 알게 되는 것: 위 과정이 실제 숫자로 어떻게 작동하는가

### 문제 설정

공부 시간으로 시험 점수를 예측한다 (회귀).

| 학생 | 공부시간(x) | 실제점수(y) |
| --- | --- | --- |
| A | 1시간 | 30점 |
| B | 2시간 | 50점 |
| C | 3시간 | 60점 |
| D | 5시간 | 80점 |

설정: `learning_rate=0.3`, `max_depth=1` (stump: 분할 1번만), 손실함수=MSE

### Round 0: 초기 예측

```
초기 예측 = 평균 = (30+50+60+80)/4 = 55

현재 예측: [55, 55, 55, 55]
잔차:      [-25, -5, +5, +25]
MSE:       (625+25+25+625)/4 = 325
```

### Round 1: 첫 번째 트리

XGBoost는 **잔차를 가장 잘 나눌 수 있는 분할점**을 찾는다.

```
후보 1: 공부시간 ≤ 2?
  왼쪽(A,B): 잔차 [-25, -5] → 평균 -15
  오른쪽(C,D): 잔차 [+5, +25] → 평균 +15
  → 잘 나뉜다! (마이너스끼리, 플러스끼리)

후보 2: 공부시간 ≤ 3?
  왼쪽(A,B,C): 잔차 [-25, -5, +5] → 섞여 있다
  오른쪽(D): 잔차 [+25]
  → 왼쪽이 깔끔하지 않다

→ "공부시간 ≤ 2"가 가장 좋은 분할!
```

**트리 1**:

```
        공부시간 ≤ 2?
       /              \
    -15               +15
   (A,B)             (C,D)
  "점수 낮춰"       "점수 높여"
```

**예측 업데이트** (학습률 0.3 적용):

```
A: 55 + 0.3×(-15) = 55 - 4.5 = 50.5   (실제 30 → 아직 멀지만 방향은 맞다)
B: 55 + 0.3×(-15) = 55 - 4.5 = 50.5   (실제 50 → 거의 맞다!)
C: 55 + 0.3×(+15) = 55 + 4.5 = 59.5   (실제 60 → 거의 맞다!)
D: 55 + 0.3×(+15) = 55 + 4.5 = 59.5   (실제 80 → 아직 멀지만 방향은 맞다)

새 잔차: [-20.5, -0.5, +0.5, +20.5]
MSE: 210.25  (325에서 감소!)
```

### Round 2: 두 번째 트리

새 잔차 `[-20.5, -0.5, +0.5, +20.5]`를 보자. A와 D는 아직 많이 남았고, B와 C는 거의 맞았다.

```
같은 과정 반복 → 또 "공부시간 ≤ 2"가 최적 분할

잎 가중치: 왼쪽 -10.5, 오른쪽 +10.5

예측 업데이트:
  A: 50.5 + 0.3×(-10.5) = 47.35
  B: 50.5 + 0.3×(-10.5) = 47.35
  C: 59.5 + 0.3×(+10.5) = 62.65
  D: 59.5 + 0.3×(+10.5) = 62.65

MSE: 154.01  (또 감소!)
```

### 수렴 추이

```
MSE 변화:
Round 0:  ████████████████████████████████ 325.0
Round 1:  ████████████████████           210.3  (-35%)
Round 2:  ███████████████                154.0  (-27%)
Round 3:  ███████████                    113.1  (-27%)
...
Round 20: █                                3.2  (-99%)
```

라운드가 진행될수록 잔차가 줄고, 예측이 정확해진다.

### 셀프 체크

> **Q: Round 2 이후 학생 B의 잔차를 직접 계산해 보라.**
>
> &lt;details&gt;&lt;summary&gt;정답 보기&lt;/summary&gt; 실제 점수 50 - 현재 예측 47.35 = +2.65. Round 1에서 잔차가 -0.5였는데 보정이 과해져서 방향이 바뀌었다. 이래도 괜찮다 — 다음 트리가 이 오차를 다시 고칠 것이다. &lt;/details&gt;

## 5. 트리는 어떻게 만들어지는가

> 이 섹션에서 알게 되는 것: 각 라운드에서 트리의 분할점을 어떻게 결정하는가

### 핵심: Gain 점수

앞 예제에서 "공부시간 ≤ 2가 가장 좋은 분할"이라고 했다. XGBoost는 이것을 **Gain**이라는 점수로 판단한다.

**Gain = "쪼개고 나서 얼마나 나아졌는가"**

```
분할 전: 잔차가 뒤죽박죽 섞여 있다        → 예측이 어렵다
분할 후: 비슷한 잔차끼리 모였다           → 예측이 쉬워졌다
                                         → Gain이 높다!
```

XGBoost는 **모든 특성의 모든 분할점**에 대해 Gain을 계산하고, 가장 높은 것을 선택한다.

```
특성: 면적  [20평, 25평, 30평, 35평, 40평]
잔차:       [-1억, -0.5억, 0, +0.5억, +2억]

후보 분할점마다 Gain 계산:
  면적 ≤ 20평?  → Gain = 1.2
  면적 ≤ 25평?  → Gain = 2.8  ← 최고!
  면적 ≤ 30평?  → Gain = 2.1
  면적 ≤ 35평?  → Gain = 1.5

→ "면적 ≤ 25평"으로 분할 결정
```

이 과정을 **모든 특성**에 대해 수행하고, 전체에서 Gain이 가장 높은 (특성, 분할점) 조합을 선택한다. 그래서 특성이 많아도 XGBoost가 알아서 가장 유용한 특성을 찾아낸다.

### 잎 노드의 최적 가중치

트리의 잎(leaf)에 도달한 샘플들에 대해, XGBoost는 최적의 예측값을 **공식 하나로 바로 계산**한다. 경사하강법처럼 여러 번 반복할 필요가 없다. 이것이 XGBoost가 빠른 이유 중 하나다.

### 분할 중단 조건

Gain이 일정 기준(gamma) 미만이면 **분할하지 않는다**. "쪼개봤자 별로 나아지지 않으면 그냥 두는 것"이다. 이것이 자동 가지치기(pruning)다.

---

&lt;details&gt; &lt;summary&gt;&lt;b&gt;심화: Gain 공식의 수학적 구성&lt;/b&gt;&lt;/summary&gt;

XGBoost의 Gain 공식은 그래디언트(G)와 헤시안(H)을 사용한다:

- **그래디언트(G)**: 손실 함수의 1차 미분. "어느 방향으로, 얼마나 틀렸는가"
- **헤시안(H)**: 손실 함수의 2차 미분. "그 틀림에 대한 확신도"

```
            Gain = G_L²/(H_L+λ) + G_R²/(H_R+λ) - (G_L+G_R)²/(H_L+H_R+λ) - γ
```

각 잎의 최적 가중치:

```
  잎 가중치 = -G / (H + λ)
```

- λ (lambda): 가중치를 작게 유지하는 정규화 항
- γ (gamma): 분할의 최소 비용. Gain이 γ보다 작으면 분할 안 함

이 공식은 손실 함수를 2차 테일러 전개하여 유도한 것이다. 상세 유도는 \[Chen & Guestrin, 2016\] 논문 참고.

&lt;/details&gt;

## 6. 과적합 방지 장치들

> 이 섹션에서 알게 되는 것: XGBoost가 훈련 데이터에 과하게 맞춰지는 것을 어떻게 막는가

### 핵심 아이디어

XGBoost의 목적 함수는 두 가지를 동시에 추구한다:

1. **예측을 정확하게** (손실 최소화)
2. **모델을 단순하게** (복잡도에 벌점)

마치 시험에서 "정답 수 - 오답 감점"으로 점수를 매기는 것과 같다. 정답만 많이 맞추는 게 아니라, 찍어서 틀리는 것에도 벌점이 있다. 이 벌점 시스템이 \*\*정규화(Regularization)\*\*이며, 일반 Gradient Boosting에는 없는 XGBoost의 핵심 차별점이다.

### 필수 장치 3가지 (먼저 이것만 알면 된다)

| 장치 | 파라미터 | 직관적 의미 |
| --- | --- | --- |
| **학습률** | `learning_rate` (0\~1) | 각 트리의 의견을 얼마나 반영할지. 0.1이면 10%만 반영 |
| **트리 깊이 제한** | `max_depth` | 트리가 몇 단계까지 질문할 수 있는지. 깊을수록 복잡 |
| **조기 종료** | `early_stopping_rounds` | 검증 성능이 N 라운드 연속 개선 안 되면 학습 중단 |

이 3가지만 잘 조절해도 대부분의 과적합을 막을 수 있다.

### 추가 장치들 (성능을 더 끌어올리고 싶을 때)

| 장치 | 파라미터 | 효과 |
| --- | --- | --- |
| 최소 분할 이득 | `gamma` | Gain이 기준 미만이면 분할 중단 |
| L2 정규화 | `lambda` | 잎 가중치를 작게 유지 |
| L1 정규화 | `alpha` | 잎 가중치를 0으로 만들어 희소성 유도 |
| 최소 자식 가중치 | `min_child_weight` | 잎에 최소한의 샘플 필요 |
| 서브샘플링 | `subsample` | 각 트리에 데이터 일부만 사용 (랜덤성 추가) |
| 특성 샘플링 | `colsample_bytree` | 각 트리에 특성 일부만 사용 (랜덤성 추가) |

### 가지치기 (Pruning)

XGBoost는 **Gain이 음수면 분할하지 않는** 방식으로 성장 중에 불필요한 가지를 잘라낸다 (pre-pruning). `max_depth`까지 키운 후 Gain이 음수인 분할을 제거하는 후처리도 병행한다.

## 7. 핵심 하이퍼파라미터 정리

> 이 섹션에서 알게 되는 것: 어떤 파라미터를 어떤 순서로 조정해야 하는가

### 처음 쓸 때 이것만 건드려라

```python
model = XGBRegressor(
    n_estimators=1000,          # 트리 개수 (충분히 크게)
    learning_rate=0.1,          # 학습률 (보수적으로)
    max_depth=6,                # 트리 깊이 (기본값으로 시작)
    early_stopping_rounds=50,   # 50 라운드 개선 없으면 중단
)
```

이 4개 설정으로 시작하면 대부분의 문제에서 합리적인 성능을 얻을 수 있다.

### 전체 파라미터 레퍼런스

| 구분 | 파라미터 | 기본값 | 역할 | 튜닝 범위 |
| --- | --- | --- | --- | --- |
| 부스팅 | `n_estimators` | 100 | 트리 개수 | early_stopping과 함께 1000+ |
| 부스팅 | `learning_rate` | 0.3 | 각 트리의 기여 비율 | 0.01\~0.1 |
| 트리 | `max_depth` | 6 | 트리 최대 깊이 | 3\~10 |
| 트리 | `min_child_weight` | 1 | 잎 노드 최소 샘플 가중치 | 1\~10 |
| 트리 | `gamma` | 0 | 분할 최소 이득 | 0\~5 |
| 정규화 | `lambda` | 1 | L2 정규화 | 0\~10 |
| 정규화 | `alpha` | 0 | L1 정규화 | 0\~1 |
| 랜덤성 | `subsample` | 1 | 데이터 샘플링 비율 | 0.6\~0.9 |
| 랜덤성 | `colsample_bytree` | 1 | 특성 샘플링 비율 | 0.6\~0.9 |

### 실전 튜닝 순서 (권장)

```
1단계: n_estimators=1000 + early_stopping_rounds=50 + learning_rate=0.1
       → 적절한 트리 수를 자동 결정

2단계: max_depth (3~10)와 min_child_weight (1~10) 그리드 탐색
       → 트리 복잡도 결정

3단계: gamma (0~5) 조정
       → 불필요한 분할 제거

4단계: subsample (0.6~0.9)와 colsample_bytree (0.6~0.9) 조정
       → 랜덤성으로 과적합 방지

5단계: lambda (0~10)와 alpha (0~1) 미세 조정
       → 정규화 강도 결정

6단계: learning_rate를 낮추고 (0.01~0.05) n_estimators 재조정
       → 최종 정밀도 향상
```

## 8. XGBoost가 빠른 이유

> 이 섹션에서 알게 되는 것: 같은 Boosting인데 XGBoost가 왜 빠른가

### 근사 분할 알고리즘

모든 값을 다 시도하지 않고, **대표값만 후보로 시도**한다.

```
원본 데이터: [1, 2, 3, 4, 5, ..., 1000000]

전수 탐색: 999,999개 분할점 모두 시도  ← 느림

근사 탐색: [250000, 500000, 750000] 대표 3개만 시도  ← 빠름
```

### 병렬 처리

트리 자체는 순차적으로 만들어야 하지만, **하나의 트리 안에서 분할점 탐색**은 특성별로 동시에 수행한다.

```
트리 분할 탐색 (병렬):
  Thread 1: 특성1(면적)의 최적 분할점 탐색
  Thread 2: 특성2(층수)의 최적 분할점 탐색
  Thread 3: 특성3(역세권)의 최적 분할점 탐색
  ...
  → 결과 모아서 최고 Gain 선택
```

### 결측치 자동 처리

별도 전처리 없이 빈 칸을 알아서 처리한다.

```
면적: [20, ?, 30, ?, 40]  (? = 결측)

XGBoost의 처리:
  1. 결측 아닌 데이터로 최적 분할점 찾기
  2. 결측값을 왼쪽으로 보냈을 때 Gain 계산
  3. 결측값을 오른쪽으로 보냈을 때 Gain 계산
  4. 더 나은 방향을 기본 방향으로 설정

→ 결측값 전처리(fillna, imputation) 없이 바로 학습 가능
```

## 9. 자주 하는 실수 3가지

### 실수 1: early_stopping 없이 n_estimators만 키우기

```python
# 나쁜 예
model = XGBRegressor(n_estimators=10000)  # 언제 멈춰야 할지 모름 → 과적합

# 좋은 예
model = XGBRegressor(n_estimators=10000, early_stopping_rounds=50)
model.fit(X_train, y_train, eval_set=[(X_val, y_val)])  # eval_set 필수!
```

### 실수 2: 학습률을 너무 크게 잡기

```python
# 기본값 0.3은 사실 꽤 크다
model = XGBRegressor(learning_rate=0.3)  # 빠르지만 과적합 위험

# 0.05~0.1로 시작하고, 트리 수로 보상
model = XGBRegressor(learning_rate=0.1, n_estimators=1000)
```

### 실수 3: eval_set을 지정하지 않기

```python
# early_stopping을 쓰려면 eval_set이 반드시 필요하다
model.fit(X_train, y_train)  # eval_set 없음 → early_stopping 작동 안 함!

# 올바른 사용
model.fit(X_train, y_train, eval_set=[(X_val, y_val)])
```

## 부록: XGBoost vs 다른 부스팅 모델

| 항목 | XGBoost | LightGBM | CatBoost |
| --- | --- | --- | --- |
| 트리 성장 | Level-wise (기본) | Leaf-wise | Symmetric (균형) |
| 범주형 특성 | 수동 인코딩 필요 | 직접 지원 | 최적 지원 |
| 정규화 | L1 + L2 | L1 + L2 | L2 + 랜덤 순열 |
| 결측치 처리 | 내장 | 내장 | 내장 |
| 대규모 데이터 속도 | 보통 | 빠름 | 보통 |
| GPU 지원 | 지원 | 지원 | 지원 |

**실전 가이드**: 범주형 특성이 많으면 CatBoost를, 대규모 데이터에서 속도가 중요하면 LightGBM을, 범용적으로 안정적인 성능이 필요하면 XGBoost를 먼저 시도하라. 성장 방식 차이가 반드시 성능 차이를 의미하지는 않으며, 데이터 특성에 따라 달라진다.
