## 3. 핵심 수학 요약

### 3.1 인벌류트 곡선 (★★★)

**파라메트릭 방정식** — 기초원 반지름 $r_b$, 파라미터 $t \geq 0$:

$$x(t) = r_b(\cos t + t \sin t), \quad y(t) = r_b(\sin t - t \cos t)$$

**인벌류트 함수**: $\text{inv}(\alpha) = \tan(\alpha) - \alpha$ (라디안)

**역관계**: 반지름 $r$에서 $t = \sqrt{(r/r_b)^2 - 1}$

**inv() 역함수** — 닫힌 형태 없음, Newton-Raphson 필수:

```python
def inv_inv(target, tol=1e-10):
    alpha = target ** (1/3)  # 초기 추정
    for _ in range(50):
        f = math.tan(alpha) - alpha - target
        df = math.tan(alpha) ** 2
        alpha -= f / df
        if abs(f) < tol:
            break
    return alpha
```

### 3.2 기본 파라미터 관계 (★★★)

| 파라미터 | 수식 |
| --- | --- |
| 피치원 지름 | $d_0 = mz$ |
| 기초원 지름 | $d_b = d_0 \cos(\alpha_0)$ |
| 이끝원 지름 | $d_a = m(z + 2 + 2x)$ |
| 이뿌리원 지름 | $d_f = m(z - 2.5 + 2x)$ |
| 어덴덤 | $h_a = m(1 + x)$ |
| 디덴덤 | $h_f = m(1.25 - x)$ |
| 원호 이두께 (피치원) | $s_0 = m(\pi/2 + 2x\tan\alpha_0)$ |
| 기초 피치 | $p_b = \pi m \cos(\alpha_0)$ |

### 3.3 단일 치형 좌표 생성 알고리즘

> **상세**: R1 §3.4 참조. 아래는 요약.

```
입력: m, z, α₀, x
출력: 하나의 이(tooth) 폐합 윤곽 좌표 배열

1. 원 반지름 계산: r_b, r_a, r_f, r_0
2. 인벌류트 범위: t_start = max(0, √((max(r_f,r_b)/r_b)²-1)), t_tip = √((r_a/r_b)²-1)
3. 이두께 오프셋: θ_offset = π/(2z) + inv(α₀) + 2x·tan(α₀)/z
4. 오른쪽 플랭크: t를 t_start→t_tip 순회, 인벌류트 점 생성 후 θ_offset 회전
5. 왼쪽 플랭크: Y축 반전 + 역순
6. 이끝 원호: 두 플랭크 상단을 r_a 원호로 연결
7. 이뿌리 필릿: 트로코이드 또는 원호 근사
8. 기어 전체: 단일 이를 Δθ = 2π/z 간격으로 z번 원형 배열
```

### 3.4 이뿌리 필릿 — 기준 수식 결정

**Critic 판정**: 3개 보고서의 필릿 수식이 상충. **R3(Fetvaci 2010)의 에피트로코이드를 정밀 구현 기준으로 채택.**

| 구현 수준 | 방법 | 출처 |
| --- | --- | --- |
| **정밀** | Fetvaci 에피트로코이드 (1차 트로코이드 + 법선 오프셋) | R3 §2.5 |
| **간략** | 원호 근사 $\rho_f \approx 0.38m$ | R1 §3.3, R2 §2.3 |

정밀 필릿 (2차 트로코이드):

$$
\begin{pmatrix}
 x_F \\
y_F 
\end{pmatrix} = \begin{pmatrix}
 x_T \\
y_T 
\end{pmatrix} + r_f \cdot \frac{(\dot{y}_T, -\dot{x}_T)}{\sqrt{\dot{x}_T^2 + \dot{y}_T^2}}
$$

여기서 $(x_T, y_T)$는 1차 트로코이드(커터 이끝 원호 중심 궤적), $r_f$는 커터 팁 필릿 반경.

### 3.5 기어 강도 기초 (경량)

**Lewis 굽힘 강도**: $\sigma_b = W_t / (F \cdot m \cdot Y)$

**Hertz 접촉 응력**: $\sigma_H = Z_E \sqrt{W_t \cdot (i+1) / (F \cdot d_1 \cdot i \cdot \sin\alpha_0 \cos\alpha_0)}$

> 상세 계수(Lewis Y 표, AGMA 수정 계수 등)는 R1 §4 참조.

### 3.6 접촉비

$$\varepsilon = \frac{\sqrt{r_{a1}^2 - r_{b1}^2} + \sqrt{r_{a2}^2 - r_{b2}^2} - a\sin(\alpha_w)}{\pi m \cos(\alpha_0)}$$

권고: $\varepsilon > 1.2$ (소음/진동 최소화)

## 4. 기어 유형별 3D 확장

### 4.1 스퍼 기어 → 3D

단순 z축 압출: $P_{3D} = (x_{profile}, y_{profile}, z)$, $z \in [0, b]$

> R2 §2.5 코드 참조. 가장 단순하고 완성도 높음.

### 4.2 헬리컬 기어 → 3D

**핵심 변환**:

| 파라미터 | 수식 |
| --- | --- |
| 횡단면 모듈 | $m_t = m_n / \cos\beta$ |
| 횡단면 압력각 | $\tan\alpha_t = \tan\alpha_n / \cos\beta$ |
| 피치원 지름 | $d = z \cdot m_n / \cos\beta$ |
| 헬릭스 리드 | $L = \pi d \cot\beta$ |
| 등가 잇수 | $z_v = z / \cos^3\beta$ |

**3D 좌표 생성**: z위치에 따라 2D 횡단면 프로파일을 $\theta(z) = 2\pi z / L$만큼 회전.

```python
twist_angle = handedness * (2 * np.pi * z / lead)
rotated_2d = profile_2d @ rotation_matrix(twist_angle).T
```

> R2 §3.3 코드 참조. **주의**: 전위 헬리컬의 이두께는 법선면 수식을 횡단면으로 변환 후 적용해야 함 (이 절차가 R2에서 명시적이지 않으므로 구현 시 주의).

### 4.3 베벨 기어 → 3D

**Tredgold 근사**: 등가 잇수 $z_v = z / \cos\delta$로 가상 스퍼 기어 생성 후 원뿔면에 배치.

**피치 콘각** (직각 베벨): $\tan\delta_1 = z_1 / z_2$

**원뿔 거리**: $R = mz_1 / (2\sin\delta_1)$

> **경고 (Critic)**: R2 §4.4의 3D 코드에서 `scaled_profile[:, 1] + r_pos`는 원뿔면 투영이 아닌 원통면 오프셋이다. 정밀 구현 시 **구면 좌표 변환**이 필요하다. 올바른 원뿔면 투영:
>
> $$x_{cone} = r_{cone}\sin\delta \cdot \cos\theta_{profile}, \quad y_{cone} = r_{cone}\sin\delta \cdot \sin\theta_{profile}, \quad z_{cone} = r_{cone}\cos\delta$$
>
> 여기서 $\theta_{profile}$은 치형 좌표를 원뿔면 각도로 변환한 값.

**스파이럴 베벨**: Gleason 독점 파라미터(\~100개) 의존. 공개 수학으로는 근사만 가능. 시뮬레이션 앱에서는 "시각적 근사" 수준으로 구현하고, 정밀 치면은 범위 밖으로 명시할 것.

### 4.4 내접 기어 → 3D

외접 기어와 동일 인벌류트 수식, 단 이끝/이뿌리 방향 반전:

|  | 외접 | 내접 |
| --- | --- | --- |
| $r_a$ | $r_0 + m$ | $r_0 - m$ |
| $r_f$ | $r_0 - 1.25m$ | $r_0 + 1.25m$ |
| 인벌류트 방향 | 바깥 볼록 | 안쪽 오목 |

중심거리: $a = (d_{w2} - d_{w1}) / 2$ (외접은 합)

간섭 조건: $z_2 > 34$ (표준 20°), $z_2 - z_1 > 3\sim5$

3D는 링 형태 원통에서 CSG 차집합.

### 4.5 랙 & 피니언

랙 = 직선 프로파일. 우측 플랭크: $x(h) = p/4 + h\tan\alpha_0$, $y(h) = h$

피니언 회전 ↔ 랙 직선 이동: $\Delta x_{rack} = r_0 \cdot \theta$

## 5. 가공 공구 설계

### 5.1 포락선(Envelope) 이론 — 공통 기반 (★★★)

모든 창성 가공의 수학적 기반. 공구 운동 궤적의 포락선 = 기어 치형.

**맞물림 방정식**: $\mathbf{N}*2 \cdot \mathbf{v}*{rel}^{(12)} = 0$

**이산화 시뮬레이션**: 해석적 풀이가 어려울 때, 공구를 미소 각도씩 회전하며 부울 차집합으로 재료 제거 시뮬레이션. 최소 360°/1000 이상의 각도 스텝 권고.

> 해석적 방법(Fetvaci)과 이산화 방법의 차이: **필릿 영역에서 이산화는 스텝 의존적 오차 발생**. 응력 해석이 목적이면 해석적 방법 필수.

### 5.2 호브(Hob) 설계

**리드 각**: $\gamma_o = \arcsin(z_{thread} \cdot m_n / d_o)$

**인덱싱**: $\dot\phi_{gear} / \dot\phi_{hob} = z_{thread} / z_{gear}$

**스위블 각**: $\eta = \gamma_o \pm \beta$ (동일 방향: $-$, 반대: $+$; 스퍼는 $\eta = \gamma_o$)

**호브 3D 좌표**: R3 §1.4 참조. ★★☆ 단일 출처(KTH 석사논문) — 구현 전 원문 대조 권고.

### 5.3 피니언 커터(Shaper Cutter) 설계

**회전 비율**: $\phi_g / \phi_c = z_c / z_g$

**커터 좌표 → 기어 좌표 변환**: R3 §2.3의 **Fetvaci(2010) Eq.12 행렬만 사용**할 것.

> **경고**: R3에서 변환 행렬이 두 번 제시됨. 첫 번째 행렬(line 200-204)에 의미 불명 항 존재. **두 번째 "정확한 형태"(Fetvaci Eq.12)만 사용.**

$$\mathbf{R}*g^i(\delta, \phi_c) = \mathbf{M}*{gc}(\phi_c) \cdot \mathbf{R}_c^i(\delta)$$

맞물림 방정식으로 $(\delta, \phi_c)$ 쌍을 Newton-Raphson으로 풀어 기어 치형 좌표 획득.

### 5.4 랙 커터

가장 단순한 창성 공구. 직선 플랭크 + 원호 필릿.

**창성 좌표** (기어 프레임):

$$\begin{pmatrix}
 x_g \\
y_g 
\end{pmatrix} = \begin{pmatrix}
 \cos\phi & -\sin\phi \\
\sin\phi & \cos\phi 
\end{pmatrix} \begin{pmatrix}
 x_{rack}(u) - r_0\phi \\
y_{rack}(u) - r_0 
\end{pmatrix}
$$

**전위 가공**: 랙을 $xm$만큼 외측 이동 → 동일 수식에서 $y_{rack}(u) - r_0$ → $y_{rack}(u) - r_0 + xm$

### 5.5 이두께 측정 수학

**스팬 측정**: $W = m\cos\alpha_0[\pi(k - 0.5) + z \cdot \text{inv}(\alpha_0)] + 2xm\sin\alpha_0$

**오버핀 측정** (짝수 이수): $M = d_0\cos\alpha_0 / \cos\alpha_d + d_p$

여기서 $\alpha_d$는 핀 접촉점 압력각, inv 역함수로 산출.

> **상세**: R5 `05-worm-gear-mathematics.md` 참조.

### 6.1 웜 기본 파라미터

| 파라미터 | 수식 | 비고 |
| --- | --- | --- |
| 리드각 | $\tan\gamma = z_1 / q$ | $q$: 직경 계수 ($d_1 / m_a$) |
| 축방향 모듈 | $m_a = p_a / \pi$ |  |
| 법선 모듈 | $m_n = m_a \cos\gamma$ |  |
| 웜 피치원 지름 | $d_1 = q \cdot m_a$ |  |
| 웜 휠 피치원 지름 | $d_2 = z_2 \cdot m_a$ |  |
| 축간 거리 | $a = (d_1 + d_2) / 2$ |  |
| 속도비 | $i = z_2 / z_1$ | $z_1$: 스레드 수 (1\~4) |

### 6.2 웜 치형 유형

| 유형 | 축단면 모선 | 구현 상태 |
| --- | --- | --- |
| **ZA** (아르키메디안) | 직선 | 완전 (3D 코드 포함) |
| **ZI** (인벌류트) | 인벌류트 | **완전** (R6 §B에서 코드 보강) |
| **ZN** (법선 직선) | 법선면 직선 | 수식 있음 (★★☆, R6에서 상향) |
| **ZK** (원뿔 숫돌) | 공구 의존 | **이산화 시뮬레이션** (R6 §B에서 보강) |

**ZA 웜 3D 파라메트릭 방정식** (★★★):

$$\mathbf{r}(u, \theta) = \begin{pmatrix}
 (r_1 - u\cos\alpha_x)\cos\theta \\
(r_1 - u\cos\alpha_x)\sin\theta \\
u\sin\alpha_x + p_a\theta / (2\pi) 
\end{pmatrix}
$$

$u$: 프로파일 파라미터, $\theta$: 회전각, $\alpha_x$: 축방향 압력각

### 6.3 웜 휠 치면 생성

웜 휠 치면은 웜과의 **포락선**으로 결정된다. 해석적 닫힌 형태가 없으며 수치적 생성 필요.

**좌표 변환**: $\mathbf{r}*2 = \mathbf{M}*{21}(\phi_1) \cdot \mathbf{r}_1$

**포락 조건**: $\mathbf{n}_1 \cdot \mathbf{v}^{(12)} = 0$

> **R6 보강 완료**: R6 §A에서 포락 조건 방정식($\mathbf{n}*1^T \cdot \frac{d\mathbf{M}*{21}}{d\phi_1} \cdot \mathbf{r}_{1h} = 0$)을 Brent's method로 수치 풀이하는 `generate_worm_wheel_precise()` 구현. 잔류 오차 $9.4 \times 10^{-10}$ 수준. R5의 점 구름 방식은 시각화용 빠른 근사로 병행 사용 가능.

### 6.4 효율 및 자기잠금 (★★★)

**효율**: $\eta = \tan\gamma / \tan(\gamma + \varphi')$

여기서 $\varphi' = \arctan(\mu / \cos\alpha_n)$ (등가 마찰각)

**자기잠금 조건**: $\gamma < \varphi'$ — 리드각이 등가 마찰각보다 작으면 역구동 불가.

- 단일 스레드($z_1 = 1$): 대부분 자기잠금
- 4중 스레드($z_1 = 4$): 대부분 역구동 가능

> 주의: 자기잠금은 정적 마찰 기준. 동적 환경(진동, 충격)에서는 동적 마찰로 교체 필요.

### 6.5 웜 기어 전위 *(2차 보강)*

> **상세**: R6 §C 참조.

축간 거리 조정을 위해 웜 휠에 전위 계수 $x_{t2}$를 적용:

| 파라미터 | 전위 적용 수식 |
| --- | --- |
| 웜 휠 이끝 높이 | $h_{a2} = (1 + x_{t2}) m_x$ |
| 웜 휠 이뿌리 높이 | $h_{f2} = (1.2 - x_{t2}) m_x$ |
| 축간 거리 | $a = a_0 + x_{t2} \cdot m_x$ |
| 웜 휠 이두께 (축방향) | $s_{t2} = \pi m_x / 2 + 2 x_{t2} m_x \tan\alpha_x$ |

유효 범위: $-0.5 \leq x_{t2} \leq 0.7$ (권고). `WormGearPair`에 `xt2` 필드 추가 + 이끝 뾰족함/이뿌리 음수 자동 검증.

## 8. 파라미터 유효성 검증

시뮬레이션 앱에서 입력 파라미터의 유효 범위를 검증하는 체크리스트. R4 §E `full_gear_parameter_check()`에서 아래 모든 조건을 Python 코드로 구현.

### 8.1 기본 범위

| 파라미터 | 유효 범위 | 위반 시 |
| --- | --- | --- |
| 이수 $z$ | $z \geq 5$ (실용), $z \geq 1$ (이론) | $z < 5$: 경고, $z < 3$: 거부 |
| 압력각 $\alpha_0$ | $14.5° \leq \alpha_0 \leq 30°$ | 범위 외: 경고 |
| 전위 계수 $x$ | $-0.5 \leq x \leq 1.5$ | 범위 외: 이끝 뾰족/이뿌리 과대 경고 |
| 헬릭스각 $\beta$ | $0° < \beta < 45°$ | $\beta > 45°$: 실용 한계 경고 |
| 모듈 $m$ | $0.1 \leq m \leq 100$ mm | 비표준 경고 |

### 8.2 언더컷 조건

$$z_{min} = \frac{2}{\sin^2(\alpha_0)} \quad (\alpha_0 = 20° \Rightarrow z_{min} \approx 17)$$

전위로 방지: $x_{min} = (z_{min} - z) / z_{min}$

### 8.3 이끝 뾰족(Pointed Tooth) 조건

전위 계수가 너무 크면 이끝 이두께가 0이 됨. 이끝원에서 이두께 $s_a > 0$ 검증 필수:

$$s_a = d_a \left(\frac{s_0}{d_0} + \text{inv}(\alpha_0) - \text{inv}(\alpha_a)\right) > 0$$

### 8.4 내접 기어 특수 조건

- $z_2 > 34$ (표준 20°, 전위 없음)
- $z_2 - z_1 > 3 \sim 5$
- 중심거리 양수 확인

### 8.5 기어쌍 간섭 조건

$$r_{a2} < a\sin(\alpha_w) + r_{b1}$$

접촉비 $\varepsilon > 1.0$ (필수), $> 1.2$ (권고)

### 8.6 소프트웨어 검증 방법

> §7.4에서 모든 검증 함수의 Python 구현을 제공. `gear_pair_validation_suite()`로 일괄 실행.

## 11. 예상 밖 핵심 발견

### 11.1 SDF/GPU 병렬 포락선 시뮬레이션

R3이 발견한 이질 도메인 유추: 포락선 이산화의 부울 차집합은 컴퓨터 그래픽스의 **SDF(Signed Distance Field)** 기반 부울 연산과 구조적으로 동일하다. Ray marching 기법을 차용하면 GPU 병렬화로 실시간 창성 애니메이션 구현이 가능하다. 이는 시뮬레이션 앱의 UX를 근본적으로 바꿀 수 있는 발견이다.

### 11.2 스파이럴 베벨의 실질적 한계

스파이럴 베벨 기어의 정확한 치면은 약 100개의 기계 파라미터(Gleason 독점)가 필요하다. 공개 수학으로는 근사에 그치며, 이는 구조적 한계다. 시뮬레이션 앱에서는 직선 베벨까지만 정밀 구현하고, 스파이럴 베벨은 시각적 근사로 한정하는 것이 현실적이다.

### 11.3 웜 휠 포락의 연산 비용 트레이드오프 *(보강)*

웜 휠 치면의 정확한 포락 생성은 해석적 닫힌 형태가 없다. 단순 스위프 근사는 FEA 수준에서 간섭 오류를 유발한다는 점이 다수 출처에서 확인되었다. 시뮬레이션 앱에서 **연산 비용 대비 정밀도 트레이드오프**가 핵심 설계 결정이 된다.

### 11.4 재연삭 효과 모델링

호브/피니언 커터는 사용에 따라 이끝원이 축소되어 전위 계수 보정이 필요하다. 이를 시뮬레이션에 모델링하면 가공 품질 예측 정확도가 크게 향상된다.
