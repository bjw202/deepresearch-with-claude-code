## 1. LLM의 공정 제어 활용 연구

### 1.1 공정 파라미터 분석/조정

LLM은 제조 공정에서 "인지적 오케스트레이션 계층"으로 기능하기 시작했다. 핵심은 데이터 해석, 계획 생성, 파라미터 최적화를 자연어 기반으로 수행하는 것이다.

**학술 연구:**

- **Penn State + TU Munich (2024)**: LLM 기반 멀티에이전트가 제품 사양, CAD 데이터, 주문 정보를 자연어로 해석하여 공정 계획과 자원 배분을 수행. GPT-4 기준 2단계 공정에서 100% 성공률, 4단계 공정에서 86% 성공률 달성 (각 50회 시행). 단, 복잡도가 높아지면 성능이 저하됨 [출처: eenewseurope.com, arXiv:2409.18009]
- **LLM-assisted APS Framework (2025)**: 프롬프트 엔지니어링과 RAG를 활용한 고급 생산 계획 및 스케줄링. 부서 간 조율에 LLM을 활용 [출처: ACM, dl.acm.org/doi/10.1145/3761668.3761693]
- **FILLIS 프레임워크**: Factory Integrated Logic and Language Interface System. LLM이 기계 작동 설명, 문서 기반 Q&A에서 높은 정확도를 보였으나, 수학적 연산에서는 외부 에이전트가 필요함을 확인 [출처: ScienceDirect]

**의사결정 연결:** LLM은 공정 파라미터의 "해석과 추천"에는 강하나, 정밀한 수치 계산은 외부 도구와 결합해야 한다. 레시피 조정 시스템에서 LLM은 "왜 이 파라미터를 바꿔야 하는지" 설명하는 역할에 적합하고, 구체적인 수치는 전통적 ML/최적화 엔진이 담당하는 하이브리드 구조가 현실적이다.

### 1.2 LLM 기반 불량 원인 분석 (Root Cause Analysis)

**학술 연구:**

- **ARKNESS (2025, arXiv:2506.13026)**: KG + RAG 결합으로 CNC 가공 공정 계획에서 수치적으로 정확하고 추적 가능한 답변 제공. 3B 파라미터 Llama-3가 ARKNESS와 결합 시 GPT-4o 수준의 정확도 달성. 핵심 성과:
  - 다지선다 정확도 +25pp 향상
  - F1 +22.4pp 향상
  - 수치 환각(hallucination) 22pp 감소
  - 완전 온프레미스 운영 가능 (프라이버시 보호)

  > **수치 투명성:** 이 성과는 155개의 산업 전문가 검증 질문 기준이며, 실제 생산 현장의 다양한 변수(소재 편차, 장비 노후화 등)에서는 성능이 달라질 수 있다.

- **온톨로지-LLM 기반 FMEA 결함 원인 식별 (2025, arXiv:2510.15428)**: 자동차 압력 센서 조립 라인 사례 연구. 온톨로지 기반 LLM 추출로 FMEA 워크시트를 통합 KG로 변환하고, RGCN으로 결함 원인 추론.
  - RAG 기반 대비 F1@20: 0.267 → 0.523 (약 2배 향상)
  - 핵심 혁신: 제조 도메인 개념화(actions, states, components, parameters)와 공정 흐름 인식 학습의 결합
  - 이종 라인 간 FMEA 지식 이전 가능성 입증

### 1.3 LLM + 시계열 데이터 결합

제조 센서의 시계열 데이터를 LLM과 결합하는 연구가 2024-2025년에 급격히 증가했다.

| 방법 | 핵심 기법 | 제조 적용 강점 | 출처 |
|------|----------|---------------|------|
| **LLMAD** | In-context learning + AnoCoT | 다관점 이상 설명 제공 | arXiv:2405.15370 (2024) |
| **SIGLLM** | 프롬프팅 + 예측 잔차 | Zero-shot, 11개 데이터셋 검증 | DSAA 2024 |
| **MIT Prompter/Detector** | 센서 데이터→텍스트 변환 | 풍력터빈 등 센서 밀집 환경 | MIT News (2024) |
| **AnomalyLLM** | LLM 기반 지식 증류 | 경량 추론, 15개 UCR 데이터셋에서 14.5% 정확도 향상 | IJCAI 2024 |

**핵심 방식:** 센서 데이터(진동, 온도, 압력 등)를 "462, 728, 627, 390" 같은 텍스트 시퀀스로 변환하여 LLM에 입력. LLM이 이상 패턴을 자연어로 설명하여 근본 원인 분석을 지원한다.

**한계:** NeurIPS 2025에서 발표된 연구에 따르면, LLM 기반 시계열 이상 탐지는 정밀도 면에서 전통적 딥러닝에 미치지 못하는 경우가 있다. 해석 가능성은 뛰어나지만 정확도는 하이브리드 접근이 필요하다.

**반증 탐색:** LLM 기반 시계열 분석이 전통적 ML보다 우월하다는 주장에 대해 NeurIPS 2025 논문이 반증을 제시. "최근 LLM을 TSAD에 적용하려는 노력은 제한적 성공을 거두었으며, 종종 전통적 방법보다 낮은 성능을 보인다." LLM의 강점은 해석 가능성이지 정밀도가 아니다.

## 2. 지식그래프/온톨로지의 공정 제어 활용

### 2.1 불량-원인-파라미터 관계의 온톨로지 모델링

**학술 연구:**

- **HCP 지식그래프 (2024, Engineering, ScienceDirect)**: Human-Cyber-Physical 삼원 데이터 기반 KG. 자동차 생산 라인과 기어 제조 공정에서 실증.
  - 3계층 구조: HCP 데이터 분류 → HCP 온톨로지 구축 → KG 기반 품질 관리
  - 사례 검색, 자동 분석, 보조 의사결정 기능 구현
  - 공유성과 재사용성에서 기존 방법 대비 우수성 입증

- **디지털 트윈 기반 다층 KG (2025, PMC)**: 제조 공정 디지털 트윈을 위한 3계층 KG 아키텍처:
  - **개념 계층**: OWL2RL 기반 범용 온톨로지 라이브러리 (결함, 원인, 파라미터)
  - **모델 계층**: 실시간 데이터(가공 파라미터, 소재 사양)를 RDF 그래프로 인스턴스화
  - **인스턴스 계층**: 구체적 결함 사례와 해결 이력 저장

**온톨로지 핵심 관계 패턴:**

| OWL 속성 | 적용 예시 |
|----------|----------|
| `owl:inverseOf` | "원인 변화" ↔ "결과 발생" 양방향 추적 |
| `owl:symmetricProperty` | 레시피 변형 간 유사성 |
| `owl:hasValue` | 공구 모델 '001' → 특정 결함 리스크 연결 |

### 2.2 제조 KG 기반 불량 진단 시스템

- **철강 모터 결함 진단 KG (2024, Springer)**: 장비 구조 클래스(V1)로 모터 하위 시스템/부품/파트의 계층 구조를 정의. 베어링 결함, 기어 결함, 모터 결함 등을 센서 데이터(진동, 온도, 압력)와 연결. 강화학습 기반 KG 추론(DeepPath) 적용.

- **지능형 생산 라인 결함 KG (2025, PMC)**: Protege 도구로 온톨로지 개발. 엔티티 유형: 결함 주체, 내용, 원인, 탐지 방법, 유지보수 솔루션. 비정형 전문가 텍스트에서 관계 추출하여 트리플 형성 후 그래프 DB로 구조화.

- **적층제조 결함 진단 KG (2023, ScienceDirect)**: GCN 기반 Re-GCN 모델로 새로운 관계 추론. 적층제조 특유의 결함-공정 파라미터 관계를 KG로 모델링.

### 2.3 공정 레시피의 온톨로지 표현

- **ISA-95 기반 제조 온톨로지**: IEC 62264 표준에 따라 제품/공정/모델 계층, 제품 역량 모델, 역할 기반 장비 계층을 정의. 생산 정보(실제 생산 내역)와 생산 스케줄링(계획)을 논리적으로 연결.

- **IDMPO (Industrial Digital Model for Pharma Manufacturing, 2025 Ontology Summit)**: 제약 제조를 위한 공급자 독립적 온톨로지. NIST/OAGI 기반으로 결함 및 레시피 모델링의 표준화된 인프라 제공.

- **다층 네트워크 기반 공정 분석 (2023, 파노니아 대학 박사논문)**: 온톨로지 모델에서 레이블링된 다층 네트워크를 생성하여 클러스터링, 커뮤니티 탐지, 조립 라인 밸런싱, 셀 형성 최적화에 활용.

**의사결정 연결:** 온톨로지 기반 레시피 표현은 "왜 이 파라미터가 이 값이어야 하는지"에 대한 추적 가능한 근거를 제공한다. 이는 LLM의 환각 문제를 보완하는 핵심 인프라가 된다.

## 3. 멀티에이전트 시스템의 제조 적용

### 3.1 LLM 기반 제조 멀티에이전트 아키텍처

**학술 연구:**

- **자율 유지보수 멀티에이전트 (2025, PHM Society)**: CNC 공작기계 HMI 시스템에 구현. 4개 에이전트 구성:
  - **Chatbot**: 사용자 입력(에러 코드 등) 처리
  - **Solution Finder**: 매뉴얼에서 대응 절차 식별
  - **Actor**: HMI와 직접 상호작용하여 조치 실행
  - **Supervisor**: 에이전트 감독 및 의사결정 규칙 적용
  - 각 에이전트는 역할별 프롬프트로 제어되어 기존에 보지 못한 알람에도 매뉴얼 기반 추론으로 대응. 로컬 LLM 사용으로 데이터 보안 확보.

- **LLM 기반 다중에이전트 제조 시스템 (2025, ScienceDirect)**: 지능형 쇼플로어 관리를 위한 시스템:
  - **MSM** (Machine Server Module): 제조 자원 직접 제어
  - **BIM** (Bid Inviter Module): 작업 입찰 관리
  - **BM** (Bidder Module): 입찰 참여
  - **TM** (Thinking Module): LLM 기반 추론
  - **DM** (Decision Module): 최종 결정
  - 에이전트 간 협상으로 인간 개입 없이 제품의 전체 가공 사이클 자율 실행. 물리적 쇼플로어에서 실험 수행.
  - **핵심:** 기존 AI 방법의 데이터 수집/학습 과정을 건너뛰고, LLM의 추론 능력으로 즉시 적용 가능.

- **LLM 기반 산업 자동화 제어 (2024, arXiv:2409.18009, IEEE ETFA 2025 채택)**: 이벤트 기반 데이터 모델링으로 다양한 의미 수준의 실시간 이벤트를 해석하여 생산 계획 생성 및 실행.

### 3.2 에이전트 간 역할 분담과 오케스트레이션

주요 아키텍처 패턴:

| 패턴 | 특징 | 제조 적용 적합성 |
|------|------|----------------|
| **중앙 집중형** | 단일 오케스트레이터가 모든 에이전트 조율 | 안전성 높음, 단일 실패점 존재 |
| **분산형** | 에이전트 간 직접 협상 | 유연성 높음, 안전 보장 어려움 |
| **하이브리드** | 중앙 오케스트레이터 + 로컬 에이전트 그룹 | 제조 공정에 가장 적합 |
| **ReAct** | 즉각 행동, 단계별 LLM 추론 | 짧고 동적인 태스크에 적합 |
| **Plan-and-Execute** | 전체 계획 먼저 생성 후 실행 | 장기 공정 계획에 적합 |

**의사결정 연결:** 제조 공정 제어에서는 "하이브리드 + Plan-and-Execute"가 가장 적합하다. 전체 레시피 조정 계획을 먼저 세우고(Plan), 각 파라미터 조정을 전담 에이전트가 실행(Execute)하되, 안전 오케스트레이터가 감독하는 구조.

### 3.3 Human-in-the-Loop 에이전트 시스템

**산업 사례:**

- **Bosch (2025 Tech Day 발표)**: 제조 현장에서 Agentic AI를 이미 활용 중. 멀티에이전트 시스템이 장비 모니터링, 유지보수 예측, 인력 스케줄링을 수행. 인간 또는 조율 에이전트가 감독. 비계획 가동중단 감소 및 생산성 향상 보고 [출처: Bosch Press Release]

- **일반 프레임워크**: 역할 기반 검증(role-based validation) 원칙 - 엔지니어가 의사결정을 승인하는 구조. 구조화된 프롬프트, 규칙 기반 검증, 피드백 루프로 안전 관리.

## 4. KG + LLM 결합(GraphRAG 등)의 공정 제어 적용

### 4.1 GraphRAG 개요와 제조 적용 잠재력

GraphRAG는 지식그래프를 활용하여 LLM의 검색-증강 생성을 수행하는 기법으로, 2024년 Microsoft Research에서 본격 도입되었다.

**기존 RAG 대비 장점 (제조 관점):**

| 측면 | 기존 RAG | GraphRAG |
|------|---------|----------|
| 정확도 | 복잡한 쿼리에서 ~50% | 관계 추론으로 80%+ |
| 적용 분야 | 단편적 사실 검색 | 공급망, 결함 네트워크 등 관계형 데이터 |
| 토큰 효율 | 높은 사용량 | 포괄적 답변에 97% 적은 토큰 |
| 기업 적합성 | 환각 위험 | 출처 추적 가능, 규정 준수 용이 |

[출처: articsledge.com, HCLTech]

> **수치 투명성:** 위 수치는 GraphRAG 일반 벤치마크 기준이며, 제조 도메인 특화 데이터에서의 성능은 별도 검증 필요. 제조 데이터는 텍스트보다 수치/시계열 비중이 높아 일반 GraphRAG 성능과 다를 수 있다.

### 4.2 제조 KG에 LLM을 연결한 구체적 사례

**학술 연구 (핵심):**

- **ARKNESS (2025, arXiv:2506.13026)**: CNC 가공 분야에서 KG + LLM 결합의 가장 구체적인 사례.
  - **Zero-shot KG 구축**: 가공 문서, 핸드북, G-코드 주석, 벤더 데이터시트에서 수동 레이블링 없이 자동으로 멀티관계 그래프 생성
  - **최소 서브그래프 주입**: 쿼리 응답에 필요한 최소한의 증거 연결 서브그래프만 LLM에 제공
  - **성과**: 3B Llama-3 + ARKNESS가 GPT-4o 수준 정확도 달성, ROUGE-L 8.1배 향상
  - **현장 적합성**: 완전 온프레미스 운영, 실시간 추론 가능

- **온톨로지-LLM FMEA 그래프 학습 (2025, arXiv:2510.15428)**:
  - FMEA 워크시트의 자연어 가변성 문제를 해결
  - LLM이 온톨로지 가이드 하에 FMEA를 구조화된 KG로 변환
  - RGCN으로 의미 관계와 공정 흐름 의존성을 동시에 학습
  - RAG 대비 F1@20: 0.267 → 0.523 (96% 향상)

- **Making KG Work for Smart Manufacturing (2024, Journal of Manufacturing Systems)**: 포괄적 서베이. KG 기법의 제조 적용을 데이터 획득부터 의사결정까지 전 과정에서 조망. 불량 진단, 근본 원인 분석, 공급망 관리 등에서의 KG 활용 동향 정리.

## 5. 폐쇄 루프 AI 제어의 연구 동향

### 5.1 AI가 실제로 설비 파라미터를 조정한 사례

**산업 사례:**

- **Imubit - Closed Loop AI Optimization (AIO)**: 가장 구체적인 폐쇄 루프 AI 제어 사례.
  - AI가 실시간으로 DCS(분산제어시스템) 또는 PLC에 직접 명령을 작성하여 운전 파라미터 자동 조정
  - 정유, 화학, 시멘트, 범용 산업에 적용
  - **성과** (McKinsey Industry 4.0 보고서 인용):
    - 처리량(Throughput) 10-30% 증가
    - 노동 생산성 15-30% 향상
    - 비계획 가동중단 30-50% 감소
  - ROI 회수 기간: 대부분 6개월 이내
  - ARC Advisory Group 2024 시장 연구에서 인정

  > **수치 투명성:** 위 수치는 McKinsey의 Industry 4.0 전반 조사 기준이며, Imubit 단독 성과가 아닌 AI 적용 제조업 전반의 범위다. 개별 현장 성과는 공정 유형, 기존 자동화 수준에 따라 크게 다를 수 있다.

- **Wizata - 폐쇄 루프 AI 자동화**: 운영자의 직관과 경험을 확장 가능한 시스템으로 캡슐화. 시멘트, 야금 분야에서 파일럿 운영.

- **AI 기반 연삭 공정 폐쇄 루프 (2026, Metrology.news)**: AI + 센서 기술로 연삭 공정의 실시간 품질 제어. 고품질, 저비용, 경쟁력 있는 제조 달성 목표.

### 5.2 자율 제어 vs Human-in-the-Loop 비교

| 측면 | 완전 자율 제어 | Human-in-the-Loop |
|------|-------------|------------------|
| 반응 속도 | 밀리초 단위 | 분 단위 |
| 안전성 | 사전 정의된 경계 내에서만 | 전문가 판단 가능 |
| 적합한 상황 | 반복적, 잘 정의된 공정 | 새로운 제품, 비정상 상황 |
| 도입 단계 | 성숙 단계 | 초기 배포/신뢰 구축기 |
| 규제 | 인증 필요 (산업별 상이) | 규제 부담 낮음 |

**전형적 도입 경로:**
1. **Open-loop** (추천만 제공) →
2. **Closed-loop with oversight** (운영자 감독 하 자동 실행) →
3. **Full autonomous** (안전 한도 내 완전 자동)

이 3단계 전환은 Imubit, Wizata 등 주요 벤더가 공통적으로 추천하는 방식이다.

### 5.3 안전 보장 메커니즘

- **사전 정의 경계(Predefined Boundaries)**: 온도, 압력 등 핵심 파라미터의 안전 한도를 설정하여 AI가 이를 초과하는 조정을 차단
- **검증 루프(Validation Loops)**: 조정 후 결과를 모니터링하여 의사결정을 사후 검증
- **운영자 개입(Override)**: 언제든지 인간이 개입할 수 있는 권한 유지
- **모델 학습 승인**: 운영자가 모델 재학습을 승인하는 프로세스

**의사결정 연결:** 온톨로지 기반 시스템에서 안전 제약 조건을 온톨로지의 일부로 명시적으로 모델링하면, AI의 조정 범위를 구조적으로 제한할 수 있다. 이는 "블랙박스 제약"보다 설명 가능하고 감사 가능한 안전 메커니즘을 제공한다.

## 6. 반대 관점과 한계

### 6.1 LLM의 수치적 추론 한계

**학술적 근거:**

- **수치 추론 취약성 (arXiv:2509.06332)**: LLM의 기초 수학 연산 능력이 불안정하며, 이 불안정성이 복잡한 문제에서 증폭됨. 큰 수 집합 벤치마크에서 소수 판별, 24 게임 등 복합적 수치 추론에서 체계적 실패 관찰.

- **수치 변동 강건성 (ACL 2025)**: GSM-ALT 데이터셋 (GSM8K에서 수치만 변경) 실험 결과, 수치 변동에 따라 LLM의 정확도가 크게 떨어짐. Llama-3-8B의 경우 어려운 변형에서 69%가 계산 오류, 31%가 추론 오류.

- **FILLIS 사례**: 제조 맥락에서 LLM이 수학적 연산에서 명시적 한계를 보임. 외부 계산 에이전트 필요.

**의사결정 연결:** 공정 레시피의 수치 조정(예: 온도를 3도 올리면 불량률이 어떻게 변하는가)은 LLM에 직접 맡기면 안 된다. LLM은 "어떤 파라미터를 조정해야 하는지" 판단하고, 구체적인 수치는 물리 모델이나 전통적 ML이 계산하는 구조가 필수적이다.

### 6.2 온톨로지의 유지보수 비용

- **구축 비용**: 도메인 전문가의 장기적 참여 필요. Protege 등 도구가 있지만, 개념 정의와 관계 설정은 본질적으로 수동 작업.
- **진화 관리**: 새 제품, 새 공정, 새 결함 유형이 추가될 때마다 온톨로지를 업데이트해야 함.
- **일관성 유지**: 여러 전문가가 참여할 경우 용어와 분류 체계의 일관성 확보가 어려움.

**최근 해결 방향:**
- LLM을 활용한 온톨로지 반자동 구축 (ARKNESS의 zero-shot KG 구축이 대표 사례)
- LLM 기반 시맨틱 어노테이션 도구 [출처: sci-k.github.io, 2025]
- 단, arXiv:2512.05594 연구에 따르면 LLM의 온톨로지 학습은 기본 공리에서는 효과적이나, 복잡한 공리 구조와 논리 규칙에서는 아직 한계가 있다.

### 6.3 기존 ML/DL 대비 이점이 있는가?

| 측면 | 기존 ML/DL | KG + LLM |
|------|-----------|----------|
| **정밀도** | 높음 (구조화된 데이터에 최적화) | 중간 (수치 추론 취약) |
| **해석 가능성** | 낮음 (블랙박스) | 높음 (추론 과정 추적 가능) |
| **도메인 전이** | 재학습 필요 | 온톨로지 기반 지식 이전 가능 |
| **데이터 요구량** | 대량의 학습 데이터 필요 | 소량/zero-shot 가능 |
| **비정형 지식 통합** | 별도 NLP 파이프라인 필요 | LLM이 자연스럽게 처리 |
| **결정적 일관성** | 동일 입력 → 동일 출력 | 확률적 (비결정적) |
| **비용** | 학습 비용 높으나 추론 저렴 | 추론 비용 높음 (LLM) |
| **안전 인증** | 검증된 방법론 존재 | 아직 표준 부재 |

**핵심 판단:** KG + LLM은 기존 ML/DL을 "대체"하는 것이 아니라 "보완"하는 것이다.

- **KG + LLM이 우위인 영역**: 비정형 지식 통합, 근본 원인의 자연어 설명, 이종 공정 간 지식 이전, 새로운 결함 유형에 대한 빠른 대응
- **기존 ML/DL이 우위인 영역**: 정밀한 수치 예측, 실시간 센서 데이터 처리, 결정적 제어, 안전 인증

> **반증 탐색 결과:** Forrester 보고서에 따르면 "규칙 기반과 RPA가 미션 크리티컬 흐름의 주요 오케스트레이터로 남아 있다." 또한 Mario Thomas의 분석에서 "LLM은 맥락 이해, 모호성 처리, 창의적 콘텐츠 생성에 뛰어나지만, 정밀도 의존 워크플로우, 수치 추론, 일관된 실행이 필요한 다단계 프로세스에서는 어려움을 겪는다." 이는 순수 LLM 기반 공정 제어의 한계를 분명히 보여준다.

## 7. 종합 평가 및 의사결정 가이드

### 7.1 근거 신뢰도 매트릭스

| 핵심 주장 | 출처 유형 | 도메인 일치도 | 확신도 | 검증 필요 여부 |
|----------|----------|------------|--------|-------------|
| LLM이 공정 파라미터 추천 가능 | 학술(arXiv, IEEE ETFA) | 높음(제조 직접) | 중-상 | 현장 파일럿 필요 |
| KG가 불량-원인 추적에 효과적 | 학술(ScienceDirect, Springer) + 산업(Siemens) | 높음 | 상 | 도메인별 온톨로지 구축 필요 |
| GraphRAG가 제조에서 RAG 대비 우수 | 학술(arXiv:2506.13026) | 중(CNC 한정) | 중 | 타 공정 검증 필요 |
| 폐쇄 루프 AI가 실제 현장에서 작동 | 산업(Imubit, Wizata) | 높음(화학/정유 중심) | 상 | 이산 제조에서의 검증 필요 |
| 멀티에이전트가 쇼플로어 관리 가능 | 학술(ScienceDirect, PHM) | 중(실험실 수준) | 중 | 규모 확장 검증 필요 |
| LLM의 수치 추론이 공정 제어에 불충분 | 학술(arXiv, ACL) + 산업(다수) | 높음 | 상 | 반증 미발견 |

### 7.2 역방향 의사결정 가이드

| 만약... | 그러면... |
|---------|----------|
| 비정형 불량 보고서가 많고 구조화가 안 되어 있다면 | → KG + LLM으로 자동 구조화 우선 (ARKNESS/FMEA KG 접근) |
| 정밀한 수치 제어가 핵심이라면 | → 전통 ML/물리 모델을 주축으로, LLM은 해석/설명 보조로 |
| 이종 라인 간 결함 지식 공유가 필요하다면 | → 온톨로지 기반 표준화 + LLM 개념화 (arXiv:2510.15428 접근) |
| 운영자 신뢰가 아직 부족하다면 | → Open-loop(추천만) → Human-in-the-loop → 점진적 자율화 |
| 빠른 ROI가 필요하다면 | → 폐쇄 루프 AIO (Imubit류)로 시작, 이후 KG/LLM 계층 추가 |
| 데이터 프라이버시가 최우선이라면 | → 온프레미스 소형 LLM + KG (ARKNESS의 3B 모델 접근) |

### 7.3 예상 밖 핵심 발견

1. **소형 LLM + KG가 대형 LLM을 이길 수 있다**: ARKNESS에서 3B Llama-3 + KG가 GPT-4o 수준 성능을 달성한 것은, 제조 현장의 프라이버시/비용 제약 조건에서 실현 가능한 경로를 보여준다. 반드시 거대 모델이 필요하지 않다.

2. **온톨로지의 "자동 구축"이 현실화되고 있다**: ARKNESS의 zero-shot KG 구축, FMEA 워크시트의 LLM 기반 구조화 등, 기존에 가장 큰 장벽이었던 온톨로지 구축 비용이 LLM에 의해 크게 낮아지고 있다.

3. **폐쇄 루프 AI의 주류는 연속 공정(화학/정유)이며, 이산 제조(조립/가공)에서의 사례는 아직 초기 단계**: 이는 시스템 설계 시 타겟 공정 유형에 따라 전략을 다르게 가져가야 함을 의미한다.

### 7.4 후속 탐색 질문

1. **온톨로지 자동 구축의 품질 상한선은 어디인가?** LLM이 생성한 온톨로지와 전문가가 구축한 온톨로지의 품질 차이를 정량적으로 비교한 연구가 필요하다.

2. **멀티에이전트 제조 시스템의 실패 모드는 무엇인가?** 에이전트 간 충돌, 교착 상태, 연쇄 오류 전파 등 실패 시나리오에 대한 체계적 연구가 부족하다.

3. **규제 환경에서 LLM 기반 공정 제어의 인증 경로는?** 의약품, 항공, 자동차 등 규제 산업에서 LLM 기반 의사결정의 인증 가능성은 아직 미탐색 영역이다.

### 7.5 문제 재정의

원래 질문("온톨로지 + LLM 멀티에이전트로 폐쇄 루프 레시피 제어를 할 수 있는가?")보다 더 적절한 핵심 질문:

> **"LLM은 공정 제어에서 '판단과 설명' 계층으로, 온톨로지는 '지식 구조화와 안전 제약' 계층으로, 전통 ML/물리 모델은 '정밀 수치 계산' 계층으로 각각 역할을 분담할 때, 이 세 계층의 최적 결합 아키텍처는 무엇인가?"**

순수 LLM 기반 폐쇄 루프 제어보다, **3계층 하이브리드 아키텍처**가 현실적이고 안전하며 현재 기술 수준에서 실현 가능한 접근이다.
