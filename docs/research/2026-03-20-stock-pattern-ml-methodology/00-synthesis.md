# 종합 보고서: 스윙 트레이딩 패턴 인식 의사결정 보조 시스템

> 2026-03-21 (v2) | Researcher 7명 + Critic 2회 통합 v1(2026-03-20) 대비 변경: 자동매매 → 의사결정 보조로 리프레이밍, 패턴 정량화·차트북·유사도 매칭·전처리 보강

---

## Executive Summary

한국 증시 2,500종목 DB를 활용하여, 스윙 트레이딩의 고확률 피봇 포인트를 정량적으로 탐지하고, 유사 과거 사례와 성공 확률을 제시하는 **의사결정 보조 시스템**을 설계한다. 자동매매가 아닌, 트레이더의 판단을 보강하는 도구이다.

### 핵심 결론 5줄 요약

1. **규칙 기반 패턴 정량화가 최우선이다.** O'Neil/Minervini/쿨라메기의 패턴 정의를 코드로 번역하고, LightGBM으로 확률을 추정한다. 딥러닝/RL은 현 단계에서 불필요.
2. **차트북(성공/실패 사례집) 자동 생성이 핵심 산출물이다.** 패턴 유형별 사례를 축적하고, 유사 패턴 검색으로 "과거에 비슷한 차트가 어떻게 됐는가"를 제시한다.
3. **전처리가 시스템 품질을 좌우한다.** 가격 정규화(피봇 대비 비율 + 로그 수익률), 거래량 정규화(로그 RVOL + 백분위), 시계열 컨텍스트(사전 상승률, 베이스 카운트, regime)를 다채널로 결합한다.
4. **+30%/10일 성공 기준의 현실성을 Phase 0에서 반드시 검증해야 한다.** 양성 샘플이 500건 미만이면 기준 완화 필요. 이 검증이 전체 프로젝트의 성패를 결정한다.
5. **모든 패턴 파라미터가 미국 시장 기반이다.** 한국 증시(상한가 30%, 개인투자자 비중, 거래세)에 맞는 교정이 Phase 0에서 선행되어야 한다.

## 1. 시스템 목적과 출력

### 자동매매가 아닌 의사결정 보조

```
입력: 종목 코드 + 현재 날짜
출력:
  1. 패턴 분류 (VCP/CWH/HTF/W/Low Cheat/Episodic Pivot/해당없음) + 품질 점수
  2. 성공 확률 (피봇 돌파 후 10일 내 +30%) + 계층별 확률(+10%, +20%)
  3. 유사 과거 사례 Top-K (차트 이미지 + 후속 성과)
  4. 시장 컨텍스트 보정 (현재 regime에서의 조건부 확률)
```

## 2. 시스템 아키텍처 (4 Layer)

### Layer 1: 패턴 정량화 & 분류 엔진 (04 기반)

6대 패턴을 **Minervini Trend Template(Stage 2 필터)** 전제 하에 규칙 기반으로 탐지한다.

| 패턴 | 핵심 수치 기준 | 피봇 정의 | 출처 |
| --- | --- | --- | --- |
| **VCP** | 수축 2-6회, 각 깊이 20-30% 감소, 볼륨 드라이업(50MA의 40-60%) | 최종 수축의 고점 | Minervini |
| **CWH** | 컵 깊이 12-33%, 기간 7-65주, 핸들 깊이 &lt;컵의 50%, 기간 1-4주 | 핸들 고점 (좌측 lip 근처) | O'Neil |
| **HTF** | 선행 상승 30%+/8주, 횡보 5-25일, 변동성 &lt;선행의 30% | 플래그 상단 | O'Neil/쿨라메기 |
| **W (이중 바닥)** | 두 저점 차이 &lt;5%, 간격 2-7주, 중간 반등 10-20% | 중간 고점 | O'Neil |
| **Low Cheat** | VCP/CWH 내부에서 피봇 이전 조기 진입, 볼륨 극도 위축 구간 | 수축 저점 근처 | 쿨라메기/Minervini |
| **Episodic Pivot** | 실적/뉴스 촉매, 갭업 +5%, 볼륨 300%+ | 갭업일 고가 또는 5일 내 고점 | 쿨라메기 |

> **Critic 경고**: 모든 수치 기준이 미국 시장 경험칙이다. \[출처 시장: 미국\] 한국 데이터로 Phase 0에서 반드시 파라미터 범위를 재설정해야 한다.

**ML 보조 분류**: 규칙 탐지의 경계 사례를 LightGBM으로 2차 분류. 규칙이 주(primary), ML이 보조(secondary).

### Layer 2: 성공 확률 추정 (02, 07 기반)

```
모델: LightGBM (확률 출력 + Platt scaling 보정)
타겟: 계층적 — P(+10%/10d), P(+20%/10d), P(+30%/10d)
      + 연속 회귀: E[MFE], E[MAE]

입력 feature:
  - 패턴 피처: 수축 깊이/횟수, 볼륨 프로파일, 패턴 기간, 품질 점수
  - 컨텍스트: prior_advance(로그), base_count, trend_duration, RS_rating
  - 시장: market_regime, sector_rs, breadth
  - 기술적: close_vs_50MA, close_vs_200MA, ADR, BB_width

검증: Walk-Forward (expanding window, 18 fold) + CPCV (S=8, PBO < 0.5)
Go/No-Go: OOS precision@top20% > random의 2배, ECE < 0.1
```

> **Critic 경고 (+30% 기준)**: 양성 비율이 \~3-8%로 극심한 클래스 불균형 예상. 연속 수익률 회귀 + 임계값 변환이 이진 분류보다 안정적. Phase 0에서 +10/15/20/30% 빈도를 동시 측정하라.

### Layer 3: 유사 패턴 매칭 (05 기반)

**Phase 1**: 규칙 기반 피처 벡터(\~20차원) + cosine similarity **Phase 2**: TS2Vec 임베딩(\~64차원) + FAISS 인덱스

```
피처 벡터 구성:
  [수축_횟수, 수축1_깊이, 수축2_깊이, 수축_깊이_비율,
   볼륨_위축률, 패턴_기간, 선행_상승률,
   RS_순위, 시총_구간, ATR_비율, 20일MA_기울기, 피봇_근접도, ...]

검색: 현재 종목 패턴 → 벡터화 → Top-K 유사 사례 반환
출력: 각 사례의 차트 이미지 + 10일/20일 forward return + 성공/실패 레이블
```

**FAISS 성능**: 17,500건 패턴 x 25차원 → 브루트포스 &lt;1ms, 메모리 &lt;15MB

> **Critic 경고 (유사도 예측력)**: J.P. Morgan 연구에서 범용 클러스터링은 예측력이 없었다. 도메인 특화 피처(VCP 수축 구조 등)가 핵심. Phase 2에서 유사 사례 Top-K의 forward return이 전체 평균과 유의미하게 다른지 검증 필수.

### Layer 4: 시장 컨텍스트 보정 (07 기반)

```
Regime 분류 (규칙 기반):
  - KOSPI > 200일선 & breadth > 50% → 강세확산
  - KOSPI > 200일선 & breadth < 50% → 강세수렴
  - KOSPI < 200일선 & breadth > 40% → 중립
  - KOSPI < 200일선 & breadth < 40% → 약세

보정: P_final = α × P_model + (1-α) × P_historical_base_rate
```

## 3. 전처리 파이프라인 (06 기반)

종목 간 비교를 가능하게 만드는 핵심 전처리:

### 3.1 가격 정규화 — 다채널 접근 (권장)

| 채널 | 방법 | 보존하는 정보 |
| --- | --- | --- |
| 채널 1 | **피봇 대비 비율** (`close / pivot_price`) | 피봇 대비 현재 위치 |
| 채널 2 | **로그 수익률** (`ln(P_t / P_{t-1})`) | 일별 변동 패턴 (대칭, 가산적) |
| 채널 3 | **윈도우 Z-score** (rolling 60일) | 통계적 이탈도 |
| 채널 4 | **범위 내 위치** (`(close - rolling_min) / (rolling_max - rolling_min)`) | 상대적 고저 위치 |

> **핵심 원칙**: "최적의 정규화 방법을 하나 고르는 것"이 아니라 "정보 손실이 다른 여러 정규화를 채널로 묶어 모델에게 선택을 위임하는 것"이 실전적 해답이다.

### 3.2 거래량 정규화

| 채널 | 방법 | 의미 |
| --- | --- | --- |
| 로그 RVOL | `ln(V_t / SMA(V, 20))` | 평소 대비 배수 (극단값 압축) |
| 백분위 랭크 | 60일 내 순위 | 분포 가정 불필요 |
| 거래량 추세 | `SMA(V, 5) / SMA(V, 20)` | 수축→확장 동적 전환 감지 |

### 3.3 시계열 컨텍스트 인코딩

"같은 VCP, 다른 결과"의 맥락을 포착하는 feature:

| Feature | 계산 | Minervini 용어 |
| --- | --- | --- |
| Prior Advance | `ln(base_start / 52wk_low)` | Stage 2 초기 vs 후기 |
| Base Count | 연속 수축-확장 사이클 수 | 1st base 최강, 3rd+ 약화 |
| Trend Duration | 200MA 위 연속일 / 252 | 추세 성숙도 |
| 52주 고점 근접도 | `close / 52wk_high` | Proper buy point 근접 |
| RS Rating | 종목 RS 백분위 | 상대 강도 |
| 시장 Regime | KOSPI MA50 vs MA200 | 시장 방향 |

### 3.4 윈도우 처리

패턴마다 길이가 다르므로(VCP: 30-180일, HTF: 14-28일):

- **멀티스케일 리샘플링**: 30/60/120일 3개 스케일로 동시 변환
- **세그먼트 기반**: 패턴을 의미 단위(수축1, 반등1, 수축2...)로 분할 후 각 세그먼트 요약 → 고정 길이 벡터
- **DTW**: 유사 패턴 검색 시 가변 길이 직접 비교 (O(n²)이나 17,500건 규모에서 실용적)

## 4. 차트북 자동 생성 (05 기반)

### 4.1 분류 파이프라인

```
패턴 탐지 → 피봇 가격 기록 → 돌파일 식별 → forward return 계산 → 분류
  - 성공 (시원한 돌파): +30%/10일
  - 보통 성공: +10~30%/10일
  - 실패: -7~8% 먼저 도달
  - 무의미: ±5% 이내 횡보
```

### 4.2 차트북 콘텐츠

각 사례에 포함: 종목코드/날짜, 캔들차트(돌파 전 60일+후 20일), 수치 요약(수축폭/횟수/기간), 돌파일 거래량 비율, 시장 컨텍스트(KOSPI 추세/업종 RS), 이동평균 위치, Forward Return(10d/20d/60d), 패턴 유형, 성공 등급

### 4.3 사례 큐레이션

- **전형적 성공**: 품질 점수 상위 25% AND +30% 달성
- **전형적 실패**: 품질 점수 상위 25% AND 실패 — **가장 교훈적인 사례**
- **경계선**: 품질 점수 중간 50%

> 실패 사례를 의도적으로 포함해야 한다. "VCP처럼 보였으나 실패한 사례"가 없으면 확인 편향이 발생한다.

## 5. Phase별 로드맵 (v2)

### v1 대비 변경 사항

| v1 (자동매매) | v2 (의사결정 보조) |
| --- | --- |
| Phase 0: 전제 검증 | Phase 0: **전제 검증 + 패턴 정량화** (최우선) |
| Phase 1: 규칙 + GBT Baseline | Phase 1: **차트북 + 확률 추정 + 유사 검색 MVP** |
| Phase 2: 멀티타겟 + 표현학습 | Phase 2: **임베딩 고도화 + 시장 컨텍스트 보정** |
| Phase 3: RL 도입 | **제거** — 사용자가 직접 결정 |
| 총 9-13개월 | **총 4.5-6.5개월** |

### Phase 0: 전제 검증 + 패턴 정량화 (4-6주)

**목표**: 패턴을 수치로 정의하고, DB에서 탐지하여, 기본 통계를 확인한다. **이 Phase가 전체 프로젝트의 성패를 결정한다.**

```
수행 항목:
1. 규칙 기반 패턴 탐지기 구현 (VCP 우선, 이후 CWH/HTF/W/Low Cheat/EP)
2. 전 종목 스캔 → 패턴 이벤트 수 실측 (17,500건 추정 검증)
3. Forward return 계산: +10%, +15%, +20%, +30% 달성 빈도 동시 측정
4. 수동 검토 50건: 규칙 탐지 vs 전문가 판단 일치율
5. 차트북 v0 생성: 탐지된 모든 패턴의 차트 이미지 + 결과

Go 조건:
  - 패턴 이벤트 ≥ 5,000건
  - +30%/10d 양성 샘플 ≥ 500건 (부족 시 기준 완화)
  - 수동 검토 일치율 ≥ 60%

No-Go 시:
  - 양성 500건 미만 → 기준을 +20%/10d 또는 +30%/20d로 완화
  - 일치율 60% 미만 → 파라미터 한국 시장에 재설정
```

### Phase 1: 확률 추정 + 유사 검색 MVP (6-8주)

**전제**: Phase 0 Go 통과

```
산출물:
1. LightGBM 확률 예측기 + Platt scaling (계층적: P(+10/20/30%))
2. 피처 벡터 기반 유사 사례 검색 (cosine similarity Top-K)
3. 의사결정 카드 생성기
4. Walk-Forward 검증 결과 (18 fold)
5. 차트북 v1: 성공/실패 자동 분류 + 큐레이션

Go/No-Go:
  - OOS precision@top20% > random의 2배
  - ECE < 0.1 (확률 보정 품질)
  - 유사 사례 Top-10의 forward return이 전체 평균과 유의미하게 다름
```

> **Critic 권고**: Phase 1을 VCP 단독(Phase 1a, 3-4주)과 나머지 패턴(Phase 1b, 4-5주)으로 분할하는 것이 현실적이다.

### Phase 2: 고도화 (8-12주, 선택)

**전제**: Phase 1이 실용적 가치를 보여야 함

```
산출물:
1. TS2Vec SSL pre-training → 패턴 임베딩 (FAISS 인덱스)
2. 시장 regime 보정 모듈 (조건부 확률 테이블)
3. CPCV (S=8) + PBO < 0.5 검증
4. Multi-output 모델 (MFE, MAE, 도달 시간)
5. 멀티스케일 유사도 (일봉 0.4 + 주봉 0.3 + regime 0.3)
```

## 6. 근거 신뢰도 매트릭스

## 8. 역방향 의사결정 가이드

| 관찰 결과 | 조정 방향 |
| --- | --- |
| Phase 0에서 +30%/10d 양성 &lt; 300건 | → 기준을 +20%/10d로 완화 |
| Phase 0에서 +30%/10d 양성 &lt; 100건 | → 기준을 +30%/20d로 변경 |
| 규칙 탐지-전문가 일치율 &lt; 50% | → 파라미터를 한국 데이터 분포에서 재추출 |
| Phase 1 OOS precision이 random 수준 | → feature engineering 재검토 (컨텍스트 feature 비중 확대) |
| 유사 사례 Top-K의 forward return이 전체 평균과 동일 | → 규칙 피처 유사도 폐기, 임베딩 실험 조기 착수 |
| 특정 regime에서만 성과 집중 | → Regime 조건부 모델 분리 |
| VCP 단독으로 Phase 1a 가치 확인 실패 | → 패턴 기반 접근 자체를 factor-based 접근으로 전환 |

## 9. 예상 밖 핵심 발견 (v1 + v2)

### 9.1 인용 수치의 도메인 공백 (v1 유지)

세 Researcher가 인용한 핵심 성과 수치 중 **한국 증시 개별주 breakout 분류**에서 나온 것이 없다. 보강분(R4\~R7)도 마찬가지.

### 9.2 VCP 직접 ML 인식 선행연구 부재 (v1 유지)

VCP를 ML로 직접 인식한 학술 논문은 발견되지 않았다.

### 9.3 유사 패턴 매칭의 이중성 (v2 신규)

J.P. Morgan 연구에서 범용 클러스터링은 예측력이 없었으나, Savin et al.에서 도메인 특화 패턴(H&S)은 5-7% 초과수익을 보였다. **"어떤 피처로 유사도를 측정하는가"가 예측력을 결정한다.**

### 9.4 컨텍스트가 패턴 형태보다 중요할 수 있음 (v2 신규)

Minervini와 쿨라메기 모두 "같은 패턴이라도 1st base와 3rd base는 완전히 다르다"고 강조한다. 시계열 컨텍스트(prior advance, base count, regime) 인코딩이 패턴 형태 인식만큼 또는 그 이상 중요하다.

### 9.5 의료 영상 진단 보조와의 구조적 동형 (v2 신규)

"패턴 인식 → 확률 추정 → 유사 사례 제시 → 전문가 판단"이라는 파이프라인이 의료 영상 AI와 동일하다. 의료 CBIR의 Coarse-to-Fine 검색, Relevance Feedback 등을 차용 가능.
