### B.2 각 Layer의 기술적 상세

#### Layer 1: 패턴 정량화 엔진

**핵심 설계 원칙**: O'Neil/Minervini/Ryan/쿨라메기/댄쟁거/TraderLion의 패턴 정의를 코드로 번역하는 것이 ML보다 선행해야 한다. ML은 규칙 기반 탐지의 "보조"이지 "대체"가 아니다.

**패턴별 수치 정의**:

| 패턴 | 핵심 피처 | 수치 기준 (초기값, 튜닝 대상) |
|------|----------|--------------------------|
| **VCP** | 수축 횟수, 각 수축 깊이, 깊이 감소율, 볼륨 위축률 | 수축 2-6회, 깊이비 <0.75, 볼륨 <50% avg, 최종 수축 <8% |
| **CWH** | 컵 깊이, 컵 기간, 핸들 깊이, 핸들 기간 | 컵 12-65%, 7-65주, 핸들 8-12%, 1-4주 |
| **HTF** | 선행 상승률, 횡보 기간, 횡보 변동성, 피봇 근접도 | 선행 +25%↑, 횡보 5-25일, ATR <선행의 30%, 고점 3% 이내 |
| **W (이중 바닥)** | 두 저점 간 깊이 차이, 간격, 중간 반등 높이 | 저점 차 <3%, 간격 2-7주, 반등 10-20% |
| **Low Cheat** | 기존 패턴 내 매수 기회, 피봇 전 조기 진입 조건 | 패턴 하단부 터치 + 볼륨 급감 + 전일 대비 축소 |
| **Episodic Pivot** | 어닝/뉴스 촉매, 갭업 크기, 볼륨 배수 | 갭 +5%↑, 볼륨 >300% avg, 이전 기간 대비 신고가 |

**수치 투명성**: 위 기준값은 Minervini("Trade Like a Stock Market Wizard"), O'Neil("How to Make Money in Stocks"), TraderLion 교육 자료에서 도출한 것이다. 이 수치가 틀릴 수 있는 조건: (1) 한국 증시의 변동성 구조가 미국과 다르므로 깊이 기준 조정 필요, (2) 시총/유동성 구간별로 최적 기준이 다를 수 있음. 따라서 초기값으로 사용하되, 백테스트를 통해 한국 데이터에 맞게 보정해야 한다.

**구현 접근**:
```
detect_pattern(ticker, date):
    1. 피봇 포인트 식별 (scipy.signal.argrelextrema, window=5-10)
    2. 피봇 시퀀스에서 수축 구간 추출
    3. 각 패턴 유형의 규칙 적용 → 매칭 여부 + 품질 점수
    4. 복수 패턴 매칭 시 품질 점수 최고 패턴 선택
    5. 결과: {pattern_type, quality_score, contractions[], pivot_price, features{}}
```

**반증 탐색**: "규칙 기반 탐지가 ML보다 낫다"는 가정에 대해 — Heliyon 2024 논문에서 DL 기반 패턴 인식이 94.9% accuracy를 달성했다. 그러나 이는 H&S, double top 등 교과서적 패턴이며, VCP/CWH 같은 Minervini 특화 패턴에 대한 DL 연구는 미발견. 또한 높은 accuracy가 반드시 실전 수익으로 이어지지 않는다(precision/recall tradeoff). 규칙 기반의 장점은 해석 가능성과 도메인 전문가(사용자)의 피드백 반영 용이성이다. 반증 부분 확인되나, 현 단계에서는 규칙 기반 우선이 합리적.

#### Layer 2: 성공 확률 추정

**기존 보고서(02)에서 거의 그대로 활용 가능**하되, 타겟 정의를 수정한다.

기존 타겟: "10일 내 +10% 달성"
수정 타겟: **"피봇 돌파 후 10일 내 +30% 달성"** (사용자 정의 "시원한 돌파")

**이 변경의 영향**:
- 성공 비율이 극도로 낮아짐 (추정 3-8%). +10% 기준의 20% 대비 크게 감소.
- 극심한 클래스 불균형 → `scale_pos_weight` 필수, focal loss 검토
- precision이 극히 중요 — false positive(모델이 성공이라 했으나 실패)의 비용이 큼
- **threshold를 높게 설정**: 기본 0.5 → 0.7-0.8로 올려서 "정말 확실한 것만" 알림

**수치 투명성**: "10일 내 +30%"는 매우 공격적인 기준이다. 한국 증시에서 일일 상한가 30%를 감안해도, 10일 내 +30%를 달성하려면 사실상 연속 강세가 필요하다. 2,500종목 x 2년에서 이 기준을 충족하는 사례가 충분한지(최소 500건) 확인이 선행되어야 한다. 부족하면 기준을 "+20% in 10d" 또는 "+30% in 20d"로 완화하는 것을 권장한다.

**실행 연결**: 타겟 사례 수를 먼저 집계하라. SQL 쿼리로 `SELECT COUNT(*) FROM forward_labels WHERE max_gain_10d >= 30`을 실행하여 500건 미만이면 기준 조정.

| 방법 | 장점 | 단점 | 권장 단계 |
|------|------|------|----------|
| **피처 벡터 + cosine similarity** | 구현 간단, 해석 가능, 빠름 | 패턴의 "형태" 정보 일부 손실 | **Phase 0 (즉시)** |
| **DTW (Dynamic Time Warping)** | 시간축 변형에 강건, 형태 직접 비교 | O(n²) 연산, 대규모 DB에서 느림 | Phase 1 |
| **TS2Vec 임베딩 + FAISS** | 의미적 유사성 포착, 확장성 좋음 | SSL 학습 필요, 해석 어려움 | Phase 2 |

**Phase 0 권장 구현**:

피처 벡터 기반 유사도 검색:
```
패턴 피처 벡터 (정규화 후):
[수축_횟수, 수축1_깊이, 수축2_깊이, ..., 수축_깊이_비율,
 볼륨_위축률, 패턴_기간(일), 선행_상승률,
 RS_순위, 시총_구간, 섹터_코드,
 ATR_비율, 20일MA_기울기, 피봇_근접도]

→ ~15-25차원 벡터
→ 전체 DB의 패턴을 벡터화하여 저장
→ 쿼리 패턴과 cosine similarity 상위 K개 반환
```

**출처 평가**: 유사 패턴 매칭 접근은 JP Morgan의 "Searching for Patterns in Daily Stock Data" 연구 [jpmorgan.com], Gabor Vecsei의 Stock Pattern Analyzer (FAISS + sliding window), ExeQution Analytics의 Time Series Similarity Search (cosine similarity for intraday patterns)에서 실증적으로 사용되고 있다. [인접 도메인: 정보 검색/추천 시스템] 벡터 유사도 검색은 검색 엔진과 추천 시스템에서 수십 년간 검증된 기술이며, 금융 시계열에의 적용은 기술적으로 성숙하다.

**FAISS 인덱스 규모 추정**:
- 17,500건 패턴 x 25차원 = 매우 소규모. 브루트포스 검색도 <1ms
- 규모가 커져도 FAISS IndexFlatIP로 충분 (IVF 불필요)

#### Layer 4: 시장 컨텍스트 보정

**기존 보고서(03 E절)의 한국 증시 특수성을 "보정 변수"로 구조화**:

```
시장 regime 분류 (규칙 기반):
  - KOSPI > 200일선 & breadth > 50% → 강세확산
  - KOSPI > 200일선 & breadth < 50% → 강세수렴
  - KOSPI < 200일선 & breadth > 40% → 중립
  - KOSPI < 200일선 & breadth < 40% → 약세

조건부 확률 테이블 (백테스트로 채움):
  regime × pattern_type → 역사적 성공률

보정 공식:
  P_final = α × P_model + (1-α) × P_historical_base_rate
  α = 모델 신뢰도 (ECE 기반, 보정이 잘 될수록 α 높임)
```

**관점 확장**: 시장 regime 외에 **섹터 로테이션 상태**도 보정 변수로 중요하다. 예를 들어 반도체 섹터가 리딩 중일 때 반도체 종목의 VCP 성공률은 전체 평균보다 유의미하게 높을 수 있다. 이는 O'Neil의 "Industry Group Rank" 개념과 일치한다.

### B.3 차트북 자동 생성

**목적**: 과거 패턴 사례를 체계적으로 축적하여 "패턴 인식 눈"을 기르는 동시에 ML 학습 데이터로 활용.

**구조**:
```
chartbook DB (SQLite):
├── patterns 테이블
│   ├── id, ticker, date, pattern_type, quality_score
│   ├── features_json (피처 벡터)
│   ├── embedding_vector (Phase 2에서 추가)
│   ├── chart_image_path (렌더링된 캔들차트)
│   ├── forward_return_10d, forward_return_20d
│   ├── mfe_10d, mae_10d
│   ├── success_label (10일 +30% 달성 여부)
│   ├── market_regime
│   └── notes (사용자 메모, 선택)
```

**차트 렌더링**: `mplfinance` 라이브러리로 캔들차트 + 볼륨 + 이동평균을 PNG로 렌더링하여 저장. 패턴 구간에 하이라이트 표시.

**성공/실패 모두 포함**: 라벨링 시 "VCP처럼 보였으나 실패한 사례"를 의도적으로 포함해야 한다(01 보고서 E.3절 생존 편향 경고). 규칙 기반 탐지기가 잡아낸 모든 패턴을 저장하고, forward label로 성공/실패를 자동 태깅.

## C. 수정된 Phase 로드맵

### Phase 0: 데이터 기반 + 규칙 엔진 (4-6주)

**목표**: 패턴을 수치로 정의하고, DB에서 자동 탐지하여, 차트북의 초기 버전을 구축한다.

**산출물**:
1. `pattern_detector.py` — 6개 패턴의 규칙 기반 탐지기
2. `chartbook.db` — 2년치 전 종목 스캔 결과 (추정 15,000-20,000건)
3. `chart_renderer.py` — 패턴별 캔들차트 PNG 생성
4. `forward_labeler.py` — 각 패턴의 10일/20일 후 수익률 자동 계산
5. **탐색적 분석 보고서**: 패턴별 성공률 base rate, 분포, regime별 차이

**Go/No-Go**:
- Go 조건: 패턴별 최소 200건 이상 탐지, 성공 기준(+30% 10d) 사례 최소 300건
- No-Go 시: 성공 기준 완화 (+20% 10d 또는 +30% 20d), 패턴 정의 기준 조정
- **이 Phase의 분석이 전체 프로젝트의 실현 가능성을 결정한다**

**핵심 라이브러리**: `pandas`, `numpy`, `scipy.signal`, `sqlite3`, `mplfinance`

**소요 기간 근거**: 규칙 기반 코딩 자체는 2주면 충분하나, 패턴 정의의 세부 기준을 한국 데이터에 맞게 조정하는 반복 작업이 2-4주 추가.

### Phase 1: 확률 추정 + 유사 검색 MVP (6-8주)

**목표**: "이 패턴이 얼마나 유망한가"에 대한 정량적 답변을 제공하고, "과거에 비슷한 사례가 어땠는가"를 보여준다.

**전제 조건**: Phase 0에서 충분한 사례 확보 + base rate 확인 완료

**산출물**:
1. `probability_model.py` — LightGBM 확률 예측기 + Platt scaling
2. `similarity_search.py` — 피처 벡터 기반 Top-K 유사 사례 검색
3. `decision_card.py` — 최종 출력 카드 생성 (B.1 아키텍처의 최종 출력)
4. Walk-Forward 검증 결과 (18개 fold OOS 성과)
5. Reliability Diagram (확률 보정 품질 확인)

**Go/No-Go**:
- Go 조건: Walk-Forward OOS에서 precision@top20% > random의 2배, ECE < 0.1
- No-Go 시: 피처 엔지니어링 재검토. 패턴 정의 기준 자체가 불충분할 가능성 → Phase 0으로 회귀

**핵심 라이브러리**: `lightgbm`, `scikit-learn`, `optuna`, `faiss-cpu`

### Phase 2: 고도화 (8-12주, 선택적)

**목표**: 임베딩 기반 유사 검색으로 업그레이드, 시장 컨텍스트 보정, CPCV 검증.

**전제 조건**: Phase 1이 실용적 가치를 보여야 함 (사용자가 실제로 사용하며 도움이 된다고 판단)

**산출물**:
1. TS2Vec SSL pre-training → 패턴 임베딩
2. FAISS 인덱스 + 임베딩 기반 유사 검색
3. 시장 regime 보정 모듈
4. CPCV (S=8) 검증 + PBO 계산
5. Multi-output 모델 (MFE, MAE, 도달 시간 동시 예측)

**Go/No-Go**:
- Go 조건: 임베딩 기반 유사 검색이 피처 기반 대비 "사용자 체감" 유사도 우수, CPCV PBO < 0.5
- No-Go 시: 피처 기반 유사 검색 유지, SSL 투자 중단

**핵심 라이브러리**: 기존 + `pytorch`, `ts2vec` (GitHub), `faiss-gpu` (선택)

## D. 핵심 설계 판단과 근거

### D.1 왜 규칙 기반을 ML보다 우선하는가

1. **해석 가능성**: 사용자가 "왜 이것이 VCP인가"를 이해할 수 있어야 의사결정에 활용 가능. "모델이 0.73이라고 했다"는 불충분.
2. **피드백 루프**: 규칙의 기준값(수축 깊이 <8% 등)을 사용자가 직접 조정 가능. 블랙박스 모델은 이것이 불가.
3. **디버깅**: 패턴 탐지가 이상할 때 어떤 규칙이 문제인지 즉시 파악 가능.
4. **데이터 효율성**: 규칙 기반은 학습 데이터 불필요. ML은 보조적으로만 사용하므로 데이터 부족 리스크 감소.
5. **도메인 지식 직접 반영**: O'Neil/Minervini의 수십 년 경험을 코드로 직접 번역.

### D.2 성공 기준 "+30% in 10d"의 현실성 검증

이 기준이 프로젝트 전체의 실현 가능성을 좌우하므로, Phase 0에서 반드시 확인해야 할 사항:

**예비 추정**:
- 한국 증시 2,500종목 x 500일 = 1,250,000 일봉
- 일봉 기준 10일 내 +30% 달성 확률: 대략 0.5-2% (시장 전체 무조건부)
- 패턴 발생 건수 ~17,500건 중 +30%/10d 달성: 추정 175-700건

이 수치가 틀릴 수 있는 조건: (1) 2020-2022 강세장 데이터 비중이 높으면 과대 추정, (2) 소형주 포함 비율에 따라 변동, (3) 상한가 30% 제한이 영향.

**권장**: Phase 0 첫 주에 forward label을 먼저 계산하여 사례 수를 확인하고, 부족하면 기준을 조정.

### D.3 유사 패턴 매칭의 가치

**의사결정 관점에서 유사 패턴 매칭이 확률 모델보다 더 직접적인 가치를 제공할 수 있다.**

이유:
1. **구체적 사례는 추상적 확률보다 설득력이 있다**: "23% 확률"보다 "비슷했던 5개 사례 중 2개 성공, 이렇게 움직였다"가 트레이더의 판단에 더 도움.
2. **패턴의 실패 모드를 학습할 수 있다**: 유사 실패 사례의 차트를 보면 "어떤 조건에서 실패하는가"를 직관적으로 파악.
3. **확률 모델의 한계를 보완**: 모델이 높은 확률을 부여했더라도, 유사 사례를 보면 "이 종류의 패턴은 올라가다 급락하는 경우가 많다" 등의 정성적 판단 가능.

[이질 도메인: 의료 진단] 영상의학에서 AI가 "악성 확률 73%"라고 하면 의사가 반드시 유사 사례 이미지를 참조하여 판단한다. 확률 숫자만으로는 불충분하며, 유사 사례의 시각적 비교가 전문가 의사결정의 핵심 보조 도구다. 이 패턴이 주식 패턴 인식에도 정확히 적용된다.

## F. 관점 확장

### F.1 숨은 변수: 패턴 정의의 주관성이 최대 리스크

모든 Layer의 품질 상한은 **Layer 1의 패턴 정의 품질**에 의해 결정된다. "이것이 VCP인가 아닌가"의 판단이 사람마다 다르다면, ML 모델은 노이즈를 학습하게 된다. 이 문제를 완화하기 위해:

1. **보수적 기준으로 시작**: 명확한 VCP만 잡고, 경계 사례는 "미분류"로 남기기
2. **사용자 피드백 루프**: 탐지 결과를 사용자가 검토하고 수정 → 라벨 품질 점진적 향상
3. **패턴 품질 점수에 연속값 사용**: binary(VCP/비VCP)보다 품질 점수(0-100)가 경계 사례 처리에 유리

### F.2 숨은 변수: "시원한 돌파"의 정의 자체가 전략의 수익성을 결정

+30%/10일이라는 기준은 매우 공격적이다. 이 기준이 높을수록:
- 성공 사례가 적어져 학습 데이터 부족
- precision은 높아지지만 recall(기회 포착률)이 급감
- 실제 매매 기회가 연 몇 건으로 줄어들 수 있음

대안적 기준 체계:
```
티어 1 (시원한 돌파): +30% in 10d → 가장 높은 우선순위
티어 2 (양호한 돌파): +20% in 10d 또는 +30% in 20d
티어 3 (보통 돌파): +10% in 10d
실패: 피봇 돌파 후 -7% 먼저 도달 (손절 기준)
```

이 티어 시스템은 multi-output 모델의 자연스러운 확장이며, 사용자에게 "시원한 돌파 확률은 낮지만 양호한 돌파 확률은 높다"는 풍부한 정보를 제공한다.

> **더 적절한 핵심 질문: "트레이더가 이미 차트를 보고 판단하는 과정에서, 어떤 정보가 추가되면 판단의 질이 향상되는가? 그리고 그 정보를 어떤 순서로 구축하는 것이 1인 개발에서 가장 빠르게 실용적 가치를 만드는가?"**

이 재정의의 핵심: 시스템의 목표는 "트레이더를 대체하는 것"이 아니라 "트레이더가 놓치는 것을 보여주는 것"이다. 구체적으로:
- 규칙이 잡아내지 못하는 미묘한 패턴 → ML 보조
- 기억하지 못하는 과거 유사 사례 → 유사 검색
- 감으로 판단하는 시장 상황 → 조건부 확률
- 확인 편향 → 실패 사례 강제 노출
