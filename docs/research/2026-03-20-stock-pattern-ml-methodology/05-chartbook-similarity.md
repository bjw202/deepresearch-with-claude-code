## A. 차트북 자동 생성

### A1. 패턴 탐지 후 성공/실패 자동 분류

**분류 기준 정의**: "피봇 돌파 후 10일 내 +30%"라는 성공 기준은 명확한 이진 레이블링을 가능하게 한다. 구현 절차는 다음과 같다:

1. **피봇 포인트 탐지**: `scipy.signal.find_peaks`로 로컬 고점/저점 식별. ZigZag 알고리즘(임계값 5~10%)으로 유의미한 스윙만 추출.
2. **패턴 규칙 매칭**: 각 패턴별 프로그래매틱 조건 정의
   - **VCP**: 2~6회 연속 수축, 각 수축 깊이 감소(DD_n > DD_{n+1} × 1.1), ATR(14) 점진 축소, 거래량 40일 평균 대비 감소
   - **CWH**: U형 바닥(V형 제외), 좌우 lip 레벨 ±5% 이내, 핸들 깊이 < 컵 깊이 50%, 핸들 기간 1~4주
   - **HTF**: 단단한 횡보 구간(일봉 변동폭 <3%), 거래량 감소, 50일선 위 유지
   - **W바닥**: 이중 저점(±3% 이내), 중간 고점 돌파, 거래량 증가
   - **Episodic Pivot**: 어닝/뉴스 후 갭업 + 거래량 300%+ 급등, 이후 눌림목 형성
3. **Forward Return 계산**: 돌파일(피봇 고점 상향 돌파 + 거래량 150%+) 기준 T+1~T+10 최고 수익률 계산. ≥30%면 성공, <30%면 실패.

**수치 투명성**: 위 임계값(수축 1.1배 감소, 핸들 50%, 변동폭 3% 등)은 Minervini의 정성적 가이드라인을 프로그래밍 가능하게 수치화한 것이다. 실제로는 종목의 평소 변동성(ATR 기준)에 따라 적응적으로 조정해야 하며, 고변동성 종목에서는 이 수치가 과소 설정될 수 있다.

### A2. 차트북 포함 정보

각 사례(차트북 한 페이지)에 포함해야 할 정보:

| 구분 | 항목 | 구현 방법 |
|------|------|----------|
| **차트 이미지** | 캔들스틱 + 이동평균선 + 거래량 | `mplfinance`로 자동 생성, 패턴 구간 하이라이트 |
| **수치 요약** | 수축 횟수/깊이, RS 순위, 거래량 dry-up 비율 | 패턴 탐지 과정에서 자동 계산 |
| **시장 컨텍스트** | 코스피 위치(200일선 대비), 업종 RS, 시장 폭(상승종목비율) | 돌파일 기준 시장 데이터 조회 |
| **Forward Return** | T+5/10/20 수익률, 최대 drawdown | 돌파 후 가격 데이터에서 계산 |
| **패턴 품질 점수** | 0~100 복합 점수 | 수축 깔끔함, 거래량 패턴, RS 등 가중 합산 |

**저장 형식**: SQLite에 메타데이터 테이블, 차트 이미지는 파일시스템(또는 BLOB). JSON으로 수치 요약 직렬화.

### A3. 사례 큐레이션 — "전형적 성공/경계선/전형적 실패" 자동 선별

**방법론**: 패턴 품질 점수 × 실제 결과의 2차원 분류

1. **전형적 성공**: 품질 점수 상위 25% AND 성공(+30% 달성)
2. **전형적 실패**: 품질 점수 상위 25% AND 실패 — 교과서적 패턴이지만 실패한 사례로 가장 교훈적
3. **경계선**: 품질 점수 중간 50% — 판단이 어려운 사례
4. **저품질**: 품질 점수 하위 25% — 차트북에서 제외하거나 별도 분류

**대안적 접근**: K-Means 또는 DBSCAN으로 패턴 피처 공간에서 클러스터링 후, 각 클러스터의 성공률과 중심(centroid)으로 전형성 판단. JP모건 연구에서 이 접근으로 autoencoder 인코딩 후 K-Means 적용, 특정 클러스터가 높은 수익과 상관됨을 확인.

### A4. 패턴 유형 간 자동 분류 (VCP vs CWH 겹침)

**문제**: VCP와 CWH는 구조적으로 겹침이 크다. 컵의 우측이 수축하면 VCP이면서 동시에 CWH의 핸들이 될 수 있다.

**해결 접근**:

1. **규칙 우선순위**: 각 패턴 규칙을 독립 실행 → 복수 매칭 시 "가장 구체적인 패턴" 우선. 예: CWH는 U형 바닥 요건이 추가되므로 VCP보다 구체적.
2. **다중 레이블 허용**: 실무적으로 한 사례가 여러 패턴에 속할 수 있음을 인정. 차트북에서 `[VCP+CWH]`로 표기.
3. **QuantConnect 토론에서의 제안**: "VCP를 Log 변환한 가격에서 선형 추세 제거 후 수축을 측정하고, CWH는 곡률(curvature) 측정으로 U형 여부 판별"하여 구분. 이 방법이 가장 실용적.

**반증 탐색**: 완벽한 자동 분류는 불가능하다는 것이 현실이다. TrendSpider, ChartPatterns.ai 등 상용 도구도 "trend-agnostic" 탐지를 하며, 최종 판단은 트레이더에게 맡긴다. 자동 분류는 "후보 제시"까지가 적절한 목표이며, 최종 분류는 차트북 큐레이션 과정에서 수동 확인하는 것이 품질을 보장한다.

## B. 유사 패턴 검색

### B1. 시계열 유사도 측정 방법 비교

| 방법 | 원리 | 장점 | 단점 | 적합 시나리오 |
|------|------|------|------|-------------|
| **DTW** | 시간축 왜곡 허용 비선형 정렬 | 템포 차이에 강건, 직관적 | O(n²) 연산, 고차원 비효율 | 소규모 비교, 기준 패턴 대조 |
| **피처 벡터 (규칙 기반)** | 수축깊이/횟수, RS, 거래량비율 등 수작업 피처 | 해석 가능, 빠름, 도메인 지식 반영 | 피처 설계에 의존, 미묘한 형태 놓침 | **1단계 구현에 최적** |
| **임베딩 (TS2Vec/AE)** | 자기지도학습으로 잠재 벡터 추출 | 범용, 자동 피처 | 학습 필요, 해석 어려움 | 대규모 데이터, 2단계 고도화 |
| **차트 이미지 CNN** | 캔들스틱 이미지를 CNN으로 피처 추출 | 시각적 패턴 직접 포착 | 시간 순서 일부 손실, GPU 필요 | 멀티 인디케이터 시각화 시 |
| **Recurrence Plot + CNN** | 시계열→2D 이미지→CNN | 비선형 동역학 포착 | 변환 비용, 해석 어려움 | 연구/실험 단계 |

**핵심 판단**: 1인 개발자 환경에서 **1단계는 규칙 기반 피처 벡터 + FAISS가 최적**이다. TS2Vec이나 CNN 기반 접근은 2단계 고도화에서 검토한다.

**반증**: 2025년 arXiv 연구(Investigating Market Strength Prediction with CNNs on Candlestick Charts)에서 "시계열 데이터가 이미지 데이터보다 CNN 패턴 인식에서 우월하다"는 결과를 보고. 즉, 굳이 이미지로 변환하지 않고 1D 시계열을 직접 다루는 것이 더 나을 수 있다.

### B2. 검색 시스템 아키텍처

**추천 아키텍처**: FAISS 기반 벡터 검색

```
[SQLite DB] → [피처 추출] → [벡터 인덱스(FAISS)] → [쿼리] → [Top-K 유사 사례] → [후속 성과 표시]
```

**Gábor Vecsei의 Stock-Pattern-Analyzer**가 정확히 이 구조의 참조 구현이다:

1. **슬라이딩 윈도우 추출**: N개 종목의 가격 시계열에서 M일 윈도우를 1일 단위로 추출
2. **Min-Max 정규화**: 각 윈도우를 [0,1]로 스케일링 → 절대 가격이 아닌 패턴(형태)만 비교
3. **인덱스 구축**: FAISS IndexIVFFlat 또는 양자화 인덱스
4. **쿼리**: 현재 종목의 최근 M일 패턴 → Top-K 유사 패턴 반환
5. **후속 성과 조회**: 각 유사 패턴의 T+10 수익률 조회 → 성공/실패 분포 시각화

**성능 벤치마크** (Vecsei 측정, 노트북 CPU 기준):

| 인덱스 유형 | 빌드 시간(ms, 윈도우=20) | 메모리(MB) | 쿼리(ms) |
|------------|------------------------|-----------|---------|
| FAISS FastIndex | 1.81 | 13.92 | 2.47 |
| FAISS MemoryEfficient | 5,959 | 2.12 | 0.25 |
| scipy cKDTree | 206 | 31.49 | 24.68 |

**2,500종목 × 500일 × 윈도우20 = ~125만 벡터**. FAISS FastIndex로 14MB 메모리, 2.5ms 쿼리면 충분히 실시간 대응 가능하다.

**수치 투명성**: 이 벤치마크는 S&P 500(~500종목) 기준이며, 2,500종목으로 확장 시 선형 스케일링 예상. FAISS IVFFlat의 nlist(클러스터 수)를 100→500으로 조정 필요.

### B3. 유사도의 예측력 — 유사 패턴이 유사 결과를 낳는가?

**긍정적 증거**:

1. **Savin, Weller, Zvingelis (2007)**: S&P500, Russell2000에서 Head-and-Shoulders 패턴이 향후 초과수익 5~7% 예측. 리스크 조정 후에도 유의미. [인접 도메인: 미국 증시, 한국 소형주와 유동성/변동성 차이 존재]
2. **Yao & Wei (2017)**: 상하이 증시에서 수정 DTW 기반 유사 패턴 매칭, 방향 예측 정확도 79%. 벤치마크 대비 15~25% 우위. 단, 20개 종목 소규모 테스트.
3. **Nakagawa et al. (2019)**: DTW + K-medoids 클러스터링으로 주가 패턴 추출, 벤치마크 대비 총수익 60%+ 초과.
4. **Wang & Chan (2007)**: NASDAQ/TWI에서 Bull Flag 패턴 매칭이 높은 수익과 상관.
5. **Barroso (2020s)**: DTW + SVM/KNN 결합, 강세 패턴 투자 시 연 43% 수익 (벤치마크 11%).

**부정적/제한적 증거**:

1. **Lo, Wang, Mamaysky (2000)**: 가장 정교한 패턴 인식 연구. 패턴의 통계적 존재는 확인했으나, "한계 증분 수익(marginal incremental returns)"은 매우 작고 거래비용 후 생존 불확실.
2. **Nature (2025)**: "차트 분석가가 주장하는 패턴은 신뢰할 수 있는 예측을 제공하기에 불충분하며, 랜덤하게 발생할 가능성이 높다." 펀더멘털(뉴스, 재무제표, 산업 사이클) 통합이 필수.
3. **CNN 이미지 연구들**: 캔들스틱 이미지 CNN 분류 정확도가 높아 보이지만, 대부분 방향(상승/하락) 분류이며, +30% 같은 고수익 예측과는 다른 문제.

**핵심 판단**: "유사 패턴 → 유사 결과"는 **통계적 경향성(edge)**이 있으나, 단독으로는 충분하지 않다. 시장 컨텍스트(regime), 펀더멘털, 거래량 등 추가 조건이 예측력을 유의미하게 높인다. VCP/CWH 같은 특정 패턴에서의 검증이 Head-and-Shoulders 같은 일반 패턴보다 학술 증거가 적다는 점에 유의해야 한다.

**이 정보로 가능한 의사결정**: 유사 패턴 검색 결과를 "확률적 참고자료"로 사용하되, 최종 매매 판단의 유일한 근거로 삼지 않는다. "유사 사례 10개 중 7개 성공"이면 진입 검토, "10개 중 4개 성공"이면 추가 확인 필요, 이런 식의 의사결정 프레임워크가 적절하다.

### B4. 멀티스케일 유사도: 일봉 + 주봉 + Regime 결합

**접근 방법**:

1. **차원별 분리 유사도 계산(Dimension-Combination)**:
   - 일봉 패턴 유사도 (20~60일 윈도우)
   - 주봉 패턴 유사도 (12~26주 윈도우)
   - Regime 유사도 (시장 상태: 강세/약세/횡보)
2. **가중 합산**: Borda Voting 또는 단순 가중 평균으로 최종 유사도 산출
   - 예: `종합유사도 = 0.4 × 일봉유사도 + 0.3 × 주봉유사도 + 0.3 × regime일치도`
3. **Regime 판별**: 코스피 200일선 위치, 시장 폭(상승종목비율), ADR(Advance-Decline Ratio) 등으로 3~5개 상태 분류

**Multiscale Dubuc Distance(MDD)**: 2024년 제안된 새로운 멀티스케일 유사도 메트릭으로, 선형 시간복잡도(DTW의 O(n²) 대비), 메트릭 공간 성질(삼각부등식 만족), 파라미터 튜닝 최소화가 장점. 아직 금융 시계열 적용 사례는 없으나, 원리적으로 적합하다.

**실행 연결**: 멀티스케일은 2단계 이후 구현이 적절하다. 1단계에서는 일봉 단일 스케일로 시작하고, 주봉 유사도를 필터로 추가하는 것이 실용적이다.

## C. 이질 도메인 차용

### C1. 의료 영상 CBIR에서 차용 가능한 구조

[이질 도메인: 의료 영상 검색]

의료 CBIR 시스템의 핵심 구조:

```
환자 증상/영상 → 피처 추출(CNN backbone) → 벡터 DB 검색 → 유사 과거 사례 K개 → 진단 보조
```

이 구조는 패턴 매칭 시스템과 **동일한 아키텍처**이다:

```
현재 차트 → 피처 추출(규칙 기반/임베딩) → 벡터 DB 검색 → 유사 과거 사례 K개 → 매매 의사결정 보조
```

**차용 가능한 패턴**:

1. **Relevance Feedback**: 사용자가 검색 결과 중 "이건 관련 있다/없다"를 피드백 → 검색 정밀도 개선. 트레이더가 유사 사례를 보고 "이건 진짜 비슷하다/아니다"를 표시하면 피처 가중치를 학습할 수 있다.
2. **멀티모달 검색**: 의료 CBIR은 영상 + 텍스트(진단명, 환자 정보)를 결합. 차트 패턴도 가격 형태 + 펀더멘털(매출 성장률, 기관 보유비율) + 시장 컨텍스트를 결합하면 검색 품질이 높아진다.
3. **계층적 검색**: 먼저 해부학적 영역(폐/뇌/간)으로 좁히고 세부 병변 매칭. 차트도 먼저 패턴 유형(VCP/CWH)으로 좁히고 세부 형태 매칭.

**도메인 차이의 영향**: 의료 영상은 정적 이미지이고 시간 축이 없다. 주가 패턴은 시간 순서가 본질적이므로, 시간 정보를 보존하는 피처 설계가 필수적이다.

### C2. 추천 시스템 Collaborative Filtering과의 유사성

[이질 도메인: 추천 시스템]

**구조적 유사성**:

| 추천 시스템 | 패턴 매칭 시스템 |
|------------|----------------|
| User | 종목(또는 시점) |
| Item | 패턴 피처/특성 |
| Rating | 후속 수익률 |
| "이 영화를 좋아한 사람은 이것도 좋아합니다" | "이 패턴을 보인 종목은 이런 결과가 나왔습니다" |

**Mean-Variance Efficient CF (MVECF, arXiv 2023)**: 기존 CF에 포트폴리오 이론의 평균-분산 정규화를 결합. 추천(예측 수익률)을 포트폴리오 가중치로 해석하고, 공분산이 높은 종목들의 동시 추천을 페널티. 2011~2020 실제 포트폴리오 데이터에서 순수 CF보다 투자 성과 개선.

**차용 가능한 아이디어**: "비슷한 패턴을 보인 종목들의 과거 결과"를 기반으로, 현재 종목의 예상 시나리오를 확률 분포로 제시. 이는 Item-based CF와 동일한 구조이다.

**단, 직접 적용의 한계**: 추천 시스템의 user-item 상호작용은 안정적(취향은 급변하지 않음)이지만, 주식 패턴의 성공률은 시장 regime에 따라 급변한다. Regime 조건부 필터링이 필수.

## D. 구현 로드맵

### Phase 1 (1~2개월): 기초 인프라 — "패턴 탐지 + 분류 + 차트북"

**목표**: 과거 데이터에서 패턴을 자동 탐지하고, 성공/실패로 분류된 차트북을 생성

1. **피봇 탐지 모듈**: `scipy.signal.find_peaks` + ZigZag 구현
2. **패턴 규칙 엔진**: VCP 우선 구현 (가장 명확한 수치 기준). 이후 CWH, HTF 순서
3. **Forward Return 계산**: 돌파일 기준 T+10 최고 수익률
4. **차트 이미지 생성**: `mplfinance` 자동화
5. **차트북 SQLite 스키마**: `patterns` 테이블 (종목코드, 날짜, 패턴유형, 품질점수, 결과, 이미지경로)
6. **기본 통계**: 패턴별 성공률, 시장상태별 성공률

**기술 스택**: Python, pandas, scipy, mplfinance, SQLite

### Phase 2 (2~3개월): 유사 패턴 검색 — "지금 이 차트와 비슷한 과거 사례"

**목표**: 현재 종목의 차트를 입력하면 유사한 과거 사례 K개를 반환

1. **피처 벡터 설계**: 규칙 기반 피처 10~20개
   - 수축 깊이 시퀀스, RS 순위, 거래량 dry-up 비율, 50일선 대비 위치, 베이스 기간, 시가총액 구간 등
2. **FAISS 인덱스 구축**: Phase 1의 모든 탐지 패턴을 벡터화 → IndexIVFFlat
3. **쿼리 인터페이스**: CLI 또는 간단한 Streamlit UI
4. **후속 성과 시각화**: Top-K 유사 사례의 T+10 수익률 분포 히스토그램

**참고 구현**: Gábor Vecsei의 Stock-Pattern-Analyzer (GitHub: gaborvecsei/Stocks-Pattern-Analyzer) — FastAPI + FAISS, 슬라이딩 윈도우 + min-max 정규화 + KDTree/FAISS 인덱스. REST API + Dash 프론트엔드. 이 코드를 참조하되, 피처를 단순 가격 윈도우에서 도메인 특화 피처 벡터로 교체.

### Phase 3 (3~4개월): 멀티스케일 + 고도화

**목표**: 일봉+주봉 결합, 시장 컨텍스트 통합, 임베딩 기반 고도화

1. **주봉 유사도 추가**: 일봉 Top-K를 주봉 유사도로 리랭킹
2. **Regime 필터**: 시장 상태별 조건부 검색 (강세장 패턴은 강세장 사례에서만 검색)
3. **TS2Vec 임베딩 실험**: 규칙 기반 피처 vs. 학습 임베딩 A/B 비교
4. **Relevance Feedback**: 사용자가 유사도 판단을 입력 → 피처 가중치 미세 조정
5. **대시보드 고도화**: 실시간 스크리닝 + 유사 사례 팝업

### Phase 4 (선택): CNN 이미지 기반 실험

1. **Recurrence Plot / GAF 변환**: 가격 시계열 → 2D 이미지
2. **사전학습 ResNet 피처 추출**: 이미지 → 2048차원 벡터 → FAISS
3. **A/B 비교**: 피처 벡터 검색 vs. 이미지 임베딩 검색의 Top-K 품질 비교

**단, 2025년 연구에서 시계열 직접 처리가 이미지 변환보다 우월하다는 결과가 있으므로, Phase 4는 실익이 확인된 경우에만 진행.**

## 관점 확장: 결론을 바꿀 수 있는 인접 질문

1. **생존자 편향**: 상장폐지/관리종목 제외 시 성공률이 과대추정될 수 있다. 2년치 DB에 상폐 종목이 포함되어 있는지 확인 필요.
2. **시장 Regime 의존성**: 2020~2022 한국 증시는 강세→급락→회복이라는 특수한 사이클. 이 기간에서 학습한 패턴 성공률이 향후 적용 가능한지 검증 필요. 최소 1개 강세/약세 사이클을 포함하는 데이터가 이상적.

## 문제 재정의

원래 질문은 "차트북 자동 생성 + 유사 패턴 검색"이지만, 조사 결과 더 적절한 핵심 질문은 **"패턴의 시각적 유사성이 아닌, 어떤 조건(패턴 형태 + 시장 상태 + 펀더멘털)의 조합이 돌파 성공을 예측하는가"**이다. 유사 패턴 검색은 이 질문에 답하기 위한 수단이지, 그 자체가 목적이 되어서는 안 된다. 검색 시스템의 가치는 "비슷하게 생긴 차트를 찾는 것"이 아니라 "비슷한 조건에서의 결과 분포를 알려주는 것"에 있다.
