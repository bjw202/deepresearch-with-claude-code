## 1. 도메인 적용성

### 1.1 인용 연구의 도메인 불일치 사례

**Researcher 1 (표현학습)**

- **TS2Vec/TF-C**: 범용 시계열 벤치마크(UCR/UEA)에서 검증된 방법론이다. 금융 시계열은 낮은 SNR, 비정상성, regime 전환이라는 근본적 차이가 있으며, 이 차이가 성능에 미치는 영향은 정량적으로 검증되지 않았다. "Yahoo Finance 포함 평가"라고 언급하나 이것이 패턴 분류 태스크인지, 예측 태스크인지 구분되지 않는다.
- **TNC (Temporal Neighborhood Coding)**: [인접 도메인: 의료 시계열]로 올바르게 태깅했으나, ICU 센서 데이터와 금융 시계열의 noise-to-signal ratio 차이가 "성능 차이 예상"이라는 모호한 표현으로 끝났다. 구체적으로 얼마나 다른지에 대한 근거가 없다.
- **DPP(2D CNN) 80.82% 정확도**: TX(대만 선물) 데이터셋 단일 종목 결과이다. 2,500개 한국 개별주에 일반화 가능성은 별도 논의 없이 "20-30% 우위"라는 수치가 핵심 주장으로 사용되었다. 선물 시장과 개별주 시장의 유동성, 변동성 구조가 근본적으로 다르다.

**Researcher 2 (예측모델)**

- **XGBoost R2 99%/95%/88% (Apple/Google/Tesla)**: 출처가 thesai.org이며, 단일 종목 주가 회귀 예측이다. R2 99%라는 수치는 주가의 자기상관(전일 종가 ~ 금일 종가) 때문에 달성 가능한 것으로, breakout 분류 태스크에서는 전혀 다른 수준의 난이도를 가진다. 이 수치가 GBT의 강점 근거로 사용되는 것은 오해의 소지가 크다.
- **LambdaMART Sharpe 3배 개선 (Poh et al. 2020)**: CRSP 미국 대형주 크로스섹션 모멘텀 전략 대비 수치이다. Researcher 2가 "breakout 셋업 랭킹과는 태스크가 다르다"고 명시한 점은 적절하나, 이 수치가 결론부의 로드맵에서 LambdaMART 도입 근거로 여전히 사용되고 있다.

**Researcher 3 (RL/구현)**

- **FinRL Contest 2024 결과**: 비트코인 트레이딩 환경에서의 결과임을 [인접 도메인: 암호화폐 트레이딩]으로 명시한 점은 적절하다.
- **"단순 피처 엔지니어링이 딥러닝을 이김" 한국 주식 2024 논문**: 예측 대상이 투자자 주문 흐름이지 패턴 돌파가 아님을 밝혔으나, 이 논문이 "한국 증시에서 딥러닝 불필요"라는 해석으로 확대될 위험이 있다.

### 1.2 권고

- 핵심 수치(80.82% 정확도, R2 99%, Sharpe 3배)가 대상 도메인(한국 증시 개별주 breakout 분류)과 일치하는 연구에서 나온 것이 **하나도 없다**. 이 사실을 종합 보고서에서 명시적으로 경고해야 한다.
- "XX 연구에서 YY% 달성"이라는 표현 시, 반드시 (1) 시장, (2) 자산 유형, (3) 태스크 유형을 함께 명시하는 컨벤션을 적용할 것.

### 2.2 권고

- **거래비용**: 세 보고서 중 Researcher 2만 한국 증시 거래비용 구조를 구체적으로 제시했다(왕복 0.4-1.0%). 그러나 인용된 성과 수치 중 거래비용이 반영된 것이 거의 없다. 종합 보고서에서 "거래비용 반영 전/후" 구분 표기를 도입해야 한다.
- **패턴 이벤트 수 추정(17,500건)**: 로드맵의 모든 단계가 이 수치에 의존한다. Phase 1 초기에 실제 규칙 기반 탐지 결과로 이 가정을 검증하는 단계를 추가해야 한다. 5,000건 미만이면 XGBoost조차 과적합 위험이 있다.
- **MTL 14% 개선 인용**: NeurIPS 논문의 정확한 서지정보를 보충하거나, 이 수치를 근거에서 제외해야 한다.

## 3. 서브에이전트 간 상충점

### 3.1 LSTM/시계열 딥러닝의 필요성

- **Researcher 1**: "TCN을 VCP 전체 구조(60-120일) 인식에 권장"하며, Phase 1에서 1D-CNN/TCN을 주모델로 제안.
- **Researcher 2**: "LSTM/TCN은 1단계 구현에서 **제외**하라. GBT에 적절한 시계열 feature를 추가하면 시계열 DL의 이점 대부분을 흡수할 수 있다."
- **Researcher 3**: Researcher 2에 동의하며, Phase 1에서 XGBoost, Phase 2에서야 Autoencoder/1D-CNN 검토.

**판단**: Researcher 1의 Phase 1 CNN/TCN 권고와 Researcher 2-3의 Phase 1 GBT 권고가 직접 상충한다. 데이터 규모(17,500건 추정)와 구현 복잡도를 고려하면, Researcher 2-3의 GBT 우선 접근이 더 현실적이다. 다만, Researcher 1이 지적한 "GBT는 시각적 패턴 형태를 직접 포착하지 못함"이라는 한계는 유효하므로, Phase 1에서 GBT의 성능 상한을 확인한 후 Phase 2에서 CNN/TCN 도입 여부를 판단하는 것이 합리적이다.

### 3.2 2D 이미지 접근법의 가치

- **Researcher 1**: "2D 차트 이미지를 보조 시그널로 활용" 권고. CNN Trend 분류 91-93% precision 인용.
- **Researcher 2**: "CNN 이미지 기반 접근은 ROI가 낮으므로 **제외**" 명시.
- **Researcher 3**: 직접 언급 없으나, 로드맵에 2D 이미지 관련 단계가 없음.

**판단**: 상충한다. Researcher 1이 인용한 91-93% precision은 "상승/하락 추세 분류"라는 단순 태스크에서의 결과이며, VCP/CWH 같은 복잡한 패턴 분류에 직접 적용 가능한 근거가 아니다. Researcher 2의 "ROI 낮음" 판단이 현 단계에서 더 적절하다. 2D 이미지는 Phase 3 이후 실험적 탐색 대상으로만 유지.

### 3.3 Foundation Model의 활용 가능성

- **Researcher 1**: "Chronos/TimesFM을 feature extractor로 사용" 대안 제시. Phase 3에 foundation model 임베딩 통합 포함.
- **Researcher 2**: Foundation model 언급 없음.
- **Researcher 3**: Foundation model 직접 언급 없으나, 데이터 규모 분석에서 Transformer를 "시기상조"로 판정.

**판단**: 상충은 아니나 불일치. Researcher 1이 zero-shot R2 음수라는 부정적 결과를 보고하면서도 "feature extractor" 활용을 권고하는 것은 논리적 긴장이 있다. 예측 성능이 음수인 모델의 임베딩이 유용한 표현을 제공한다는 근거가 없다. Foundation model 활용은 근거가 보강될 때까지 로드맵에서 우선순위를 낮춰야 한다.

### 3.4 Phase/로드맵 타임라인

- **Researcher 1**: Phase 1(2-4주) → Phase 2(4-8주) → Phase 3(선택)
- **Researcher 3**: Phase 1(2-3개월) → Phase 2(3-4개월) → Phase 3(4-6개월)

**판단**: Researcher 1의 2-4주 Phase 1은 데이터 파이프라인 구축, walk-forward 검증 인프라, 라벨링을 포함하면 비현실적이다. Researcher 3의 2-3개월이 더 현실적인 추정이다. 종합 보고서에서는 Researcher 3의 타임라인을 기준으로 하되, 각 Phase의 Go/No-Go 기준을 명확히 유지해야 한다.

### 3.5 SMOTE에 대한 입장

- **Researcher 2**: "SMOTE는 시계열 특성 때문에 **사용하지 않는 것을 권장**"
- **Researcher 3**: "Phase 1에서 교차 종목 학습 + **SMOTE(필요 시)**로 충분"

**판단**: 직접 상충한다. Researcher 2의 논거(시계열 구조 파괴 위험)가 더 설득력이 있다. 다만 Researcher 3이 언급한 것처럼 피처 기반 모델(XGBoost)에서는 시계열 구조가 이미 피처로 변환된 후이므로, "피처 공간에서의 SMOTE"와 "원본 시계열에서의 SMOTE"를 구분해야 한다. 피처 기반 SMOTE는 조건부 허용 가능하나, class weight 조정을 우선 시도해야 한다.

## 4. 누락 관점

### 4.1 비용 분석의 불완전성

- **컴퓨팅 비용**: 세 보고서 모두 모델 학습/추론의 컴퓨팅 비용을 정량적으로 다루지 않았다. "Python + SQLite"라는 사용자 인프라 제약 조건에서 TS2Vec pre-training, CPCV S=8 (70경로), LambdaMART 학습이 현실적으로 가능한지에 대한 검토가 없다. GPU 없이 CPU만으로 실행 가능한 범위를 명시해야 한다.
- **라벨링 비용**: Researcher 1이 "수동 라벨 200-500개" 필요성을 언급했으나, 이 라벨링에 드는 시간/노력을 추정하지 않았다. VCP 패턴 1개를 확인하는 데 차트 검토 2-5분으로 가정하면, 500개에 17-42시간이 필요하다. 이것이 현실적으로 가능한지 논의가 필요하다.
- **데이터 비용**: 2,500종목 x 2년 일봉 데이터의 획득/유지 비용이 언급되지 않았다. 투자자별 매매동향, 프로그램 매매 데이터 등 추가 데이터 소스의 비용과 접근성도 미언급.

### 4.2 실행 가능성

- **단일 개발자 제약**: 사용자가 1인 개발자인지 팀인지 불명확하나, 인프라(Python + SQLite)로 보아 소규모로 추정된다. Researcher 3의 Phase 1-3 총 9-13개월 로드맵은 1인 기준으로 현실적인가? 풀타임 개발자 기준인지, 파트타임(투자자가 부업으로 하는 경우) 기준인지에 따라 크게 달라진다.
- **운영 복잡도**: Phase 2 이후 멀티타겟 모델, 확률 보정, CPCV 검증, LambdaMART 랭킹이 모두 결합되면 시스템 복잡도가 급격히 증가한다. 이 복잡도를 유지/디버그할 수 있는 역량에 대한 논의가 없다.

### 4.3 실패 모드

다음 실패 시나리오가 충분히 다뤄지지 않았다:

1. **라벨 정의 실패**: 규칙 기반 VCP 탐지가 전문가 판단과 50% 이상 불일치하는 경우. 이 경우 모든 supervised 학습이 무의미해진다. → 초기에 전문가 라벨과의 교차 검증 단계가 필요하다.
2. **Regime 불일치**: 2020-2025 공매도 금지 기간 데이터로 학습한 모델이 2025년 이후 공매도 재개 환경에서 전혀 작동하지 않을 가능성. Researcher 3이 언급했으나 대응 전략이 "주의 필요" 수준에 그쳤다. → 공매도 금지 이전 데이터(2018-2020)를 별도 검증 세트로 활용하는 구체적 방안이 필요하다.
3. **Alpha Decay**: Researcher 2가 "전략 수명 6-18개월"을 언급했으나, 이에 대한 대응 전략이 없다. 모델 재학습 주기, 새로운 feature 발굴 프로세스, alpha 소멸 감지 메트릭에 대한 설계가 필요하다.
4. **과적합의 연쇄**: Phase 1에서 과적합된 모델이 Phase 2의 baseline이 되고, Phase 3의 RL이 이 위에 구축되면, 과적합이 누적적으로 증폭된다. 각 Phase에서 독립적으로 OOS 검증을 통과해야 한다는 원칙은 Researcher 3이 Go/No-Go로 제시했으나, 구체적 임계값(예: OOS precision > random + 몇 표준편차?)이 없다.

### 4.4 법적/규제 리스크

- 알고리즘 트레이딩의 한국 규제 환경에 대한 논의가 전무하다. 한국거래소의 알고리즘 매매 신고 의무, 금융위원회의 규제 동향 등이 실행 가능성에 영향을 줄 수 있다.

## 5. 확신도 교정

### 5.1 과도한 확신이 의심되는 주장

| 주장 | 표현된 확신도 | 교정 의견 |
|------|-------------|----------|
| "GBT가 tabular 금융 데이터에서 최고" (R2) | 강함 | **중-강**으로 하향. 근거 대부분이 주가 회귀/Kaggle 대회이며, breakout 이벤트 분류에서의 직접 검증이 없다. "GBT가 합리적 첫 선택"이라는 표현이 적절. |
| "CPCV가 Walk-Forward보다 우수" (R2) | 강함 | 유지. Arian et al. (2024)의 직접 비교 결과이며, 도메인 일치도도 높다. |
| "End-to-end RL 비권장" (R3) | 높음 | 유지. FinRL Contest 결과, 데이터 규모 제약, 유동성 문제 등 다각적 근거가 있다. |
| "RL의 최적 적용: Exit 최적화 1순위" (R3) | 높음 | **중간**으로 하향. "실행 최적화"가 RL의 성공 사례라는 근거(Kearns et al.)는 HFT 실행 최적화이며, 스윙 트레이딩 exit 최적화와는 시간 스케일이 다르다. Exit 최적화에서 RL이 규칙 기반 trailing stop을 이긴다는 직접적 증거는 제시되지 않았다. |
| "LightGBM이 1단계 최적" (R2) | 높음 | **중-강**으로 소폭 하향. XGBoost vs LightGBM의 성능 차이는 대부분 미미하며, "학습 속도 2-5배" 이점이 주된 근거다. 성능 기준이 아닌 실용성 기준의 선택임을 명시해야 한다. |
| "SSL pre-training은 해서 손해 볼 것이 없는 전략" (R1) | 중간 (표현은 강함) | **하향**. 추가 구현 복잡도, 학습 시간, 하이퍼파라미터 탐색 비용이 "손해"에 해당한다. 소규모 팀에서 SSL 구현에 2-4주를 투자한 후 성능 향상이 없으면, 이는 기회비용 손해이다. |
| "한국 주식 2024 논문: 피처 엔지니어링이 딥러닝을 이김" (R3) | 중간 | **낮음**으로 하향. 예측 대상(투자자 주문 흐름)이 다르므로 이 결론을 패턴 돌파 예측에 확장하는 것은 부적절하다. |

## 6. 문제 정의 검토

### 6.1 원래 질문의 범위

사용자의 원래 질문은 "눌림목/수축 패턴의 ML 기반 인식과 트레이딩"으로 추정된다. 세 Researcher 모두 이 질문의 범위 내에서 작업했으나, 각각 문제 재정의를 시도했다:

- R1: "아키텍처 → 라벨 품질 + 인식→수익성 연결"
- R2: "모델 아키텍처 → 검증 방법론의 엄격성"
- R3: "RL 적용 → RL 없이 어디까지 가능한가"

이 세 재정의는 상호 보완적이며, 핵심 메시지는 수렴한다: **모델 아키텍처/알고리즘 선택보다 라벨링, 검증, 비용 관리가 더 중요하다.**

### 6.2 문제 정의의 과도한 좁힘

원래 질문이 **"패턴 인식 ML"에 과도하게 고정**되어 있어, 다음 상위 질문이 충분히 다뤄지지 않았다:

1. **"패턴 인식이 수익의 원천인가?"**: VCP/CWH 같은 시각적 패턴이 통계적으로 유의미한 엣지를 제공한다는 학술적 증거가 세 보고서 어디에도 제시되지 않았다. Researcher 2의 SVM breakout 연구(Precision 0.18)가 이 문제를 간접적으로 보여준다. 패턴 자체가 아닌 "패턴 + 맥락(섹터, 시장 regime, 수급)"의 조합이 엣지의 원천일 가능성이 높으며, 이 경우 패턴 인식은 전체 시스템의 한 구성요소에 불과하다.

2. **"2,500종목 전체를 대상으로 해야 하는가?"**: 유동성, 시장 충격, 거래 비용을 고려하면 실제 매매 가능한 유니버스는 훨씬 좁다(시총 상위 500-800종목?). 전체 2,500종목에서 패턴을 탐지하되, 실행은 유동성 필터를 통과한 종목에서만 하는 전략이 현실적이나, 이 유니버스 축소가 모델 학습 데이터 규모에 미치는 영향이 분석되지 않았다.

3. **"스윙 트레이딩 자체의 수익성 전제가 유효한가?"**: Researcher 3이 "강세장에서 패시브 전략이 우위"라고 언급했으나, 더 근본적인 질문은 "개인 투자자가 거래비용과 세금을 지불하면서 스윙 트레이딩으로 장기적으로 시장을 이길 수 있는가?"이다. 이 전제가 검증되지 않은 상태에서 ML 시스템 구축에 9-13개월을 투자하는 것의 기대값을 먼저 추정해야 한다.

### 6.3 솔루션 고정 위험

사용자의 질문이 **"ML로 패턴을 인식하고 RL로 매매를 최적화"**라는 특정 솔루션에 고정되어 있다. 세 Researcher 모두 이 프레임 안에서 답변했다. 그러나:

- **대안 접근 1: 규칙 기반 스크리닝 + 수동 매매 보조**. ML 없이도 규칙 기반 VCP 탐지 → 수동 차트 확인 → 매매 결정이라는 워크플로우로 시작할 수 있다. 이 접근의 성과 baseline이 없는 상태에서 ML 도입의 한계 이득(marginal gain)을 추정할 수 없다.
- **대안 접근 2: Factor-based 접근**. 패턴 인식이 아닌, 팩터(모멘텀, 변동성, 수급) 기반 종목 선택이 더 학술적으로 검증된 접근이다. 패턴 인식을 "변동성 수축 팩터"로 재정의하면, 시각적 패턴에 의존하지 않고도 동일한 트레이딩 아이디어를 구현할 수 있다.

### 6.4 권고

종합 보고서에서 다음을 명시해야 한다:

1. **Phase 0 추가**: ML 도입 전에 규칙 기반 VCP 탐지 + 수동 매매의 성과 baseline을 먼저 확보. 이 baseline이 없으면 ML의 부가 가치를 측정할 수 없다.
2. **검증 가능한 전제 목록**: (a) 패턴이 통계적 엣지를 제공하는가, (b) 그 엣지가 거래비용을 초과하는가, (c) ML이 규칙 기반 대비 유의미한 개선을 주는가. 이 세 전제가 순차적으로 검증되어야 하며, 어느 하나라도 실패하면 후속 단계의 의미가 없다.
3. **투자 대비 수익(ROI) 프레임**: 9-13개월의 개발 시간을 금전적 가치로 환산하고, 이 투자를 정당화하려면 얼마의 추가 수익이 필요한지 역산. 소규모 운용 자금에서는 ML 시스템의 ROI가 음수일 가능성이 있다.

## 1. 도메인 적용성

### 1.1 Minervini/O'Neil 수치의 한국 증시 적용 문제

**Researcher 4 (피봇 포인트 정의)**

- 6대 패턴의 정량적 파라미터가 **전부 미국 시장 기반 경험칙**에서 유래한다. Minervini의 VCP 수축 비율(20~30% 감소), O'Neil의 CWH 컵 깊이(12~33%), HTF 선행 상승률(100%/8주) 등 모든 핵심 수치가 미국 NASDAQ/NYSE 성장주에서 관찰된 것이다. 한국 증시에서의 검증 데이터가 **단 하나도 인용되지 않았다**.
- **상한가 제도(+30%)와의 충돌**: 성공 기준 "+30% in 10일"이 한국 상한가 1회 도달 수준이라는 사실을 Researcher 5가 언급했지만, Researcher 4는 이 기준을 별도 검증 없이 그대로 채택했다. 미국 시장에는 상한가 제도가 없으므로, 이 기준의 원래 맥락과 한국 시장의 구조가 근본적으로 다르다.
- **거래세 영향 미반영**: 한국 증시의 증권거래세(매도 시 0.23%→0.15% 인하 중)와 위탁 수수료가 패턴별 손절선(-3~8%)의 실질 비용에 영향을 주나, 이 조정이 누락되었다.
- **개인투자자 비중**: 한국 코스닥의 개인투자자 비중(약 70~80%)은 미국 대비 훨씬 높다. Minervini의 "기관 매집" 논리에 기반한 거래량 드라이업 해석이 개인 중심 시장에서도 동일하게 작동하는지 검증이 필요하다.

**Researcher 5 (차트북 & 유사도)**

- J.P. Morgan AI Research의 반증(범용 클러스터링 = 예측력 없음)이 S&P 500 대상임을 적시한 것은 적절하다. 그러나 긍정적 증거로 인용된 Savin et al.(H&S, S&P 500), FHSU(캔들스틱, S&P 500), DPP(대만/닛케이) 역시 **모두 비한국 시장**이다. 한국 증시 직접 증거가 없다는 점은 기존 Critic(01~03)에서 지적한 도메인 불일치 문제가 여전히 해소되지 않았음을 보여준다.

**Researcher 6 (전처리)**

- 정규화 방법론 자체는 도메인 비의존적이므로 적용성 문제가 상대적으로 적다. 다만, RVOL lookback=50이라는 기본값이 한국 증시의 거래 패턴(월말/분기말 프로그램 매매 급증, 배당락일 거래량 왜곡)에 적합한지 논의가 없다.

**Researcher 7 (시스템 리프레이밍)**

- 기존 01~03의 도메인 불일치를 재평가하면서 "+30% in 10일 기준의 한국 증시 현실성"을 Phase 0 필수 검증 사항으로 격상한 것은 **매우 적절한 판단**이다. 이것이 보강분 4개 보고서 중 도메인 적용성을 가장 잘 인식한 부분이다.

### 1.2 권고

| 양성 비율 ~5-8% (+30% 기준) | Researcher 7 자체 추정 | "+10% 달성 비율이 ~20%라면, +30% 달성 비율은 ~5-8%"라는 추정의 근거가 "~20%"라는 전제에 의존하며, 이 전제 자체의 출처가 없다. 기존 Critic이 지적한 "패턴 이벤트 17,500건" 추정과 결합하면 양성 샘플 ~900-1,400건이 되는데, 이 계산의 전체 체인이 미검증 가정에 기반한다 |
| Z-Score 윈도우 60일 | Researcher 6 일반 기준 | "일반적"이라고 명시했으나, Springer 논문 인용이 구체적이지 않다(정확한 저자/연도/DOI 없음). 60일이 VCP 베이스 기간(3~12주)과 겹치면 패턴 자체가 통계를 오염시키는 문제를 다뤘으나, 이에 대한 정량적 검증(시뮬레이션 등)이 없다 |

### 2.2 권고

- Researcher 4의 핵심 기여(6대 패턴 정량화)가 보고서의 가장 높은 실용적 가치를 가지지만, 모든 임계값이 **경험적 관찰**이지 통계적 최적값이 아님을 종합 보고서에서 강조해야 한다. "이 파라미터로 시작하되, Phase 0에서 한국 데이터로 분포를 확인한 후 재설정"이라는 프로세스를 필수화해야 한다.
- Researcher 7의 양성 비율 추정(5-8%)은 핵심 로드맵 결정에 영향을 미치므로, Phase 0의 가장 첫 번째 산출물로 실제 빈도를 측정해야 한다. 이 수치가 3% 미만이면 +30% 기준 자체를 재고해야 한다.

## 3. 서브에이전트 간 상충점

### 3.1 04(패턴 정의)와 05(차트북)의 패턴 분류 기준 일관성

- **일관적**: Researcher 5의 패턴 구별 피처 테이블(05 A4절)이 Researcher 4의 정량적 파라미터와 정확히 대응한다. VCP 수축 횟수 >= 3, CWH 컵 깊이 12~33%, HTF 직전 8주 +100% 등.
- **부분 불일치**: Researcher 5의 HTF 기준이 "직전 8주 내 +100% 이상 상승"(O'Neil 원래 정의)인 반면, Researcher 4는 쿨라메기 완화 기준(30~100%)도 병기했다. Researcher 7의 시스템 아키텍처에서는 HTF를 "+25% in 8주 이내"로 더욱 완화했다(07 C.2 모듈1). **세 보고서의 HTF 기준이 각각 다르다:** 100%(R5) vs 30~100%(R4) vs 25%(R7).
- **권고**: HTF 진입 기준을 통일해야 한다. 쿨라메기 기준(30%+)을 채택하고, O'Neil 원래 정의(100%+)는 "클래식 HTF" 하위 분류로 유지하는 것이 현실적이다.

### 3.2 05(유사도 매칭)와 06(전처리)의 호환성

- **호환적**: Researcher 5가 권장한 "규칙 기반 피처 벡터 → 코사인 유사도"와 Researcher 6의 "규칙 기반 Feature 추출" 접근이 구조적으로 일치한다.
- **잠재적 충돌**: Researcher 5는 유사도 검색에 Min-Max 정규화된 시계열을 DTW로 비교하는 방법을 권장하면서, 정규화 세부사항을 명시하지 않았다. Researcher 6은 Min-Max의 이상치 왜곡 문제를 상세히 다루며 "percentile clipping 선행" 필수를 권고했다. 05의 DTW 파이프라인에 06의 clipping이 반영되지 않으면 왜곡된 유사도 계산이 발생할 수 있다.
- **리샘플링 길이 불일치**: Researcher 6은 고정 길이 64를 권장했으나, Researcher 5의 DTW는 가변 길이를 전제한다. 두 방법이 병행될 때의 파이프라인 분기가 정의되어 있지 않다.
- **권고**: 종합 보고서에서 "유사도 검색 전처리는 06의 `pattern_safe_normalize` → clipping → 누적 수익률 변환을 선행한다"는 규칙을 명시해야 한다.

### 3.3 07(리프레이밍)과 기존 01~03의 정합성

- **적절한 재평가**: Researcher 7이 01~03 각 보고서의 항목별 유효도(핵심 유효/유효/축소/제거)를 명시적으로 평가한 것은 우수하다.
- **RL 제거 판단의 정당성**: 03 보고서의 RL 관련 내용 대부분을 "제거"로 판정한 것은 "의사결정 보조"라는 새 목적에 정확히 부합한다. 기존 Critic이 지적한 "Exit 최적화에서 RL이 규칙 기반을 이긴다는 직접 증거 부재" 문제도 제거로 해소되었다.
- **2D 이미지 역할 전환의 정합성**: 기존 Critic은 "2D 이미지 CNN은 ROI 낮음"으로 판단했는데, Researcher 7이 이를 "모델 입력이 아닌 시각화/검색 UI"로 역할 전환한 것은 기존 상충점의 **창의적 해결**이다.
- **Phase 타임라인 조정**: 기존 Critic은 Researcher 1의 2-4주 Phase 1을 "비현실적"으로 판정하고 Researcher 3의 2-3개월을 지지했다. Researcher 7의 새 로드맵은 Phase 0(2-3주) + Phase 1(6-8주) + Phase 2(8-10주) = 약 4-5개월로, 기존 Critic의 권고와 대체로 일관된다.

### 3.4 04와 07의 성공 기준 정의 차이

- Researcher 4: "+30% in 10일"을 "시원한 돌파(성공)"로 정의, "+20% in 20일"을 "보통 성공"으로 정의.
- Researcher 7: "+30% in 10일" 단일 기준의 양성 비율 급감 문제를 지적하며, **계층적 확률(+10%, +20%, +30% 모두 예측)** 을 권고.
- **판단**: 두 보고서는 상충이 아니라 보완 관계이다. R4의 성공/실패 기준은 라벨링 정의, R7의 계층적 확률은 모델 출력 설계이다. 다만, R4의 라벨링에서 "보통 성공"(+20%/20일)과 "성공"(+30%/10일)이 모델 학습 시 **어느 라벨로 취급되는지**가 명확하지 않다. 이진 분류라면 "보통 성공"은 양성인가 음성인가? 이 정의가 클래스 불균형의 심각도를 좌우한다.
- **권고**: 라벨링을 이진이 아닌 다단계(성공/보통/실패/무의미)로 유지하되, 모델 학습 시에는 R7이 권고한 연속 수익률 회귀 + 임계값 변환을 채택하는 것이 클래스 불균형 문제를 우회하는 가장 합리적인 방법이다.

### 3.5 05의 "유사도 예측력" 판단과 기존 01~03과의 관계

- Researcher 5가 J.P. Morgan 연구를 "가장 강력한 반증"으로 인용한 것은 기존 Researcher 1의 "GBT가 시각적 패턴을 포착 못함" 지적과 구조적으로 연결된다. 범용 클러스터링이 예측력이 없다면, 도메인 특화 패턴 정의(R4)의 가치가 더 커진다.
- 이 논리 체인은 일관적이며, "규칙 기반 피처 → GBT"가 "데이터 주도 임베딩 → 클러스터링"보다 현 단계에서 우월하다는 기존 Critic의 판단을 강화한다.

## 4. 누락 관점

### 4.1 +30%/10일 성공 기준의 양성 비율과 클래스 불균형

- Researcher 7이 양성 비율 ~5-8% 추정과 양성 샘플 ~900-1,400건 추정을 제시한 것은 기존 Critic의 "패턴 이벤트 수 검증" 권고에 대한 응답이다. 그러나:
  - **추정 체인의 취약성**: 17,500건(R3 추정) x 5-8% = 900-1,400건이라는 계산에서 두 변수 모두 미검증이다. 17,500건이 5,000건이고 양성 비율이 3%면 양성 샘플은 150건에 불과하며, 이 경우 LightGBM조차 의미 있는 학습이 불가능하다.
  - **클래스 불균형 대응의 구체성 부족**: R7이 "계층적 확률" 접근을 제안했으나, 연속 수익률 회귀 + 임계값 변환의 구체적 구현 방법과 검증 방법이 없다. 기존 Critic이 지적한 SMOTE 상충(R2 vs R3) 문제도 이 보강분에서 재논의되지 않았다.
- **권고**: Phase 0에서 "+30% in 10일" 빈도 측정 시, 동시에 "+10%", "+15%", "+20%" 빈도도 측정하여 **어떤 임계값에서 양성 샘플이 최소 500건 이상 확보되는지** 확인해야 한다.

### 4.2 1인 개발자 환경의 구현 현실성

- Researcher 7의 새 로드맵(Phase 0~3, 총 6-9개월)이 기존 9-13개월에서 단축된 것은 긍정적이다. 그러나:
  - **Phase 1 (6-8주) 범위**: 6개 패턴 탐지 규칙 코드화 + setup_events/forward_labels 파이프라인 + 차트북 자동 생성 + 통계 대시보드를 6-8주 내에 1인이 완수하는 것은 공격적이다. VCP 단일 패턴만 해도 수축 감지, Higher Low 확인, 거래량 드라이업 판정 등 복잡한 로직이 필요하다.
  - **Researcher 6의 전처리 파이프라인 복잡도**: `PatternPreprocessor` 클래스가 clipping → 로그수익률 → Min-Max → RVOL → 리샘플링 → 컨텍스트 벡터 → 규칙 Feature를 모두 포함하며, 이 파이프라인의 구현 + 디버깅 + 검증에 소요되는 시간이 로드맵에 명시적으로 반영되어 있지 않다.
- **권고**: Phase 1을 "VCP만 우선 구현"하는 Phase 1a(3-4주)와 "나머지 패턴 추가"하는 Phase 1b(4-5주)로 분할하는 것이 현실적이다. VCP 단독으로도 차트북과 통계의 가치를 확인할 수 있다.

### 4.3 실패 모드 보강

기존 Critic이 지적한 4가지 실패 모드(라벨 정의 실패, Regime 불일치, Alpha Decay, 과적합 연쇄) 중:

- **라벨 정의 실패**: Researcher 4가 규칙 기반 패턴 정의를 상세히 제공했으나, "규칙 기반 탐지와 전문가 판단의 일치율"은 Phase 1 Go/No-Go(70% 동의)에서 다뤄졌다. 적절하다.
- **Regime 불일치**: Researcher 7이 시장 컨텍스트 모듈(Module 4)로 대응했다. 그러나 기존 Critic이 요구한 "공매도 가능 기간(2018-2020) 데이터 별도 검증"은 여전히 구체적 구현 방안이 없다.
- **Alpha Decay**: 4개 보고서 모두에서 충분히 다뤄지지 않았다. 모델 재학습 주기, alpha 소멸 감지 메트릭에 대한 설계가 여전히 누락이다.
- **유사도 검색의 과신 실패**: Researcher 5가 "독립 사례 수 vs effective sample size" 문제를 관점 확장에서 언급했으나, 구체적 대응(예: regime별 분산 가중, bootstrap confidence interval)이 없다. "유사 사례 10개 중 8개 성공"이 같은 강세장 기간에 집중된 경우, 이 통계는 무의미하다.

### 4.4 법적/규제 리스크

기존 Critic이 지적한 "알고리즘 트레이딩 한국 규제" 문제는 Researcher 7이 "자동매매가 아닌 의사결정 보조"로 리프레이밍함으로써 **대부분 해소**되었다. 의사결정 보조 도구는 알고리즘 매매 신고 의무 대상이 아니다. 적절한 해결이다.

| "컨텍스트 벡터(사전 상승률, 베이스 카운트, 시장 regime)가 패턴 형태만큼 중요" (R6) | 중간 (부차적 feature로 취급) | **상향 필요**. 쿨라메기와 Minervini 모두 패턴 자체보다 맥락(시장 상태, 추세 단계, 베이스 순서)이 수익에 더 큰 영향을 미친다고 명시적으로 말한다(R4 섹션 9.4에서도 인용). 컨텍스트가 feature 목록의 일부가 아니라, 시스템의 **핵심 차별화 요소**임을 강조해야 한다 |
| "Phase 0에서 +30% 기준 검증 필수" (R7) | 높음 | **최고로 상향**. 이 검증이 실패하면 전체 시스템의 라벨링/평가 체계가 바뀌므로, Phase 0 이전에 아무 작업도 시작하지 않아야 한다. 현재 로드맵에서 Phase 0가 "선택적"으로 읽힐 여지가 있다 |

## 6. 문제 정의 검토

### 6.1 "자동매매가 아닌 의사결정 보조"의 일관성

- **Researcher 7이 가장 잘 반영**: 시스템 전체를 "의사결정 보조"로 재구성하면서, RL 제거, Position Sizing 제거, Exit 최적화 제거, LambdaMART 축소를 일관되게 수행했다.
- **Researcher 4는 부분 불일치**: 패턴 정의에서 "진입 신호", "손절선"을 정의하고 있어 자동매매적 프레이밍이 남아 있다. 의사결정 보조 관점에서는 "이 지점이 피봇이다"와 "피봇 대비 현재가의 위치"를 정보로 제공하는 것이지, 진입/손절 신호를 자동 생성하는 것이 아니다. 다만, 이 정보가 **사용자의 수동 판단 재료**로 사용된다면 용어의 차이일 뿐 실질적 문제는 아니다.
- **Researcher 5는 일관적**: 차트북과 유사도 매칭 모두 "정보 제공"에 초점. "확률적 참고 자료로 사용해야 하며, 자동매매 신호로 사용하면 안 됨"을 명시.
- **Researcher 6은 도구적**: 전처리 방법론은 목적 비의존적이므로 일관성 문제 없음.

### 6.2 차트북 구축과 유사 패턴 매칭의 충분성

- 사용자의 핵심 산출물 요구("차트북 자동 생성 + 유사 패턴 매칭")가 Researcher 5와 7에서 **충분히 다뤄졌다**.
- Researcher 5: 차트북 자동 분류 파이프라인, 유사도 측정 방법 비교, 구현 로드맵(Phase 1~5)
- Researcher 7: 차트북을 Phase 1 핵심 산출물로 격상, 유사 매칭을 Phase 2 핵심으로 배치
- **누락**: 차트북의 **사용자 인터페이스 설계**가 구체적이지 않다. HTML 갤러리(R5)인지 Streamlit 앱(R7)인지, 검색/필터 기능은 어떻게 제공되는지가 실제 사용성을 좌우하지만 상세 설계가 없다. 1인 개발자에게 UI 구현은 상당한 부담이므로, 최소 UI 요건을 정의해야 한다.

### 6.3 문제 재정의의 가치

각 Researcher의 문제 재정의가 유용했는지 평가:

| Researcher | 원래 질문 | 재정의 | 평가 |
|------------|----------|--------|------|

## 종합 소견

### 강점

4개 Researcher 산출물은 다음을 잘 수행했다:

- **수치 투명성**: R4의 모든 파라미터에 출처와 "틀릴 수 있는 조건"이 명시됨. 기존 01~03 대비 투명성이 향상됨
- **반증 탐색**: R5의 J.P. Morgan 반증 인용, R4의 생존자 편향 지적이 적절
- **실행 연결**: R6의 pseudocode 수준 구현 가이드, R7의 모듈별 입출력 정의가 실행 가능성을 높임
- **기존 Critic 반영**: R7이 기존 Critic의 "Phase 0 추가", "규칙 기반 baseline 우선", "타임라인 현실화" 권고를 대부분 반영함
- **문제 재정의 수렴**: R4, R5, R7 모두 "모델 정교함 < 문제 정의 + 검증 엄격성"이라는 메시지로 수렴

### 결론

보강분 4개 보고서는 기존 01~03의 기술적 분석을 **실행 가능한 시스템 설계**로 발전시켰다. 특히 Researcher 7의 시스템 리프레이밍이 기존 Critic의 핵심 권고(Phase 0 추가, 규칙 기반 우선, RL 제거)를 대부분 수용하여 전체 프로젝트의 방향성이 크게 개선되었다.

남은 핵심 과제는 (1) +30% 기준의 한국 데이터 실증, (2) 미국 시장 기반 파라미터의 한국 시장 교정, (3) HTF 기준 통일이다. 이 세 가지는 모두 Phase 0에서 데이터로 답을 얻을 수 있으므로, **Phase 0의 실행이 전체 프로젝트의 성패를 결정하는 관문**이다.
