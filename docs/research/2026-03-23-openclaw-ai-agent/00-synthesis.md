# OpenClaw: "내 컴퓨터에서 돌아가는 AI 비서"는 쓸모가 있는가?

> 종합 보고서 | 2026-03-23

---

## 요약: 30초 안에 읽기

**OpenClaw**은 내 컴퓨터에 설치해서 파일 관리, 웹 검색, 이메일 처리, 메신저 응답까지 "알아서 해주는" 오픈소스 AI 에이전트다. GitHub 역사상 가장 빠르게 성장한 프로젝트(60일 만에 318K+ 스타)이며, 창시자는 OpenAI에 합류했다.

**쓸모가 있는가?** 개발자이고 반복 업무가 많으면 **시도할 가치가 있다**. 하지만 보안 사고가 실제로 다수 발생했고, 창시자 본인도 "비개발자에게 아직 추천하지 않는다"고 말했다. **강력한 도구이지만, 아직 거칠다.**

---

## 1. OpenClaw가 뭔가? (쉽게 설명)

### 한 줄 정의

> 내 컴퓨터에서 직접 돌아가면서, 채팅으로 시키면 실제로 "일"을 해주는 AI 비서.

일반 챗봇(ChatGPT, Claude)은 **대화만** 한다. OpenClaw는 대화를 넘어서:

- 파일을 만들고, 검색하고, 정리한다
- 웹 브라우저를 열어서 정보를 찾아온다
- 이메일을 분류하고 답장 초안을 작성한다
- WhatsApp, Telegram, Slack 등 메신저에서 직접 AI와 대화한다
- **알아서 주기적으로 확인**한다 (새 이메일, 주가 변동 등)

### 탄생 이야기

| 시기 | 이름 | 무슨 일이 있었나 |
|------|------|------------------|
| 2025.11 | Clawdbot | 오스트리아 개발자 Peter Steinberger가 처음 공개 |
| 2026.01 초 | Moltbot | Anthropic 상표 문제로 이름 변경 |
| 2026.01 중 | **OpenClaw** | "Moltbot은 발음이 어색하다"며 최종 변경 |
| 2026.02 | — | 창시자가 **OpenAI에 합류**. 프로젝트는 독립 오픈소스 재단으로 이관 |

Steinberger는 iPad PDF 도구(PSPDFKit)를 만들어 10억 대 기기에 탑재시킨 경력자다. AI의 "패러다임 전환"을 직감하고 44개 AI 프로젝트 경험 끝에 OpenClaw을 만들었다.

## 2. 어떻게 작동하는가?

비유로 설명하면, **"24시간 상주하는 개인 비서에게 사무실, 전화, 컴퓨터를 제공하는 것"**이다.

### 구조를 4단계로

```
① 내 컴퓨터에 "비서 사무실" 설치 (Gateway)
② AI "두뇌" 연결 (Claude, GPT-4o 등 선택)
③ 메신저/이메일 등 "소통 채널" 연결
④ 비서가 스스로 관찰 → 계획 → 실행 → 보고 (에이전트 루프)
```

**핵심**: 모든 설정이 텍스트 파일(Markdown)로 되어 있다. 메모장으로 열어서 "친근하게 말해", "매주 월요일 보고서 만들어"처럼 직접 수정할 수 있다.

| 설정 파일 | 역할 | 비유 |
|-----------|------|------|
| AGENTS.md | 행동 규칙 | "이렇게 일해라" |
| SOUL.md | 성격/톤 | "친근하게 말해라" |
| MEMORY.md | 장기 기억 | "사장님은 커피를 좋아한다" |
| HEARTBEAT.md | 주기적 체크 지침 | "30분마다 이메일 확인해라" |

### Heartbeat — 가장 차별적인 기능

다른 AI 도구는 **시키면 한다**. OpenClaw은 **알아서 체크한다**.

30분마다(조정 가능) 스스로 깨어나서:
1. 할 일이 있는지 확인
2. 필요하면 실행
3. 결과를 사용자에게 알림

예: 아침 브리핑 자동 생성, 새 이메일 도착 알림, 주가 급변 경고

## 3. 실제로 사람들이 어떻게 쓰고 있나?

### 검증된 사용사례

| 분야 | 실사용 예시 | 세팅 난이도 |
|------|-----------|------------|
| **DevOps** | WhatsApp에서 서버 재시작, CI 빌드 실패 알림 + 원인 분석 | 중~고 |
| **이메일 관리** | 수백 개 이메일 분류, 핵심 추출, 답장 초안 | 중 |
| **마케팅** | 소셜 미디어 포스트 생성, 리드 팔로업, 경쟁사 모니터링 | 중 |
| **콘텐츠 제작** | 플랫폼별 맞춤 포스트, 번역, 스케줄링 | 중 |
| **고객 지원** | 메신저에서 24/7 자동 응답, 리드 정보 수집 | 중~고 |
| **일상** | 그룹채팅 장보기 목록 수집, 회의록 요약 | 낮~중 |

### 하지만 — 실패 사례도 있다

Reddit r/openclaw (207K 멤버)에서:
> "설치는 문제없었지만, **매일 안정적으로 쓸 수 있는 사용사례를 찾지 못해** 셧다운했다"

**핵심 판단**: "이런 일이 가능하다"와 "이것을 매일 안정적으로 쓰고 있다" 사이에는 큰 차이가 있다. 기술적 숙련도가 높고 명확한 자동화 니즈가 있는 사용자에게는 실용적이지만, 그렇지 않으면 세팅 비용이 효과를 초과할 수 있다.

## 4. 비슷한 도구들과 뭐가 다른가?

| | OpenClaw | Open Interpreter | Claude Code | Manus AI |
|--|----------|------------------|-------------|----------|
| **핵심 특징** | 메신저 통합 + 24/7 자율 운영 | 터미널에서 코드 실행 | 코딩 전문 에이전트 | 클라우드 자율 에이전트 |
| **메신저 연결** | WhatsApp, Telegram 등 10+ | 없음 | 없음 | 웹만 |
| **알아서 작업** | Heartbeat로 주기적 자율 실행 | 사용자 명령만 | 사용자 명령만 | 가능 |
| **설치** | 내 PC (로컬) | 내 PC | 내 PC | 클라우드 (설치 불필요) |
| **강점** | 생활 전반 자동화 | 가볍고 빠른 시작 | 코딩에 최적화 | 비개발자 친화 |
| **약점** | 보안 우려, 세팅 복잡 | 기능 제한적 | 코딩 외 용도 약함 | 비공개, 비용 |

### OpenClaw만의 차별점
1. **메시징 네이티브**: 기존 채팅 앱에서 바로 AI와 대화
2. **24/7 자율성**: 시키지 않아도 알아서 확인하고 실행
3. **스킬 마켓**: 162~200+ 커뮤니티 스킬로 기능 확장
4. **자기 확장**: 필요한 기능을 스스로 코딩하여 추가

### OpenClaw이 약한 곳
- 코딩 전문 도구(Claude Code, Cursor)에 비해 개발 작업 약함
- 화면을 "보고" 조작하는 GUI 자동화는 Anthropic Computer Use가 강함
- 기업용 워크플로우(감사 추적, 규정 준수)에는 부적합

[출처: DataCamp, Heyuan110, TechCrunch 2026/02/16]

### 전문가와 규제 당국의 반응

- **네덜란드 DPA**: OpenClaw 사용 자제 권고
- **Cisco**: "사이버보안 재앙"
- **Gartner**: "에이전틱 AI의 위험한 예고편"
- **Simon Willison** (보안 연구자): "코딩 에이전트 보안에서 챌린저 우주왕복선 사고에 가장 가까운 후보"
- **Peter Steinberger** (OpenClaw 창시자): **"현재 비기술적 사용자에게 추천할 수 없다"**

### 왜 이렇게 위험한가?

일반 앱은 할 수 있는 일이 제한되어 있다. OpenClaw는 **셸 명령 실행 + 파일 접근 + 브라우저 제어**를 모두 갖고 있어, 뚫리면 컴퓨터 전체가 위험해진다. 특히 **프롬프트 인젝션**(악성 텍스트를 AI가 읽게 만들어 조종하는 공격)은 AI 업계 전체가 아직 풀지 못한 근본 문제다.

## 6. 비용은 얼마나 드나?

OpenClaw 자체는 **무료**(MIT 오픈소스)다. 하지만 AI 모델 API 비용은 사용자가 부담한다.

| 사용 패턴 | 예상 월 비용 | 설명 |
|-----------|-------------|------|
| **경량 사용** (하루 10회 정도 명령) | $5~15 | 가끔 질문하고, 간단한 작업 의뢰 |
| **적극 사용** (Heartbeat + 다채널) | $50~200+ | 메신저 연결, 주기적 자동 체크, 이메일 관리 |
| **무제한 자율운영** | $500+ | 24/7 풀가동, 복수 채널, 복잡한 작업 |

> 참고: Steinberger 본인은 개발 과정에서 **월 $10,000 서버 비용**을 감당했다. 비용 최적화(로컬 모델 사용, Heartbeat 간격 늘리기 등)를 하지 않으면 비용이 빠르게 불어날 수 있다.

## 7. "그래서, 써볼까 말까?"

### 판단 기준 표

| 당신의 상황 | 추천 | 이유 |
|------------|------|------|
| 개발자 + 반복 업무 많음 (DevOps, 이메일 등) | **시도 가치 있음** | 30~90분 세팅으로 일상 시간 절약 가능 |
| 개인 사이드 프로젝트 자동화 | **흥미로운 실험** | 격리 환경(Docker)에서 최소 권한으로 시도 |
| 비개발자 + 당장 쓰고 싶음 | **아직 이르다** | 창시자 본인이 비추천. GUI 부족 |
| 민감한 데이터 다루는 업무 | **강력 비추천** | 실제 보안 사고 다수. 규제 당국 경고 |
| 기업 RPA 대체 목적 | **비추천** | 감사 추적, 규정 준수 기능 부재 |

### 안전하게 시작하려면 (최소 조건)

1. **격리된 환경**에서 실행 (Docker 컨테이너 또는 별도 VM)
2. **전용 API 키** 사용 (주 계정 절대 연결 금지)
3. 스킬 설치 전 **소스코드 전체 검토**
4. 네트워크 접근을 **필요한 곳만** 허용
5. **최신 버전 유지** (보안 패치 빈번)

### 비개발자를 위한 현실적 대안

OpenClaw이 아직 이르다면:
- **Claude / ChatGPT**: 대화형 AI로 충분한 경우가 많다
- **Apple Shortcuts / Google Home**: 비개발자도 쉽게 쓸 수 있는 자동화
- **Zapier / Make**: 앱 간 연결 자동화 (코딩 불필요)
- **Manus AI**: 클라우드 기반 자율 에이전트 (설치 불필요, 비용 있음)

## 8. 왜 지금 이렇게 핫한가?

### 기술적 배경
- AI 모델이 "대화"를 넘어 "실행"을 할 수 있는 수준에 도달
- API 비용이 지속적으로 하락하여 개인 사용이 경제적으로 가능
- MCP(Model Context Protocol) 같은 에이전트 표준이 등장

### 사회적 배경
- OpenAI, Google, Microsoft 모두 2026년을 **"에이전트의 해"**로 선언
- 영국 직장인 71%가 이미 비승인 AI 도구를 업무에 사용 중 ("Shadow AI")
- 원격 근무 확산으로 자동화할 수 있는 반복 작업이 가시화

### OpenClaw 특유의 타이밍
- 오픈소스 + GitHub 바이럴 효과 (스타 경쟁)
- 창시자의 OpenAI 합류 → "이것이 미래 방향"이라는 시그널
- Andrej Karpathy(AI 업계 거물)의 "SF 수준의 이륙"이라는 지지
- AI 에이전트끼리 소셜 네트워크를 형성하는 "Moltbook" 사건 → 미디어 관심 폭발

## 9. 미래: 어디까지 갈 수 있을까?

### 낙관적 시나리오
AI 에이전트가 "원격 근무자가 할 수 있는 모든 일"을 수행. 여러 에이전트가 팀처럼 협업하는 풀프로세스 자동화.

### 현실적 시나리오
- 복잡한 추론과 예외 상황에서 여전히 부족 (IBM 벤치마크)
- **프롬프트 인젝션**이 해결되지 않는 한 "완전 자율"은 불가능
- 기업용 RPA와 AI 에이전트가 **수렴** 중 — UiPath도 에이전틱 기능을 통합하고 있다

### 비개발자는 언제쯤 쓸 수 있나?
기업용 도구(UiPath Autopilot 등)는 1~2년 내 노코드 인터페이스를 확보할 수 있다. 하지만 OpenClaw 자체는 커뮤니티 주도 개발이므로 GUI 개선 시기를 예측하기 어렵다. **보안 문제의 산업 전반적 해결이 선행되어야** 비개발자에게 안심하고 추천할 수 있다.

### 숨은 위협: OS가 직접 한다면?
Apple Intelligence, Google Gemini 내장 등 **OS 자체에 에이전트 기능이 통합**되면, OpenClaw 같은 서드파티 도구의 존재 이유가 약해질 수 있다. 이것이 OpenClaw의 장기 생존을 결정하는 가장 큰 변수다.

## 11. 예상 밖 핵심 발견

1. **프롬프트 인젝션은 업계 전체의 미해결 과제**: OpenClaw만의 문제가 아니라, 자연어로 작동하는 AI 에이전트의 구조적 한계다. 이것이 해결되지 않으면 "완전 자율 에이전트"의 천장이 된다.

2. **"잘 만든 래퍼" 논쟁**: 일부 AI 전문가는 OpenClaw이 "혁신"이 아니라 기존 LLM 위에 잘 만든 통합 레이어에 불과하다고 평가한다 (TechCrunch). 즉, 핵심 가치는 OpenClaw 자체가 아니라 뒤에 있는 AI 모델에 있다는 시각.

3. **조직심리학의 위임 이론과 구조적 유사성**: AI 에이전트에게 권한을 주는 과정은 신입 직원에게 업무를 위임하는 과정과 같다. "작은 것부터 시작해 성공을 확인한 후 범위를 넓히는" 점진적 신뢰 확장이 AI 에이전트 도입에도 가장 안전한 전략이다.

## 12. 후속 탐색 질문

이번 리서치에서 답하지 않았지만, 다음에 조사할 가치가 있는 질문:

1. **"프롬프트 인젝션은 해결 가능한 문제인가, 아니면 자연어 AI의 구조적 한계인가?"** — 이 답이 AI 에이전트 산업 전체의 천장을 결정한다.

2. **"Apple Intelligence / Google Gemini 내장 에이전트가 OpenClaw 같은 서드파티를 대체할 시점은?"** — OS 레벨 통합이 서드파티 에이전트의 가장 큰 위협이다.

3. **"개인용 AI 에이전트의 적정 권한 범위는 어디까지인가?"** — 이것은 기술 문제가 아니라 신뢰 설계의 문제다. 보안/편의의 균형점을 찾는 프레임워크가 필요하다.
