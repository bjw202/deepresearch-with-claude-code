## 사용자 요청 요약

CadQuery, LLM, 제조 온톨로지를 결합하여 다음을 자동화하는 통합 시스템의 실현 가능성을 리서치해 달라는 요청:

1. **기어 모델링 자동화** — CadQuery를 사용한 파라메트릭 기어 3D 모델 생성
2. **설계 지원** — LLM 기반 설계 파라미터 추론 및 추천
3. **공정 구성** — 온톨로지 기반 제조 공정 자동 구성 (CAPP)
4. **공구 선정** — 가공 조건에 따른 공구 자동 선정
5. **공정 최적화** — 비용, 시간, 품질 기준의 공정 최적화

---

## 문제 정의

> "Python 기반 파라메트릭 CAD(CadQuery)와 LLM의 자연어 이해 능력, 그리고 제조 도메인 온톨로지를 결합하면, 기어 설계부터 공정 계획까지의 엔드투엔드 자동화가 현재 기술 수준에서 실현 가능한가?"

### 세부 질문

- CadQuery로 산업 수준의 기어 모델링이 가능한가? (인벌류트 기어, 헬리컬, 베벨 등)
- 제조 공정 지식을 온톨로지로 체계화한 선행 연구 및 표준이 있는가?
- LLM이 CAD/CAM 워크플로우에 통합된 실제 사례가 있는가?
- CAPP(Computer-Aided Process Planning) 자동화의 현재 수준과 한계는?
- 이 네 요소를 하나의 파이프라인으로 결합할 때의 기술적 갭은?

---

## 핵심 가정

1. **CadQuery는 충분히 성숙하다** — OpenCASCADE 커널 기반이므로 복잡한 기어 형상도 표현 가능할 것이라고 가정. 검증 필요.
2. **LLM은 도메인 지식을 충분히 활용할 수 있다** — RAG 또는 fine-tuning을 통해 제조 도메인 지식을 주입하면 설계 파라미터 추론이 가능하다고 가정.
3. **온톨로지가 공정 지식 표현에 적합하다** — OWL/RDF 기반 온톨로지가 제조 공정의 규칙, 제약, 관계를 표현할 수 있다고 가정.
4. **통합이 기술적으로 가능하다** — Python 생태계 내에서 이 세 요소를 연결하는 것이 아키텍처적으로 가능하다고 가정.

## 의사결정 로그

### 리서치를 4개 영역으로 분할한 이유

단일 에이전트가 전체를 조사하면 깊이가 부족해지고, 각 영역의 전문성이 다르기 때문에 분할했다.

| # | 영역 | 분할 근거 | 담당 |
| --- | --- | --- | --- |
| 1 | **CadQuery 기어 모델링 역량** | CAD 커널의 기하학적 역량은 독립적으로 평가 가능. 기어 형상(인벌류트 곡선, 언더컷 등)의 수학적 복잡성이 높아 전담 조사 필요. | Researcher-1 |
| 2 | **제조 온톨로지 현황** | 온톨로지 공학은 독립된 연구 분야. MASON, MSDL, ISO 10303 등 기존 표준/연구의 성숙도를 별도 평가 필요. | Researcher-2 |
| 3 | **LLM + CAD/CAM 통합 사례** | LLM의 CAD 코드 생성 능력은 최근 급속히 발전 중. 최신 사례(2024-2026)를 집중 추적해야 함. | Researcher-3 |
| 4 | **CAPP 공정자동화 현황** | 공정 계획 자동화는 30년 이상의 연구 역사가 있음. 기존 접근법의 한계와 AI 기반 새 접근법을 비교해야 함. | Researcher-4 |

### 왜 5개가 아닌 4개인가

- "공구 선정"과 "공정 최적화"는 CAPP의 하위 문제로 볼 수 있어, 영역 4에 포함시켰다.
- 별도 분리하면 조사 범위가 좁아져 컨텍스트 부족이 우려된다.

## 서브 에이전트 결과 요약

### 1. CadQuery 기어 모델링
- CadQuery 2.6.0, cq_gears v0.51로 인볼루트 기어 생성 가능
- Spur/Helical/Bevel A등급, Worm C등급(미지원)
- STEP export A등급, 대량 변형 자동화 A등급
- LLM 코드 생성 타겟으로 학술적 표준 (Text-to-CadQuery 90%+ 성공률)
- build123d가 차세대 대안

### 2. 제조 온톨로지
- 기어 전용 OWL 온톨로지 세계적으로 부재 → 신규 구축 필요
- IOF Core(BFO 기반)가 확장 기반으로 유력
- MASON(Entity/Operation/Resource) 구조도 확장 후보
- LLM+온톨로지 결합 급성장: Ontogenia로 OWL 초안 자동 생성, 환각률 63%→1.7% 감소
- Siemens KG 사례가 산업 적용의 선두

### 3. LLM+CAD/CAM 통합
- CAM이 CAD보다 실무 배치에서 앞서 있음 (CloudNC 1000개 샵)
- Text-to-CadQuery가 가장 유망한 경로 (GPT-4o 90%+ 실행 성공률)
- RAG+설계 표준이 즉시 적용 가능한 최고 가치 (TRL 6-8)
- 대형 CAD 벤더 AI는 문서 지원 수준에 머물러 있음

### 4. CAPP 공정자동화
- 기어 CAPP 특화 LLM 연구 사실상 부재
- KG+LLM 하이브리드가 가장 유망 (ARKNESS: 3B 모델이 GPT-4o급)
- 공구 카탈로그 RAG가 즉시 실현 가능한 최고 ROI
- KISSsoft COM API가 LLM 연동의 현실적 진입점
- LLM은 대체가 아닌 "접근성 민주화" 역할

## 의사결정 로그 (통합 단계)
- 4개 보고서 모두 KG+LLM 하이브리드를 유망한 방향으로 지목 → 높은 신뢰도
- CadQuery + LLM 코드 생성 경로가 기어 형상 생성에 가장 실현 가능
- 기어 전용 온톨로지 부재가 가장 큰 장벽 → 구축 비용이 핵심 변수
- 공구 RAG → KG 기반 공정 매핑 → KISSsoft 연동 순서가 점진적 구현 경로
