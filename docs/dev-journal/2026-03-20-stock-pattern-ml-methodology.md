## 사용자 요청 요약

- 한국 증시 눌림목 돌파 스윙 트레이딩을 하는 개인 트레이더
- VCP, CWH, HTF, W 등 패턴을 눈으로 익혔지만 실제 차트에서 읽어내기 어려움
- 기계학습/강화학습으로 확률 기반 예측이 가능한지 탐색 중
- 이미 한국 증시 DB 보유 (SQLite, 2,500종목, 2년치 일봉/주봉, 기술지표, RS)
- DB를 AI 학습용으로 확장하는 설계(feature store, event store, label store)가 이미 완료
- ChatGPT와 이전에 4단 구조(표현학습→확률예측→랭킹→실행최적화)를 논의한 바 있음

## 핵심 가정

1. 사용자의 DB 확장 설계(feature_snapshot, setup_events, forward_labels)가 구현될 예정
2. Python + SQLite 환경에서 구현
3. 데이터 규모: ~2,500종목 × ~500일 ≈ 125만 row
4. 사용자는 ML/RL 초보자이지만 트레이딩 경험은 풍부

## 의사결정 로그

### D1: 멀티에이전트 구성 — Researcher 3명 + Critic 1명

**결정**: 3개 독립 연구 축으로 분리

| 에이전트 | 역할 | 담당 영역 |
|----------|------|----------|
| R1 | Researcher | 금융 시계열 표현학습 및 패턴 인식 방법론 |
| R2 | Researcher | 이벤트 기반 확률 예측 모델 비교 |
| R3 | Researcher | RL 적용 범위 및 실전 구현 로드맵 |
| Critic | Critic | R1-R3 결과의 교차 검증 및 비판 |

**근거**: 표현학습/예측모델/RL은 각각 독립적 조사 영역이며, 관점 분리가 통합 품질에 기여한다.

### D2: 이질 도메인 유추 — 의료 영상 이상 패턴 탐지

**근거**: 정상/이상 불균형 데이터에서 형태학적 패턴 인식 구조가 유사. 소수 이상 사례에서 패턴을 검출하는 문제 구조가 눌림목 패턴 식별과 대응된다.

### D3: Phase 0 추가 결정

**결정**: Critic의 "패턴이 거래비용을 초과하는 엣지를 제공하는가" 미검증 지적 반영, Phase 0(전제 검증) 추가

**근거**: ML 파이프라인 구축 전에 패턴 자체의 통계적 유의성을 먼저 검증해야 함. 검증 실패 시 이후 단계 전체가 무의미해지므로 조기 확인이 필수.

### D4: GBT 우선 결정

**결정**: R1의 CNN/TCN Phase 1 권고 대신 R2-R3의 GBT 우선 채택

**근거**: 데이터 규모(~125만 row), 구현 복잡도, 1인 개발 환경을 고려하면 GBT(XGBoost/LightGBM)가 빠른 반복과 해석 가능성 면에서 우월.

### D5: RL 조건부 도입

**결정**: 규칙 기반 대비 우위 확인 후에만 RL 도입 (R3 제안, Critic 보정)

**근거**: RL의 보상 설계와 학습 불안정성을 감안하면, 규칙 기반 실행 대비 명확한 성과 개선이 확인된 후에 도입하는 것이 리스크 관리에 적합.

## 이슈 / 해결

- Critic이 5건의 상충점 식별: LSTM/TCN 필요성, 2D 이미지 가치, Foundation model, 타임라인, SMOTE
- 해결: GBT 우선(R2-R3 채택), 2D 이미지 제외(R2 채택), Foundation model 최하위, R3 타임라인 채택, Class weight 우선(R2 채택)
- Critic의 핵심 지적: Phase 0(전제 검증) 추가 필요 → 종합 보고서에 반영

## Rejected Alternatives

### RA1: End-to-end RL만으로 전체 매매 시스템 구축

- **기각 사유**: ChatGPT 논의에서 이미 과적합 위험으로 기각됨
- **판단**: 금융 시계열의 낮은 신호 대 잡음 비율에서 end-to-end RL은 보상 신호가 너무 희소하여 학습이 불안정

### RA2: Foundation model(TimesFM 등)만으로 해결

- **기각 사유**: 2,500종목이라는 규모에서 fine-tuning 비용 대비 효과 불명확
- **판단**: 사용자의 자원 제약(개인 트레이더)과 도메인 특수성(한국 증시 눌림목 패턴)을 고려하면, 범용 시계열 모델보다 목적 특화 접근이 우선

### RA3: Phase 1에서 CNN/TCN 시작

- **기각 사유**: 데이터 규모 부족, GBT 대비 구현 복잡도 높음
- **판단**: R1 제안이었으나 R2-R3-Critic이 반대. 1인 개발 환경에서 GBT 우선이 현실적

### RA4: 2D 차트 이미지 보조 활용

- **기각 사유**: ROI 낮음, 근거 불충분
- **판단**: R1 제안이었으나 R2-Critic이 반대. 1D 피처 기반 접근으로 충분

### RA5: Foundation model feature extractor

- **기각 사유**: zero-shot R² 음수 모델의 임베딩 유용성 근거 없음
- **판단**: R1 제안이었으나 Critic이 반대. 최하위 우선순위로 강등

### RA6: SMOTE 시계열 증강

- **기각 사유**: 시계열 구조 파괴 위험
- **판단**: R3 조건부 허용이었으나 R2-Critic이 반대. Class weight 우선 채택
