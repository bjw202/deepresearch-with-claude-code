# 세션 저널: ML 학습 데이터 구성 가이드

## 세션 메타
- 날짜: 2026-03-18
- 프로젝트: 스윙트레이딩 돌파 패턴 ML 분류
- 산출물: `docs/research/training-data-practical-guide.md`

## 사용자 요청 요약
CWH/VCP/HTF 돌파 패턴의 성공/실패를 ML로 분류하기 위한 실전 학습 데이터 구성 가이드. 데이터 소싱부터 피처 엔지니어링, 라벨링, 데이터 분할, 구현 로드맵까지 코드 수준으로 작성 요청.

## 문제 정의
4단 구조(표현학습 → 확률예측 → 랭킹 → 실행최적화)에서 가장 기초가 되는 학습 데이터 파이프라인이 없으면 아무것도 시작할 수 없다. 특히:
1. 어떤 데이터를 어디서 구하는지
2. 어떤 피처를 어떻게 계산하는지
3. 성공/실패를 어떻게 정의하는지
4. 시계열 누수 없이 어떻게 분할하는지

## 핵심 가정
- 스윙트레이딩 보유 기간: 5~20일
- 대상 시장: 미국 주식 (우선), 한국 주식 (보조)
- MVP는 XGBoost/LightGBM으로 충분
- 일봉 데이터가 기본 단위

## 의사결정 로그

### 데이터 소스 선택
- **결정**: yfinance (MVP) → Tiingo (검증) → Polygon (확장)
- **이유**: 무료, Python 네이티브, 충분한 히스토리. 유료는 데이터 품질 검증 후 전환.

### 라벨링 방식
- **결정**: MFE/MAE 기반 다중 라벨 (이진 + 확률 + 회귀)
- **이유**: 단순 방향 예측보다 기대값(EV) 계산이 트레이딩에 실질적. 4단 구조의 2단계(확률예측)와 직결.

### 데이터 분할
- **결정**: Walk-Forward (기본) + Purged K-Fold (보조 검증)
- **이유**: Walk-Forward가 실전과 가장 유사. Purged K-Fold는 CV 성능 추정치의 분산을 줄여줌.

### 정규화
- **결정**: 비율 변환 우선 → RobustScaler 보조
- **이유**: 가격/거래량의 스케일 차이를 비율로 제거하면 종목 간 비교 가능. MinMax/Standard보다 이상치에 강건한 Robust 사용.

## 이슈 / 해결
- **Bash 권한 제한**: search.sh 실행 불가 → MCP Tavily 도구로 대체하여 검색 수행
- **VCP 탐지 알고리즘 표준 부재**: 공개된 정식 구현이 없음 → 스윙 고저점 기반 수축 패턴 탐지 알고리즘 직접 설계

## Rejected Alternatives
- **분봉 데이터 기본 사용**: 데이터 크기 폭발 + 무료 소스 제한 + 스윙 주기와 불일치 → 기각
- **End-to-End RL**: 과적합 위험 + 해석 불가 → 4단 분리 구조 유지
- **SMOTE 기본 적용**: 시계열 데이터에서 합성 샘플의 의미가 불분명 → class_weight 우선 사용
- **딥러닝 (LSTM/Transformer) MVP**: XGBoost로 피처 중요도 파악이 선행되어야 함 → Phase 5로 연기
