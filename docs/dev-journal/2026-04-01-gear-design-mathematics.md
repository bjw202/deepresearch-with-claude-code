## 사용자 요청 요약

기어 설계 및 기계 가공 시뮬레이션 앱 개발을 위한 종합 수학 레퍼런스 자료 구축. 기어 파라미터 입력 → 3D 형상 모델링 → 가공 공구 설계까지의 전 과정에 필요한 수학적 요소 정리.

---

## 문제 정의

인벌류트 기어(스퍼/헬리컬/베벨/내접)의 기하학, 3D 좌표 생성, 가공 공구(호브/피니언 커터) 설계에 필요한 파라메트릭 수식을 프로그래밍 구현 가능 수준으로 정리하는 구조화된 레퍼런스.

**범위 명확화:**

- 포함: 인벌류트 치형 기하학, 3D 형상 좌표 수식, 공구 설계 수학, 기초 강도 공식
- 제외: 사이클로이드 등 비인벌류트 치형, 메쉬 시뮬레이션 (별도 리서치 권장)

---

## 핵심 가정

1. 인벌류트 기어 중심 (사이클로이드 등 비인벌류트 제외)
2. 대상 기어: 스퍼, 헬리컬, 베벨, 내접 기어 (+ 랙&피니언)
3. 공구: 호빙(hobbing) + 셰이핑(shaping) 커터 설계
4. 수학 수준: 좌표 계산 가능한 파라메트릭 수식 (프로그래밍 직접 구현 목표)
5. 강도 계산: 핵심 공식만 경량 포함 (AGMA/ISO 기초)
6. 메쉬 시뮬레이션: 제외

---

## Question Expansion (리서치 착수 전 수행)

**1. 핵심 질문**기어 3D 모델링 및 공구 설계에 필요한 모든 수학적 요소를 프로그래밍 구현 가능 수준으로 정리하라.

**2. 검증 필요한 전제**

- 인벌류트 곡선만으로 모든 대상 기어의 치형을 충분히 표현할 수 있는가?
- 해석적 수식만으로 충분한가, 수치적 방법(Newton-Raphson 등)이 필수적인 부분이 있는가?

**3. 인접 질문 (사용자가 놓쳤을 가능성)**

- 전위 기어(profile shifted gear)의 수학은 기본 기어와 어떻게 다른가?
- 베벨 기어의 구면 인벌류트(spherical involute)는 평면 인벌류트와 근본적으로 다른 접근이 필요한가?
- 가공 시뮬레이션의 정밀도를 결정하는 핵심 파라미터는 무엇인가?

**4. 결론을 뒤집을 수 있는 반대 시나리오**기존 오픈소스 라이브러리(OpenGear, FreeCAD Gear Workbench, KISSsoft API 등)의 수학 모듈을 참조하는 것이 자체 구현보다 효율적일 수 있음. 특히 베벨 기어 구면 인벌류트 계산은 구현 복잡도가 높아 라이브러리 활용이 합리적 선택일 수 있음.

**5. 이질 도메인 유추**\[이질 도메인: CAM 툴패스 생성\] — CAM의 envelope 이론(공구가 지나간 영역의 포락선 계산)은 기어 호빙 시뮬레이션의 창성(generation) 원리와 동일한 수학 구조. CAM 분야의 오프셋 곡선, 스위핑 솔리드 알고리즘을 기어 가공 시뮬레이션에 직접 차용 가능.
